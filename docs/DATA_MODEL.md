| character_consumable_items | Per-character support item inventory | `character_id`, `item_key`, `quantity` | `AdventureSupportService`, `InventoryController`, `PlayerControlManager`, `NewcomerRegistrationCampaignService`, 倉庫/補給商会 | 救助保険証、緊急救助要請、探索力の小瓶/薬など、補給商会の所持型支援アイテムを保持する。探索力回復商品は購入時、管理者の支援送付時、7月登録キャンペーンの自動配布時にここへ入り、使用時に `characters.explore_stamina` を回復する。 |

| pass_purchase_logs | Paid support pass purchase ledger | `user_id`, `character_id`, `pass_type`, `price_currency`, `price_amount`, `purchased_at`, `previous_expires_at`, `new_expires_at` | `SupportPassService` | 冒険者支援パス購入/延長時に、50輝石の購入額と前後の有効期限を保存する。輝石消費自体は既存の `kiseki_transactions` と補給商会購入ログにも残る。 |
| admin_item_grant_logs | Admin item grant audit log | `character_id`, `admin_user_id`, `grant_type`, `target_type`, `target_id`, `target_name`, `quantity`, `enhance_level`, `metadata` | `PlayerControlManager` | 管理画面のアイテム送付で、素材・武器・防具・装飾・探索アイテム・サポートアイテムの送付成功時に1件記録する。送付履歴は選択中キャラクター単位で直近20件を表示する。 |
| game_settings | Runtime operation/balance settings | `exploration.mode`, `exploration.stamina_*`, `support_pass.enabled`, `adventure_support.item_enabled.*`, `adventure_support.item_visible.*`, `adventure_support.item_campaign.*`, `extra_content.enabled.*`, `extra_content.starts_at.*`, `extra_content.ends_at.*`, existing cooldown/drop settings | `GameSettingService`, `ExplorationStaminaService`, `SupportPassService`, `AdventureSupportItemControlService`, `ExtraContentControlService`, admin reward/support/extra-content settings | `exploration.mode=cooldown` なら従来の連戦待機方式、`stamina` なら探索力消費方式。旧 `exploration.stamina_enabled` は判定に使わない。`support_pass.enabled` は冒険者支援パス全体の公開ON/OFF。`adventure_support.item_enabled.{item_key}` は補給商会商品の販売ON/OFF、`adventure_support.item_visible.{item_key}` は表示/非表示、`adventure_support.item_campaign.{item_key}.*` はキャンペーン価格/開始/終了日時。`extra_content.enabled.{content_key}` はエクスト塔/地下コンテンツの公開ON/OFF、`extra_content.starts_at/ends_at.{content_key}` は開催開始/終了日時。非表示商品は一覧に出さず購入も停止する。 |
| game_texts | Admin-editable UI/help text overrides | `key`, `value`, `description` | `GameTextService`, `FacilityTextManager`, `HelpTextManager`, `HelpContentService` | Help/案内所 keys use the `help.*` prefix; missing rows fall back to config defaults. |

| arena_npc_rankings | NPC rank-battle entries mixed into arena standings | `npc_id`, `rank`, `level`, `battle_profile`, `wins`, `losses`, `is_active` | `ArenaNpcRankingService`, `ArenaNpcBattleService`, `ColosseumScreen`, `ColosseumRanking` | Uses existing `npc_master` portraits/names. | arena_npc_logs | Player-vs-NPC rank-battle result logs | `attacker_id`, `arena_npc_ranking_id`, `npc_id`, `is_attacker_win`, old/new ranks | `ArenaNpcBattleService`, `ColosseumScreen`, `BattleController` | Kept separate from `arena_logs` because `arena_logs.defender_id` is a required character FK. |

| tower_merchant_purchases | Star-lantern merchant purchase log | `tower_run_id`, `character_id`, `floor`, `item_key`, `item_name`, `price`, `effect_type`, `effect_value`, `activated_at`, `used_at` | `TowerMerchantService`, `TowerBattleService` | Audits star-lantern merchant purchases. ## Rules

- Schema changes require migration.
- Type changes require generated/manual type update.
- RLS/security behavior must be verified when auth-sensitive.
