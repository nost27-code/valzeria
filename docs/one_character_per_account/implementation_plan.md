# 1アカウント1キャラクター制限の実装計画

Googleログインやゲストログインにおいて、1つのアカウント（User）で作成および使用できるキャラクターを1体のみに制限し、複数キャラクターの所持を禁止します。

## ユーザーレビューが必要な箇所

> [!NOTE]
> すでに複数キャラクターを所持している既存ユーザーがいる場合、この制限が適用されると自動的に「最後に更新されたキャラクター」または「最初に作成されたキャラクター」のいずれか1体が自動的に選択され、残りのキャラクターが選択できなくなります。
> 必要に応じて、余分なキャラクターを削除するなどのデータ整理が必要になる場合があります。今回の実装では、1つ以上のキャラクターが存在する場合は自動的に最初の1体が選択されて直接ホーム画面へ遷移する仕様とします。

## 提案される変更

### キャラクター選択コンポーネント

#### [MODIFY] [CharacterSelect.php](file:///c:/Users/yuta/tool/tool/ffa/app/Livewire/CharacterSelect.php)
* `mount()` メソッドにて、キャラクター数をチェックする処理を追加します。
  * キャラクター数がちょうど **1体** の場合：
    * そのキャラクターを自動的に選択し、直接 `home` 画面へリダイレクトします。これによってユーザーは「キャラクター選択」の手間を省くことができます。
  * キャラクター数が **0体** の場合：
    * 自動的に `character.create`（新規キャラ作成画面）へリダイレクトします。
* `render()` で表示される新規作成ボタンについても、すでに1体以上キャラクターが存在する場合は非表示にする（あるいはボタン自体の動作を禁止する）設計とします。

### キャラクター作成コンポーネント

#### [MODIFY] [CharacterCreate.php](file:///c:/Users/yuta/tool/tool/ffa/app/Livewire/CharacterCreate.php)
* `mount()` メソッドにて、すでにログインユーザーがキャラクターを1体以上所持しているかをチェックします。
  * 既にキャラクターが1体以上存在する場合：
    * 直接 `character.select`（または自動で `home`）へリダイレクトし、作成画面への直リンクによる不正アクセスを防ぎます。
* `create()` メソッドの最初でも同様のセキュリティガード（所持数チェック）を行い、重複作成を完全にブロックします。

## 検証計画

### 自動テスト / 手動検証
1. **新規ユーザーログイン時の動作検証**:
   * 新しいゲストアカウント等でログインした際、自動的にキャラクター作成画面 (`character.create`) にリダイレクトされることを確認。
   * キャラクターを1体作成した際、自動的に `home` 画面に遷移することを確認。
 2. **既存ユーザーログイン時（1キャラ所持）の動作検証**:
   * 1キャラ所持しているアカウントでログインした際、キャラクター選択画面が表示されず、自動的に `home` 画面に直接遷移することを確認。
3. **複数キャラ作成防止の検証**:
   * キャラ作成後にブラウザ等で直接 `/character/create` や `/character/select` にアクセスした際、作成画面に留まらず、自動的に `home` 画面にリダイレクトされることを確認。
4. **本番デプロイ後の動作確認**:
   * `local_deploy.php` によるデプロイおよび本番動作検証。
