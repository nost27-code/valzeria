# 冒険都市ヴァルゼリア UI改善追加仕様書

## ファイル名

valzeria_ui_lively_ffa_additional_spec.md

## 1. 目的

本仕様書は、冒険都市ヴァルゼリアの画面UIについて、昔のFFA風ブラウザRPGらしい賑やかさ、情報密度、オンライン感を追加するための仕様を整理するものである。

現在の画面イメージは、3カラム構成で見やすく、現代的なWebアプリとしては整理されている。

一方で、昔のCGIゲーム系FFAにあった以下の要素が弱い。

* ログインした瞬間に情報が目に入るガチャガチャ感
* 他プレイヤーが今この場にいる感覚
* 街の掲示板や酒場のような雑多な空気
* ランキング、チャット、レアドロップ、転職などが同時に流れるライブ感
* 個人サイト系ブラウザゲームの少し怪しくて楽しい雰囲気

本作は「冒険都市ヴァルゼリア」というタイトルであり、単なる育成画面ではなく、冒険者が集う都市にログインしている感覚を出す必要がある。

そのため、UIの基本方針を以下に変更する。

```text
見やすい管理画面
↓
見やすいが、街がざわついているFFA風画面
```

## 2. 基本方針

## 2.1 現在の3カラム構成は維持する

現在の画面構成は、以下のように整理されており、土台としては優れている。

* 左カラム：自分のキャラクター情報、装備、メニュー
* 中央カラム：システムログ、チャット
* 右カラム：街にいる冒険者一覧

この3カラム構成は維持する。

ただし、中央エリアと右カラムの情報量を増やし、FFAらしいオンライン感を強化する。

## 2.2 画面の目標

画面を見たユーザーが、以下のように感じることを目標にする。

* 今このゲームに人がいる
* 他の冒険者が動いている
* 街の中にログインした感じがある
* ランキングや決闘が動いている
* 自分も何か行動したくなる
* 昔のFFAっぽい懐かしさがある

## 2.3 避けたい方向性

以下の方向には寄せすぎない。

* 業務システムの管理画面
* SaaSダッシュボード
* 情報が少ないミニマルUI
* 静かすぎるRPG画面
* 白背景中心の無機質な画面
* スマホゲーム風の過度に整理されたUI

本作では、少し情報量が多いこと自体を味として扱う。

## 3. 画面全体レイアウト

## 3.1 基本レイアウト

画面は引き続き3カラム構成とする。

```text
左カラム：プレイヤー情報、装備、メニュー
中央：街の広場ログ、チャット、速報、掲示板
右カラム：ログイン中冒険者、詳細、決闘、手紙
```

## 3.2 推奨レイアウト

```text
┌───────────────────────────────────────────────┐
│ 冒険都市ヴァルゼリア                            │
│ 最強冒険者が集うFFA風RPG                      │
│ 現在の冒険者：5人｜本日の決闘：128戦｜王者：アスナ │
└───────────────────────────────────────────────┘

┌──────────┐ ┌──────────────────────┐ ┌────────────┐
│ 自分の情報 │ │ 街の広場ログ           │ │ 街にいる冒険者 │
│ ステータス │ │ システム/チャット/速報 │ │ 詳細/決闘/手紙 │
│ 装備       │ │                      │ │              │
│ メニュー   │ │ チャット入力欄         │ │              │
└──────────┘ └──────────────────────┘ └────────────┘
```

## 3.3 中央下の余白対策

現在の画面では、中央ログの下に大きな余白がある。
この余白は、FFAらしい情報密度を出すために活用する。

追加表示候補は以下。

* 本日のランキングTOP3
* 最近のレアドロップ
* 最新の転職者
* ギルド募集
* 運営からのお知らせ
* 今日のPvP連勝記録
* 新しく街に来た冒険者
* 次に解放できるエリア案内

## 4. ヘッダー仕様

## 4.1 ヘッダーの役割

ヘッダーは、現在地表示ではなく、ゲームの世界観を示す場所とする。

現在の「宿場町 - 冒険者の集う場所」のような表示は悪くないが、タイトルとしての強さが足りない。

## 4.2 表示内容

ヘッダーには以下を表示する。

* ゲームタイトル
* サブタイトル
* 現在ログイン中の冒険者数
* 本日の決闘数
* 現在のPvP王者
* 最新ニュースの短文

表示例。

```text
冒険都市ヴァルゼリア
最強冒険者が集うFFA風RPG

現在の冒険者：5人｜本日の決闘：128戦｜現在の王者：アスナ
```

## 4.3 ヘッダー内速報

ヘッダー下部に短い速報を流してもよい。

例。

```text
速報：ギリトが魔王城に到達しました
速報：名無しの槍兵が伝説の剣を手に入れました
速報：黒猫旅団がギルドメンバーを募集しています
```

初期実装では、アニメーションで流す必要はない。
まずは固定表示または一定時間ごとの差し替えでよい。

## 5. 左カラム仕様

## 5.1 左カラムの役割

左カラムは、自分の現在状態と主要メニューを表示する場所とする。

現在の構成は良いが、FFAらしさを強めるために、以下を追加する。

* PvPランク
* 転職回数
* 次の目標
* 新着手紙数
* 所属ギルド
* 現在の称号

## 5.2 キャラクターカード表示項目

表示項目は以下。

* キャラクター名
* アイコン
* レベル
* 職業
* PvPランク
* 転職回数
* 所属ギルド
* 称号
* HP
* EXP
* GOLD

表示例。

```text
テストプレイ
Lv.1 / 僧侶
PvPランク：F
転職回数：0
所属ギルド：未所属
称号：駆け出し冒険者
```

## 5.3 ステータス表示

ステータスは現在の英字略称を活かす。

* STR：攻撃
* DEF：防御
* AGI：素早さ
* MAG：魔力
* LUK：運

ただし、アイコンや小さな装飾を足してゲーム感を出す。

表示例。

```text
⚔ STR 9
🛡 DEF 9
💨 AGI 8
🔮 MAG 12
🍀 LUK 6
🪙 GOLD 100G
```

## 5.4 装備表示

現在の装備表示は見やすいが、少し事務的である。

装備カテゴリには以下の表示を使う。

* 武：武器
* 鎧：防具
* 飾：アクセサリー

また、装備のレアリティに応じて色を変える。

| レアリティ  | 表示色イメージ |
| ------ | ------- |
| normal | 白または薄灰  |
| rare   | 青       |
| epic   | 紫       |
| legend | 金       |

## 5.5 メニュー項目

左メニューは現在より増やす。

初期表示するメニューは以下。

* 宿場町
* 迷宮へ向かう
* 闘技場
* 武具屋
* 転職所
* ギルド会館
* 手紙箱
* ランキング
* 冒険者名簿
* お知らせ

未実装の機能は「準備中」表示でもよい。
昔のFFAらしさを出すには、メニューが少し多い方が雰囲気が出る。

## 5.6 次の目標表示

左カラムに「次の目標」を表示する。

例。

```text
次の目標
小鬼の森のボスを倒して、古びた洞窟を解放しよう
```

または、

```text
次の目標
Lv100を目指して転職条件を満たそう
```

これにより、初心者が次に何をすべきか迷いにくくなる。

## 6. 中央カラム仕様

## 6.1 中央カラムの役割

中央カラムは「街の広場」として扱う。

単なるシステムログではなく、以下の情報が流れる場所にする。

* システムログ
* チャット
* 決闘速報
* レアドロップ
* 転職通知
* エリア到達通知
* ギルド募集
* 運営お知らせ

## 6.2 タブ構成

中央ログにはタブを設ける。

初期タブ案。

* 全体
* システム
* チャット
* 決闘
* ギルド
* お知らせ

表示例。

```text
[全体] [システム] [チャット] [決闘] [ギルド] [お知らせ]
```

初期実装で全タブの中身が完成していなくても、UI上の導線として用意してよい。

## 6.3 ログ表示例

中央ログには、以下のような文言を表示する。

```text
【システム】ギリトさんが魔王城に到達しました！
【チャット】アスナ：今日はスライム狩りに行きます。
【システム】名無しの槍兵さんが伝説の剣を手に入れました！
【転職】初心者さんが戦士からナイトへ転職しました。
【決闘】アスナさんがギリトさんに勝利しました。
【ギルド】黒猫旅団がギルドメンバーを募集しています。
【到達】ミケさんが精霊の神殿を突破しました。
```

## 6.4 ログ種別と色

ログ種別ごとに色を変える。

| 種別     | 色イメージ | 用途     |
| ------ | ----- | ------ |
| システム   | 黄     | 重要通知   |
| チャット   | 緑     | 通常会話   |
| 決闘     | 赤     | PvP結果  |
| 転職     | 紫     | 転職通知   |
| レアドロップ | 金     | レア装備入手 |
| ギルド    | 青     | ギルド関連  |
| お知らせ   | 水色    | 運営通知   |

現在の黒背景ログは良いので維持する。
ただし、表示するログ種類を増やして、街が動いている感じを出す。

## 6.5 チャット入力欄

中央下にチャット入力欄を配置する。

現在の入力欄はシンプルで良いが、ゲーム内チャット感を出すために、以下を追加する。

* 入力欄の左に発言種別を選ぶプルダウン
* 送信ボタンを少しゲーム風にする
* 発言後、即時ログに反映する

発言種別案。

* 全体
* ギルド
* 個人宛て

ただし、個人宛てはメッセージ機能と分けてもよい。

## 7. 右カラム仕様

## 7.1 右カラムの役割

右カラムは「街にいる冒険者」を表示する場所とする。

現在は名前、レベル、職業、詳細ボタンのみであり、情報量が少ない。
ここを強化することで、オンライン感とPvP導線を作る。

## 7.2 表示項目

ログイン中プレイヤーには以下を表示する。

* キャラクター名
* レベル
* 職業
* PvPランク
* 所属ギルド
* 現在状態
* 詳細ボタン
* 決闘ボタン
* 手紙ボタン

表示例。

```text
ギリト
Lv45 / 魔法剣士 / Bランク
所属：黒猫旅団
状態：迷宮探索中

[見る] [決闘] [手紙]
```

## 7.3 現在状態

現在状態は、最終行動から推定して表示する。

例。

| 状態       | 条件          |
| -------- | ----------- |
| 宿場町に滞在中  | 最後にホームを表示   |
| 迷宮探索中    | 最後に通常戦闘を実行  |
| 闘技場に滞在中  | 最後にPvP画面を表示 |
| 武具屋で買い物中 | 最後にショップを表示  |
| 転職所に滞在中  | 最後に転職画面を表示  |

厳密でなくてよい。
雰囲気を出すための表示として扱う。

## 7.4 冒険者詳細

名前または「見る」ボタンを押すと、公開ステータス画面に遷移する。

公開ステータスで表示する項目は以下。

* キャラクター名
* アイコン
* レベル
* 職業
* 転職回数
* PvPランク
* PvPレート
* 所属ギルド
* 装備
* 勝敗数
* 称号
* 自己紹介
* 最終ログイン時刻

Googleアカウントのメールアドレスや内部IDは表示しない。

## 7.5 決闘ボタン

「決闘」ボタンから、対象プレイヤーにPvPを挑めるようにする。

初期実装では、非同期PvPとする。
相手がログイン中でなくても戦えるが、ログイン中一覧から挑めることで昔のFFA感が出る。

## 7.6 手紙ボタン

「手紙」ボタンから、対象プレイヤーへメッセージを送れるようにする。

メッセージ機能が未実装の場合は、以下の表示でよい。

```text
手紙機能は準備中です
```

## 8. 街の情報パネル追加仕様

## 8.1 追加パネルの目的

中央ログ下の余白を活用し、街が動いている感じを出す。

表示するパネルは、すべて一度に出す必要はない。
画面サイズに応じて、2個から4個表示する。

## 8.2 本日のランキングTOP3

PvPランキングまたは総合ランキングの上位3名を表示する。

表示例。

```text
本日の決闘ランキング

1位 アスナ Sランク 12勝
2位 ギリト Aランク 9勝
3位 ミケ Bランク 7勝
```

## 8.3 最近のレアドロップ

直近のレア装備入手ログを表示する。

表示例。

```text
最近のレアドロップ

名無しの槍兵が「黒鉄の大剣」を入手
アスナが「聖銀のローブ」を入手
ミケが「俊足の指輪」を入手
```

## 8.4 ギルド募集

ギルド募集情報を表示する。

表示例。

```text
ギルド募集

黒猫旅団：初心者歓迎
竜牙騎士団：PvP好き募集
まったり宿場町：雑談中心
```

## 8.5 運営からのお知らせ

TOPページだけでなく、ログイン後画面にもお知らせを表示する。

表示例。

```text
お知らせ

6/5 βテストを開始しました
6/6 小鬼の森を追加予定
```

## 9. 配色・デザイン方針

## 9.1 現在の課題

現在の画面は白と薄いグレー中心で、清潔感はある。
しかし、冒険都市、FFA、レトロブラウザRPGとしては少し上品すぎる。

## 9.2 推奨配色

推奨する色味は以下。

| 役割     | 色味                 |
| ------ | ------------------ |
| 背景     | 薄い石壁色、薄ベージュ、灰青     |
| ヘッダー   | 黒鉄色、濃紺、深い紫         |
| メインパネル | 羊皮紙風の薄いクリーム色、または薄灰 |
| ログ背景   | 黒に近い濃紺             |
| アクセント  | 金、赤、紫、青            |
| 通常テキスト | 濃紺または黒             |
| 重要テキスト | 金または赤              |

## 9.3 レトロ感の足し方

全体を古臭くする必要はない。
一部だけレトロ感を入れる。

例。

* ログ画面は黒背景に色付き文字
* メニューは少しリンク集っぽくする
* 冒険者一覧は表形式に近づける
* 重要ログには【速報】【転職】【決闘】などのラベルを使う
* 罫線や小さなアイコンを増やす
* 装備や職業名に色を付ける

## 9.4 避けるデザイン

以下は避ける。

* 余白が広すぎる
* 白背景が多すぎる
* ボタンがすべて同じ色
* ログが少なすぎる
* メニューが少なすぎる
* 重要情報が上部に出ない

## 12. 実装メモ

## 12.1 オンライン判定

ログイン中プレイヤーは、厳密なログインセッションではなく、最終アクセス時刻で判定する。

```text
last_accessed_at が現在時刻から5分以内
```

この条件を満たすキャラクターを「街にいる冒険者」として表示する。

## 12.2 ログ生成タイミング

以下のイベント発生時に、公開ログを作成する。

* レア装備入手
* PvP勝利
* PvP連勝
* 転職
* 上位職解放
* エリア初到達
* ボス撃破
* ギルド作成
* ギルド募集
* 新規冒険者登録

## 12.3 ログ保存テーブル案

```text
public_logs
```

保持項目。

| カラム                 | 内容       |
| ------------------- | -------- |
| id                  | 主キー      |
| type                | ログ種別     |
| character_id        | 関連キャラクター |
| target_character_id | 対象キャラクター |
| guild_id            | 関連ギルド    |
| message             | 表示文      |
| importance          | 重要度      |
| created_at          | 作成日時     |

## 12.4 ログ表示件数

中央ログには、直近30件程度を表示する。

古いログはページングまたは「もっと見る」で表示する。

## 12.5 画面更新

初期実装では、完全リアルタイムでなくてよい。

以下のいずれかで実装する。

* ページ遷移時に更新
* 30秒ごとにポーリング
* 手動更新ボタン

まずはページ遷移時更新でよい。
余裕があれば30秒ごとの自動更新を入れる。

## 13. 画面文言例

## 14. 結論

現在の画面は見やすく、基本構成としては問題ない。
ただし、冒険都市ヴァルゼリアが目指すFFA風ブラウザRPGとしては、まだ画面が静かすぎる。

改善の方向性は、UIを壊すことではなく、情報密度とライブ感を追加することである。

特に重要なのは以下。

* 3カラム構成は維持する
* ヘッダーに正式タイトルとサブタイトルを出す
* 現在の冒険者数を目立たせる
* 中央ログを街の広場として扱う
* 決闘、転職、レアドロップ、ギルド募集をログに流す
* 右カラムの冒険者一覧から詳細、決闘、手紙へ行けるようにする
* 左メニューを増やし、昔のFFAらしいリンク集感を出す
* 中央下の余白にランキング、レアドロップ、お知らせを表示する

目指す画面は、単なる管理画面ではない。

```text
見やすいけれど、街がざわついている画面
```

この方向で調整することで、昔のFFAらしさと現代的な遊びやすさを両立できる。
