# 職業マスター時の永続ボーナス実装計画

## 目的
あらゆる職業をマスターするモチベーションを強化するため、かつて廃止されていた「マスターボーナス（永続パッシブ）」を、既存の `job_classes` テーブルの `bonus_xxx` の値を用いて復活させます。

## 実装方針（アプローチ）
1. **マスターボーナスの定義**:
   - 職業のランクを★10（`is_mastered = true`）にした際、その職業が持つ `bonus_hp` や `bonus_str` などの設定値を「**永続ステータスボーナス（パッシブ）**」としてキャラクターに付与します。
   - 例: 「戦士（HPボーナス20）」と「剣士（ATKボーナス20）」をマスターしている場合、現在「魔法使い」に転職していても、常に基礎ステータスとして **HP+20, ATK+20** が加算され続けます。

2. **先ほどの「現在の職業ランク補正」との違い**:
   - 先ほど実装した「現在の職業ランク補正（★の数 × ボーナス値 × 0.5）」は、**現在の職業に対する補正（装備品などと同じ＋扱いの緑色文字）**として引き続き残します。
   - 今回追加する「永続ボーナス」は、**キャラクター自身の素の強さ（ベースステータス、白色文字）**に加算されるように処理を分けます。
   - これにより「素のステータスが高い歴戦の勇者」であることを視覚的にも表現できます。

3. **実装箇所**:
   - `app/Services/JobService.php` の `calculateFinalStats()` メソッドを改修します。
   - このメソッドはキャラクターの「ベースステータス（素の能力）」を返しているため、ここで「自分がマスターしている全職業の `bonus_xxx` の総和」を取得し、それぞれのベースステータスに加算して返すようにします。

## 変更対象ファイル
### `app/Services/JobService.php`
- [MODIFY] `calculateFinalStats()` メソッド内に、マスター済み職業の `bonus_xxx` を集計して `baseStats` に足し合わせるロジックを追加。
