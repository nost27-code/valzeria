# 鍛冶屋（合成機能）の実装計画

## 概要
街の施設画面（宿屋の横）に新たに「鍛冶屋」を配置し、先ほどインポートしたレシピデータをもとにアイテムを合成できる機能を追加します。
各街ごとに、その街（`city_name`）に紐づいたレシピのみが作成可能な仕組みとします。

## オープンクエスチョン・確認事項
> [!IMPORTANT]  
> 1. 現在、武具屋（Shop）の画面の中にも「鍛冶屋（準備中）」のボタンが存在しますが、今回は「街の施設一覧（宿屋の隣）」に新設するという認識でよろしいでしょうか？（武具屋の中のボタンは削除、または別の強化施設として残すかご相談させてください）
> 2. 生成したアイコンと背景画像をプロジェクト内にコピーし、以下のようなファイル名で配置しようと考えています。問題ないでしょうか？
>    - シンボルアイコン: `facilities/blacksmith_symbol.png`
>    - 背景画像: `facilities/blacksmith.png`

## 提案する実装内容

### 1. 鍛冶屋画像の配置
- 先ほどAI生成したシンボル画像と背景画像を、それぞれ `public/images/facilities/` フォルダへコピー・配置します。

### 2. 街の画面（MainScreen）の改修
- `app/Livewire/MainScreen.php` の `town` 施設リストにて、宿屋の横（次）に「鍛冶屋」を追加します。
  - 名称: 鍛冶屋
  - アクション: 「入る」
  - ルート: `smith.index`

### 3. コントローラーとビューの実装
- `app/Http/Controllers/SmithController.php` を作成します。
- `resources/views/smith/index.blade.php` を作成し、ゲームの既存UI（スライダーや確認モーダル等）に合わせた世界観の画面を作ります。

### 4. 街ごとのレシピ出し分けロジック
- 鍛冶屋のトップ画面で、キャラクターの現在の街（`$character->currentCity->name` 等）を取得します。
- `Recipe` モデルから、その街に該当するレシピ（`city_name` が一致するもの）のみを検索・一覧表示します。

### 5. 合成処理（トランザクション保護）
- 「合成」ボタンを押した際、必要な素材をキャラクターが持っているか（個数・キー素材の解放条件など）を確認します。
- 条件を満たしている場合、素材（`character_materials` など）を減らし、完成アイテムを付与する処理をトランザクション内で安全に実行します。

## 検証手順
- アークレアなど、レシピが存在する街で鍛冶屋に入れるか確認。
- 該当する街のレシピのみが表示されるか確認。
- 実際に素材を消費して武具を作成できるか確認。
