# 闘技場システムの実装計画

ユーザー様からのご要望に基づき、「1位を目指す順位制のランク戦」をメインとした闘技場システムの実装計画を作成いたしました。

## ゴール
* 現在の階級制（F〜Sなど）とは異なる、**「サーバー内での絶対的な順位（1位、2位…）」**を競うシステムへの刷新。
* メインコンテンツを「ランク戦」に絞り、シンプルかつ熱中できるPvP（対人戦）を実装する。
* 自分がログインしていない間の「防衛・被弾ログ」や、対戦ログを確認できる機能の追加。

## 仕様 (Approved)

1. **順位の決定方式（1ランクアップ制）**
   自分より上位の相手に挑戦して勝利した場合、**「自分の順位が1つ上がり、自分の1つ上にいたプレイヤーの順位が1つ下がる（自分と1つ上の人が入れ替わる）」**方式。
   ※例：10位のプレイヤーが5位に勝つと、10位だった自分が9位になり、元9位のプレイヤーが10位に落ちる。負けた5位のプレイヤーの順位は変動しない。
2. **挑戦回数の制限**
   制限なし（スタミナ等なしで何度でも挑戦可能）
3. **「挑む」ボタンのマッチング**
   上位プレイヤーをターゲットにしたランク戦として、ランダムマッチ等の「挑む」機能を提供する。

## Proposed Changes

### 1. データベースの拡張 (Database Migration)
闘技場専用の情報を管理するため、以下のテーブル・カラムを追加します。

* **`arena_rankings` テーブル（新規）**
  * `character_id`: キャラクターID
  * `rank`: 現在の順位 (整数)
  * ※闘技場画面を初めて開いた際に、現在の参加者数＋1位（最下位）として登録される仕組み。
* **`arena_logs` テーブル（新規）**
  * `attacker_id`: 挑んだキャラクター
  * `defender_id`: 挑まれたキャラクター
  * `is_attacker_win`: 挑戦者の勝利か否か
  * `attacker_old_rank`, `attacker_new_rank`: 挑戦者の順位変動
  * `defender_old_rank`, `defender_new_rank`: 防衛者の順位変動
  * `created_at`: 対戦日時

### 2. Livewireコンポーネント (UI)
**[NEW] `app/Livewire/ColosseumScreen.php`**
**[NEW] `resources/views/livewire/colosseum-screen.blade.php`**
* **画面構成:**
  * **ヘッダー**: 自分の現在の順位、勝敗数などを表示。
  * **ランキングエリア**: トップランカー（1〜3位）と、自分の順位周辺のプレイヤーを表示。
  * **対戦アクション**: 「ランク戦に挑む」ボタン。
  * **対戦ログエリア**: `arena_logs` から、自分が関わった最新のログ（攻撃・防衛）を時系列で表示。

### 3. バトルロジック (Services)
**[NEW] `app/Services/PvPBattleService.php`**
* 既存のPvE（対モンスター）のBattleServiceをベースにしつつ、キャラクター対キャラクター専用の計算式やスキル処理を行うサービス。
* バトル終了時に勝敗を判定し、`arena_rankings` の順位入れ替えと `arena_logs` へのログ記録を実行する。

## Verification Plan
### Automated Tests
* なし (PvPバトルのユニットテストが存在しないため)
### Manual Verification
* ブラウザ上で複数のキャラクターを作成し、それぞれで闘技場に参加（初期順位の付与）できるか確認。
* 下位のキャラクターが上位に挑み、勝利した際に正しく順位が入れ替わり、ログに反映されるか確認。
* 敗北した場合は順位が変動せず、敗北ログが残るか確認。
