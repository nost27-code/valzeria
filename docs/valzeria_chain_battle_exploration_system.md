\# ヴァルゼリアの冒険者 連戦・探索度システム実装指示書



\## 1. 目的



本仕様は、「ヴァルゼリアの冒険者」において、通常探索を単発戦闘の繰り返しではなく、連続して戦うほど価値が高まるゲーム体験へ拡張することを目的とする。



現在の基本ループは、ダンジョン探索、戦闘、報酬獲得、HP/MP消耗、宿屋回復、再探索で構成される。連戦・探索度システムを追加することで、プレイヤーに以下の判断を発生させる。



\- もう1戦続けるか

\- 回復薬を使って探索を継続するか

\- 探索度を維持したまま深く潜るか

\- 宿屋に戻って安全にリセットするか

\- 探索度300以上を狙って特殊敵や宝箱を狙うか



本作は、短時間で戦闘を繰り返しながら成長するFFA風ブラウザRPGであり、通常戦闘のテンポ、収集欲、成長実感が中心体験である。連戦システムは、この中心体験を強化するために導入する。:contentReference\[oaicite:0]{index=0}



\---



\## 2. 基本方針



連戦システムでは、プレイヤーがダンジョン内で戦闘を継続するたびに「探索度」が上昇する。



探索度は永続成長ではなく、現在の探索中だけ有効な一時的な値とする。



宿屋で休む、街へ帰還する、敗北する、ボス挑戦へ移行する、などのタイミングで探索度はリセットされる。



探索度は、単純なステータス永続強化ではなく、「深く探索している状態」を表す値である。



探索度が高まるほど、以下のような特殊イベントが解放される。



\- 輝く宝箱

\- 黄金ゴブリン

\- ダンジョン主

\- 秘境への入口



初期実装では、探索度100、200、300の3段階を実装対象とする。探索度500の秘境への入口は将来拡張枠として設計だけ用意する。



\---



\## 3. 用語定義



\### 3.1 探索度



現在のダンジョン探索における進行深度を表す値。



戦闘勝利や特殊イベントによって加算される。



宿屋、帰還、敗北などで0に戻る。



\### 3.2 連戦数



現在の探索中に連続して勝利した戦闘回数。



探索度とは別管理とする。



探索度はイベント解放に使う。  

連戦数は称号、戦闘結果表示、将来のランキング・実績に使える。



\### 3.3 探索中状態



プレイヤーが街に戻らず、同じダンジョン内で探索を続けている状態。



この状態では、HP/MP、持込アイテム、探索度、連戦数が維持される。



\### 3.4 帰還



プレイヤーが現在の探索を終了し、街へ戻ること。



帰還時、探索度と連戦数はリセットする。



HP/MPをどこまで回復するかは別仕様とする。初期実装では、帰還のみでは回復せず、宿屋で回復する。



\---



\## 4. 探索度の加算ルール



探索度は、戦闘勝利時に加算する。



初期実装では、敵種別ごとに以下の加算値を設定する。



| 敵種別 | 探索度加算 |

|---|---:|

| 通常敵 | +10 |

| やや強い敵 | +15 |

| レア敵 | +25 |

| 黄金ゴブリン | +0 |

| ダンジョン主 | +0 |

| 通常ボス | 対象外 |



補足：



\- 通常探索で出現する敵のみ探索度加算対象とする。

\- ボス挑戦は通常探索とは別枠のため、探索度加算対象外とする。

\- 黄金ゴブリンやダンジョン主は探索度によって出現する特殊イベントのため、撃破しても探索度は増やさない。

\- 将来的に「深層敵」「異界敵」などを追加する場合は、個別に加算値を設定できるようにする。



\---



\## 5. 探索度のリセット条件



探索度は以下のタイミングで0に戻す。



| 条件 | 探索度 | 連戦数 |

|---|---:|---:|

| 宿屋で休む | 0 | 0 |

| 街へ帰還する | 0 | 0 |

| 戦闘に敗北する | 0 | 0 |

| ダンジョンを変更する | 0 | 0 |

| ボス挑戦を開始する | 0 | 0 |

| ログアウト | 維持しない方針 | 維持しない方針 |



初期実装では、ログアウトやセッション切れ時も探索度は維持しない。



理由は、探索度が「現在の冒険の勢い」を表す値であり、永続リソースではないため。



\---



\## 6. 探索度イベント



探索度が一定値以上になると、通常探索時に特殊イベントの抽選対象が解放される。



イベントは「探索する」実行時、通常敵の抽選前に判定する。



判定順は以下とする。



1\. 探索度500以上：秘境への入口抽選

2\. 探索度300以上：ダンジョン主抽選

3\. 探索度200以上：黄金ゴブリン抽選

4\. 探索度100以上：輝く宝箱抽選

5\. 上記に当選しない場合、通常敵と戦闘



ただし、初期実装では探索度500の秘境への入口は未実装でよい。

\## 7. 探索度100：輝く宝箱



\### 7.1 概要



探索度100以上で解放される小報酬イベント。



通常探索中、一定確率で「輝く宝箱」を発見する。



戦闘は発生せず、即時報酬を獲得する。



\### 7.2 出現条件



| 項目 | 内容 |

|---|---|

| 必要探索度 | 100以上 |

| 抽選タイミング | 探索実行時 |

| 基本出現率 | 5% |

| 出現後 | 探索度は維持 |

| 1探索中の出現回数 | 制限なし |



\### 7.3 報酬内容



輝く宝箱からは、以下のいずれかをランダムで入手する。



| 報酬種別 | 確率 | 内容 |

|---|---:|---|

| ゴールド | 45% | 現在ダンジョンの標準獲得Gold × 3〜5 |

| 薬草・回復薬 | 25% | エリアランクに応じた回復アイテム |

| 素材 | 20% | そのダンジョンに対応する素材 |

| 印 | 8% | 低〜中レア印 |

| レア装備 | 2% | そのエリア帯の低確率装備 |



初期実装では、レア装備は未実装でもよい。  

その場合、レア装備2%は素材または印に振り分ける。

7.5 役割



輝く宝箱は、連戦の最初のご褒美として機能する。

8\. 探索度200：黄金ゴブリン

8.2 出現条件

9\. 探索度300：ダンジョン主

9.1 概要



探索度300以上で出現する、通常ボスとは別枠の強敵。



通常ボスはダンジョンクリア用。

ダンジョン主は周回・やり込み用。



各ダンジョンに1体ずつ設定できる。



9.2 出現条件

項目	内容

必要探索度	300以上

抽選タイミング	探索実行時

基本出現率	2%

出現回数制限	1日1回撃破まで推奨

出現後	戦闘開始前に挑戦確認を出す

敗北時	探索度リセット

勝利時	探索度リセットまたは維持を選択可



初期実装では、勝利時も探索度をリセットする。



理由は、ダンジョン主を倒した時点で今回の探索の区切りにした方がわかりやすいため。



9.3 挑戦確認



ダンジョン主は強敵のため、即戦闘ではなく確認画面を挟む。



表示例：



周囲の空気が変わった……。



このダンジョンを支配する強敵、

「森の暴君グランドゴブリン」が姿を現した！



挑戦しますか？



\[挑戦する]

\[今は退く]



「今は退く」を選んだ場合、探索度は維持して通常探索画面に戻る。



ただし、同一探索中に何度も出ると煩雑なため、退いた場合は一定回数、または同一探索中は再出現しないようにしてもよい。



初期実装では、退いた場合は同一探索中再出現なしとする。



9.4 敵性能方針



ダンジョン主は、通常ボスより少し弱いか同程度にする。



通常ボスより強くしすぎると、探索度300まで到達したうえで負ける体験が重くなるため、初期実装では「通常ボスの90〜110%」程度にする。

ATK	通常ボスの90〜105%

DEF	通常ボスの90〜105%

MAG	通常ボスの90〜105%

SPR	通常ボスの90〜105%

SPD	通常ボスの90〜110%

EXP	通常敵より多い

Gold	通常敵より多い

職業EXP	通常敵より多い

ドロップ	固有素材、固有印、低確率固有装備

ダンジョン主は、通常ボス撃破後もそのダンジョンを周回する理由を作る。



70ダンジョンある構成なら、各ダンジョンに1体ずつ配置することで、長期的なやり込み対象にできる。



10\. 探索度500：秘境への入口

10.1 概要



探索度500以上で発見できる将来拡張用イベント。



初期実装では未実装でよい。



探索度500到達時に、低確率で「秘境への入口」が出現する。



秘境は、通常ダンジョンとは別の短期隠しエリアであり、特殊な敵、宝箱、印、素材、称号を入手できる。



10.2 将来仕様案

項目	内容

必要探索度	500以上

基本出現率	1%

入場条件	確認画面で入場選択

秘境内戦闘回数	3〜5戦

最奥	秘境ボスまたは特別宝箱

報酬	秘境限定素材、印、称号

帰還後	探索度リセット

10.3 表示文言例

目の前に、見たことのない小道が現れた。

「秘境への入口」を発見した！



入りますか？

10.4 役割



秘境は、探索度300以降もさらに潜る理由を作る上級者向け要素。



初期実装では入れず、アップデート要素として残す。



11\. 回復薬システム

11.1 目的



連戦を成立させるため、探索中に使用できる回復アイテムを導入する。



宿屋で回復すると探索度がリセットされるため、プレイヤーは回復薬を使うことで探索度を維持したまま連戦を継続できる。



11.2 初期実装アイテム

アイテム名	効果	主な用途

薬草	HPを30%回復	序盤の継戦

回復薬	HPを60%回復	中盤以降の主力

高級回復薬	HPを100%回復	ダンジョン主・深層用

魔力水	MPを30%回復	魔法職・必殺技用

生命の雫	HP/MPを50%回復	レア消耗品



初期実装では、薬草、回復薬、魔力水の3種類だけでもよい。



11.3 持込上限



回復アイテムは、所持数とは別に「探索への持込上限」を設ける。



アイテム名	持込上限

薬草	10

回復薬	5

高級回復薬	2

魔力水	3

生命の雫	1



所持数は倉庫・アイテム欄に保持する。

探索開始時、持込可能数まで自動的に持ち込むか、将来的に手動選択できるようにする。



初期実装では、自動持込でよい。



例：



薬草を20個所持している場合、探索中に使える薬草は10個

回復薬を3個所持している場合、探索中に使える回復薬は3個

探索終了後、未使用分はそのまま所持数として残る

11.4 使用タイミング



探索画面、または戦闘結果画面から使用できるようにする。



戦闘中使用は初期実装では不要。



理由は、戦闘中にアイテム使用を入れると戦闘ロジックが複雑になるため。



初期実装では、戦闘終了後に以下ボタンを表示する。



\[探索を続ける]

\[回復薬を使う]

\[街へ帰還する]



HPまたはMPが減っている場合のみ、回復アイテム使用ボタンを表示する。

13\. データベース設計案

13.1 player\_exploration\_states テーブル



現在の探索状態を管理する。



CREATE TABLE player\_exploration\_states (

&#x20;   id BIGINT UNSIGNED AUTO\_INCREMENT PRIMARY KEY,

&#x20;   player\_id BIGINT UNSIGNED NOT NULL,

&#x20;   dungeon\_id BIGINT UNSIGNED NOT NULL,

&#x20;   exploration\_point INT NOT NULL DEFAULT 0,

&#x20;   chain\_count INT NOT NULL DEFAULT 0,

&#x20;   is\_active TINYINT(1) NOT NULL DEFAULT 1,

&#x20;   dungeon\_lord\_encountered TINYINT(1) NOT NULL DEFAULT 0,

&#x20;   started\_at DATETIME NULL,

&#x20;   updated\_at DATETIME NULL,

&#x20;   created\_at DATETIME NULL,

&#x20;   UNIQUE KEY uq\_player\_active\_exploration (player\_id, is\_active)

);



注意：



MySQLでは player\_id, is\_active のユニーク制約だと、非アクティブ履歴を複数持ちにくい場合がある。

履歴を残す必要がなければ、プレイヤーごとに1レコード更新方式でもよい。



初期実装では、履歴不要。

player\_id 単位で1レコードを保持し、ダンジョン変更時にリセット更新する方式でよい。



推奨カラム：



カラム	内容

player\_id	プレイヤーID

dungeon\_id	現在探索中のダンジョンID

exploration\_point	探索度

chain\_count	連戦数

dungeon\_lord\_encountered	同一探索中にダンジョン主に遭遇済みか

updated\_at	更新日時

14\. 探索処理フロー

14.1 通常探索開始

プレイヤーがダンジョン画面で「探索する」を押す

現在の探索状態を取得

探索状態が存在しない場合は作成

現在ダンジョンと探索状態のdungeon\_idが異なる場合、探索度と連戦数をリセット

探索度イベント抽選を実行

イベントに当選した場合、イベント処理へ

当選しない場合、通常敵抽選

戦闘実行

勝利時、探索度と連戦数を加算

敗北時、探索状態をリセット

14.2 探索度イベント抽選



疑似コード：



$state = getExplorationState($playerId, $dungeonId);

$events = getEnabledEventsOrderByPriority();



foreach ($events as $event) {

&#x20;   if ($state->exploration\_point < $event->required\_exploration\_point) {

&#x20;       continue;

&#x20;   }



&#x20;   if ($event->event\_key === 'dungeon\_lord') {

&#x20;       if ($state->dungeon\_lord\_encountered) {

&#x20;           continue;

&#x20;       }



&#x20;       if (hasDefeatedDungeonLordToday($playerId, $dungeonId)) {

&#x20;           continue;

&#x20;       }

&#x20;   }



&#x20;   if (randomPercent() <= $event->base\_rate) {

&#x20;       return $event;

&#x20;   }

}



return null;

if ($battleResult->isWin()) {

16\. ボス挑戦との関係



通常ボスは、探索度システムとは分離する。



ボス挑戦を開始した時点で、探索度と連戦数はリセットする。



理由：



探索度を溜めてから通常ボスを倒すだけの最適解を避ける

通常ボスは進行管理、探索度は周回やり込みとして役割を分ける

ボス攻略は装備・職業・レベルによる準備を重視する



ただし、将来的に「探索度を一定まで溜めるとボスに追加報酬」などは検討可能。



初期実装では入れない。



17\. ステータス上昇効果について



探索度によるATK/DEF上昇などのステータスバフは、初期実装では採用しない。



理由：



探索度を溜めてからボスへ行く最適解が生まれやすい

連戦が必須化しやすい

バランス調整が難しい

探索度の役割が「発見・収集」から「火力強化」に寄りすぎる



将来的に導入する場合は、通常探索中のみ有効な一時バフとする。



例：



探索度	効果

100	通常探索中のみATK+3%

200	通常探索中のみDEF+3%

300	通常探索中のみSPD+3%



ただし、初期実装では見送る。

22\. 実装時の注意点

22.1 連戦を必須化しすぎない



探索度報酬を強くしすぎると、毎回探索度300まで行くのが前提になってしまう。



初期実装では、探索度イベントは「得をする寄り道」に留める。



22.2 通常ボス攻略と分離する



探索度によって通常ボスが楽になる設計は避ける。



通常ボスはレベル、装備、職業育成で倒す。

探索度は周回、金策、宝箱、ダンジョン主用にする。



22.3 回復薬の価格に注意する



回復薬が安すぎると無限連戦になる。

高すぎると誰も使わない。



初期価格案：



アイテム	価格

薬草	50G

回復薬	300G

高級回復薬	1,500G

魔力水	400G

生命の雫	非売品または5,000G以上

22.4 序盤は軽くする



序盤から探索度300を前提にしない。



最初の街では、



探索度100の宝箱

探索度200の黄金ゴブリン



まででも十分。



ダンジョン主は、2〜3番目のダンジョンから解放してもよい。



23\. 最終的な設計意図



連戦・探索度システムの本質は、プレイヤーに「もう少しだけ潜る理由」を与えることである。



単発戦闘だけだと、探索は作業になりやすい。



しかし、探索度があることで、



あと少しで宝箱が出るかもしれない

もう少しで黄金ゴブリンが出る

探索度300まで行けばダンジョン主に会える

回復薬を使えば、まだ帰らずに済む



という判断が生まれる。



これにより、宿屋は単なる回復施設ではなく、「今回の探索を終える場所」になる。



回復薬は単なる消耗品ではなく、「探索を継続するためのリソース」になる。



通常探索は単なるレベル上げではなく、「深く潜る遊び」になる。



この設計により、ヴァルゼリアの通常戦闘は、FFA風のテンポを保ちながら、現代的な周回・収集・やり込み要素を持つシステムになる。





この仕様で進めるなら、最初に作るべき最小単位は「探索度と連戦数の保存・表示・リセット」です。  

いきなり宝箱やダンジョン主まで作るより、まず探索画面に「探索度：120」「連戦数：12」が出るだけで、ゲーム体験がかなり変わります。
