## 1. 目的

本仕様書は、冒険都市ヴァルゼリアのフェーズ1実装として、ゲームの中核となる「迷宮探索」「戦闘」「経験値・GOLD獲得」「レベルアップ」「エリア解放」「戦闘ログ反映」を実装するための仕様を定義する。

現在、画面UIとして以下の基本構成は成立している。

* 左カラム：プレイヤーステータス
* 上部ナビ：宿場町、迷宮へ、闘技場、武具屋、転職所、ギルド、手紙箱、名簿
* 中央エリア：選択中タブの施設カード、探索エリアカード
* 下部ログ：システム、チャット、決闘、ギルド、ランキング、レアドロップ、お知らせ
* ヘッダー：冒険者数、本日の決闘数、現在の王者

フェーズ1では、UIの見た目をさらに作り込むことよりも、まず以下のゲームループを実際に動かすことを最優先とする。

```text
迷宮へ
↓
探索する
↓
敵と戦う
↓
勝敗が決まる
↓
経験値・GOLDを得る
↓
レベルアップする
↓
エリアボスを倒す
↓
次のエリアが解放される
↓
街ログに反映される
```

このループが通ることで、冒険都市ヴァルゼリアは「見た目だけの画面」から「遊べるFFA風RPG」に進む。

## 4.4 ステータスインフレ抑制方針

本作では、昔のFFAで起こりがちなステータスインフレを避ける。

フェーズ1では、レベルアップ時の能力上昇を小さく抑える。

基本方針は以下。

```text
レベルアップ時の能力上昇は、各能力ごとに+0〜+5程度に抑える
```

フェーズ1では、職業ごとの成長差は簡易実装でよい。
ただし、将来の転職システムに接続できるよう、成長処理は職業別に分岐できる設計にする。

## 5. エリア仕様

## 5.1 初期エリア

フェーズ1で実装するエリアは以下。

| エリアID | エリア名    | 推奨Lv | 初期状態              |
| ----: | ------- | ---- | ----------------- |
|     1 | はじまりの草原 | 1〜5  | 解放済み              |
|     2 | 小鬼の森    | 4〜10 | 解放済み、または草原ボス撃破で解放 |
|     3 | 古びた洞窟   | 10〜  | 小鬼の森ボス撃破で解放       |

初期テストをしやすくするため、はじまりの草原と小鬼の森は最初から解放済みでもよい。
正式仕様では「はじまりの草原のボス撃破で小鬼の森解放」とする。

迷宮へ画面のエリアカードには以下を表示する。

* エリア名
* 短い説明文
* 推奨レベル
* ボス状態
* 解放条件
* 探索ボタン
* ボス挑戦ボタン
* 未解放バッジ

## 5.3 エリアカード例

### 解放済みエリア

```text
はじまりの草原

```

### ボス未撃破エリア

```text

推奨Lv：4〜10

```

## 5.4 エリア解放条件

エリア解放は、前エリアのボス撃破によって行う。

```text
はじまりの草原ボス撃破
↓
小鬼の森 解放

小鬼の森ボス撃破
↓
古びた洞窟 解放
```

## 5.5 キャラクターごとのエリア進行管理

エリアの解放状態は、キャラクター単位で管理する。

キャラクターごとに以下を持つ。

| 項目               | 内容       |
| ---------------- | -------- |
| character_id     | 対象キャラクター |
| area_id          | 対象エリア    |
| is_unlocked      | 解放済みか    |
| boss_defeated    | ボス撃破済みか  |
| unlocked_at      | 解放日時     |
| boss_defeated_at | ボス撃破日時   |

## 7. 探索仕様

## 7.1 探索開始条件

探索ボタンを押したとき、以下をチェックする。

* ログイン済みである
* キャラクター作成済みである
* 対象エリアが解放済みである
* 現在HPが1以上である
* 戦闘クールタイム中ではない
* 同時戦闘処理中ではない

条件を満たす場合、探索処理を開始する。

## 7.2 探索処理フロー

```text
探索ボタン押下
↓
探索開始条件チェック
↓
対象エリアの通常敵から1体抽選
↓
自動戦闘処理
↓
勝敗決定
↓
報酬処理
↓
レベルアップ判定
↓
個人戦闘ログ保存
↓
公開ログ保存
↓
戦闘結果表示
```

## 7.3 敵抽選

通常探索では、対象エリア内の `is_boss = false` の敵から抽選する。

出現重みを使う。

例。

| 敵名     | 出現重み |
| ------ | ---: |
| スライム   |   60 |
| 迷いウサギ  |   30 |
| 草原コウモリ |   10 |

重みが高いほど出現しやすい。

## 7.4 探索結果表示

探索後は、メインエリアまたは専用結果画面に以下を表示する。

* 勝敗
* 戦闘ログ
* 獲得EXP
* 獲得GOLD
* レベルアップ結果
* エリア解放結果
* もう一度探索するボタン
* 迷宮へ戻るボタン

## 8. ボス戦仕様

## 8.1 ボス挑戦条件

ボスに挑戦するには以下を満たす必要がある。

* 対象エリアが解放済みである
* 対象エリアのボスが未撃破である
* 現在HPが1以上である
* 戦闘クールタイム中ではない

## 8.2 ボス戦フロー

```text
ボスに挑む
↓
ボス挑戦条件チェック
↓
ボス敵データ取得
↓
自動戦闘
↓
勝敗判定
↓
勝利なら boss_defeated = true
↓
次エリアを解放
↓
ログ保存
↓
結果表示
```

## 8.3 ボス撃破時の処理

ボス撃破時は以下を行う。

* 対象エリアの boss_defeated を true にする
* boss_defeated_at を保存
* 次エリアの is_unlocked を true にする
* unlocked_at を保存
* 公開ログに到達ログを流す
* 報酬EXP、GOLDを付与する

## 8.4 ボス撃破ログ例

```text
【到達】テストプレイさんが小鬼の森を突破し、古びた洞窟を解放しました！
```

または、

```text
【撃破】テストプレイさんが小鬼の森の親分を倒しました！
```

## 9. 戦闘仕様

## 9.1 戦闘方式

戦闘は自動ターン制とする。

プレイヤーが操作するのは「探索する」「ボスに挑む」まで。
戦闘中の手動コマンド選択はフェーズ1では実装しない。

## 9.2 戦闘フロー

```text
プレイヤーと敵のステータス取得
↓
先攻判定
↓
ターン開始
↓
攻撃処理
↓
ダメージ計算
↓
HP減少
↓
勝敗判定
↓
決着まで繰り返し
↓
戦闘ログ生成
```

## 9.3 先攻判定

AGIをもとに先攻を決定する。

```text
プレイヤー先攻値 = プレイヤーAGI + ランダム値 1〜10
敵先攻値 = 敵AGI + ランダム値 1〜10
```

数値が高い方が先攻。

同値の場合はプレイヤー先攻でよい。

## 9.4 ダメージ計算

物理ダメージは以下を基本とする。

```text
基本ダメージ = 攻撃側STR - 防御側DEF × 0.5
```

最低ダメージは1。

乱数補正として、最終ダメージに80％〜120％のブレを持たせる。

```text
最終ダメージ = 基本ダメージ × ランダム補正
```

小数点以下は四捨五入または切り捨てでよい。

## 9.5 魔力の扱い

フェーズ1ではMAGを通常戦闘に強く反映しなくてもよい。

ただし、僧侶や魔法使いの存在を考慮し、将来のスキル実装に備えてMAGは保持する。

フェーズ1では簡易仕様として、一定確率でMAG依存の追加ダメージを発生させてもよい。

例。

```text
魔法追加ダメージ = MAG × 0.5
```

ただし、実装負荷を下げる場合、フェーズ1では通常攻撃のみでもよい。

## 9.6 回避判定

AGI差によって低確率で回避を発生させる。

```text
回避率 = 5％ + (防御側AGI - 攻撃側AGI) × 0.5％
```

下限3％、上限20％。

## 9.7 会心判定

LUKによって会心率を決める。

```text
会心率 = 5％ + LUK × 0.2％
```

上限25％。

会心時はダメージ1.5倍。

## 9.8 最大ターン数

無限ループ防止のため、最大ターン数を設定する。

```text
最大ターン数：30ターン
```

30ターン経過しても決着がつかない場合は、残HP割合が高い方を勝者とする。

残HP割合も同じ場合はプレイヤー敗北とする。

## 10. HP仕様

## 10.1 戦闘後のHP

フェーズ1では、戦闘後のHPは減ったままにする。

理由は、宿屋の意味を持たせるため。

ただし、序盤離脱を防ぐため、敗北時でもHPは1残す。

## 10.2 勝利時

勝利時は、戦闘終了時の残HPを保存する。

## 10.3 敗北時

敗北時は、現在HPを1にする。

GOLDや装備は失わない。

## 10.4 宿屋回復

宿場町の宿屋でHPを全回復できる。

フェーズ1では料金無料とする。

表示文。

```text
HPを全回復しました。
```

公開ログには流さなくてよい。

## 10.5 HPが0または1のとき

現在HPが1以下でも探索は可能にしてよいが、実質すぐ負ける可能性が高い。

仕様としては、探索開始条件を「現在HPが1以上」とする。

## 11. 経験値・レベルアップ仕様

## 11.1 EXP獲得

戦闘勝利時のみEXPを獲得する。

敗北時はEXPを獲得しない。

## 11.2 必要経験値

フェーズ1の必要経験値は以下の式を基本とする。

```text
次レベル必要経験値 = 現在レベル × 現在レベル × 20 + 現在レベル × 30
```

例。

| 現在Lv | 次Lv必要EXP |
| ---: | -------: |
|    1 |       50 |
|    2 |      140 |
|    3 |      270 |
|    4 |      440 |
|    5 |      650 |

## 11.3 レベルアップ処理

EXPが必要値以上になった場合、レベルアップする。

複数レベル分のEXPを獲得した場合は、連続レベルアップを許可する。

処理フロー。

```text
EXP加算
↓
必要EXP以上か判定
↓
レベル +1
↓
必要EXP分を消費
↓
ステータス上昇
↓
HP全回復
↓
まだ必要EXP以上なら繰り返し
```

## 11.4 レベルアップ時の成長

フェーズ1では職業「僧侶」を基準に以下の範囲で成長させる。

| 能力   |   上昇量 |
| ---- | ----: |
| 最大HP | +2〜+4 |
| STR  | +0〜+2 |
| DEF  | +1〜+3 |
| AGI  | +1〜+2 |
| MAG  | +2〜+4 |
| LUK  | +1〜+2 |

今後、職業ごとに成長テーブルを分ける。

## 11.5 レベルアップログ

個人戦闘ログ内に表示する。

```text
レベルが2に上がった！
最大HPが3上がった！
MAGが4上がった！
```

公開ログには、節目レベルのみ流す。

例。

* Lv10到達
* Lv50到達
* Lv100到達

公開ログ例。

```text
【成長】テストプレイさんがLv10に到達しました！
```

## 12. GOLD仕様

## 12.1 GOLD獲得

戦闘勝利時のみGOLDを獲得する。

敗北時はGOLDを獲得しない。

## 12.2 GOLD喪失

フェーズ1では、敗北してもGOLDは減らない。

理由は、序盤ストレスを抑えるため。

## 12.3 将来拡張

将来的には以下を検討する。

* 銀行
* 敗北時の所持金減少
* 宿屋料金
* 武具屋購入
* 鍛冶屋強化費用

フェーズ1ではGOLDは「戦闘報酬として増える」だけでよい。

## 15. 戦闘クールタイム仕様

## 15.1 基本方針

戦闘後、短い待ち時間を発生させる。

目的は以下。

* ボタン連打防止
* サーバー負荷軽減
* 昔のFFAらしいテンポ再現
* 戦闘ログ確認の余白を作る

## 15.2 クールタイム秒数

フェーズ1では以下とする。

| 戦闘種別 | クールタイム |
| ---- | -----: |
| 通常探索 |     5秒 |
| ボス戦  |    15秒 |

## 15.3 クールタイム判定

キャラクターの `last_battle_at` を参照する。

現在時刻との差が必要秒数未満の場合、探索を拒否する。

表示例。

```text
少し息を整えています。あと3秒で探索できます。
```

## 15.4 フロント側制御

探索ボタン押下後は、ボタンを一時的にdisabledにする。

ただし、不正対策としてサーバー側判定を必須とする。

## 24. 重要な判断

## 24.1 まず迷宮探索を動かす

フェーズ1では、宿場町の細かい機能よりも「迷宮へ」を優先する。

理由は、ゲームループの中心だからである。

## 24.2 戦闘はシンプルでよい

フェーズ1では、職業スキルや魔法、装備効果を複雑にしない。

まずは以下だけで十分。

* 通常攻撃
* 回避
* 会心
* 勝敗
* 報酬

## 24.3 HPは減ったままにする

宿屋の意味を持たせるため、戦闘後HPは自動全回復しない。

ただし、敗北時はHP1で残す。

## 24.4 ログはゲーム体験の一部

戦闘結果やボス撃破は、単なる内部処理ではなく、街ログに反映する。

これにより、昔のFFAらしい「街が動いている感」が出る。

## 26. 結論

このフェーズ1では、冒険都市ヴァルゼリアを「見た目だけのFFA風画面」から「実際に遊べるFFA風RPG」へ進める。

最初に実装すべきは、迷宮探索と戦闘である。

```text
探索する
↓
敵と戦う
↓
勝つ
↓
EXPとGOLDを得る
↓
強くなる
↓
ボスを倒す
↓
次のエリアが解放される
```

この基本ループが成立すれば、次フェーズで武具屋、転職所、闘技場を追加しても、ゲーム全体が破綻しにくい。

フェーズ1の主目的は、ゲームの核を通すことである。
