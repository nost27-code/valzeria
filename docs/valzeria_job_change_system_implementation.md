# 冒険都市ヴァルゼリア 転職システム実装指示書

## 1. 目的

本実装では、FFA風ブラウザRPG「冒険都市ヴァルゼリア」において、職業を段階的にマスターし、複数職業の組み合わせによって中級職・上級職・伝説職へ転職できるシステムを実装する。

本システムの目的は、単に職業数を増やすことではない。

プレイヤーに対して、

* 次にどの職業を育てるか考える楽しさ
* 職業をマスターしていく積み上げ感
* 転職条件を満たしたときの達成感
* 職業ごとの役割差によるビルド選択
* 長期的な育成目標

を提供することを目的とする。

本ゲームは昔ながらのFFA風CGIゲームをベースとしているため、操作はシンプルでよい。ただし、育成の奥行きはしっかり持たせる。

## 2. 実装方針

職業は以下の4階層で管理する。

| 階層  | 区分       | 説明                          |
| --- | -------- | --------------------------- |
| 一般職 | normal   | 最初から転職可能な基本職                |
| 中級職 | middle   | 一般職を複数マスターすると解放             |
| 上級職 | advanced | 中級職を複数マスターすると解放             |
| 伝説職 | legend   | 上級職マスター、称号、アイテム、イベント達成などで解放 |

基本ルールは以下とする。

* 職業には職業レベルを持たせる
* 職業レベルが最大に到達すると、その職業をマスターした扱いにする
* マスター済み職業はプレイヤーごとに永続保存する
* 中級職・上級職・伝説職は、必要職業のマスター状況を元に解放判定する
* 転職後、キャラクターレベルはリセットしない
* 転職後、現在職業のみ変更する
* 職業ごとの補正値は、現在職業に応じて戦闘計算時に反映する
* マスター報酬は永続効果として一部反映する
* スキル継承は今後の拡張を見越してDB設計だけ先に考慮する

## 3. 用語定義

| 用語       | 意味                                      |
| -------- | --------------------------------------- |
| 職業       | プレイヤーが就くジョブ                             |
| 現在職業     | 現在プレイヤーに設定されている職業                       |
| 職業レベル    | 職業ごとの熟練度レベル                             |
| 職業経験値    | 職業レベルを上げるための経験値                         |
| マスター     | 職業レベルが最大に到達した状態                         |
| 転職条件     | その職業へ転職するために必要な条件                       |
| マスターボーナス | 職業マスター時に得られる永続補正                        |
| 職業ランク    | normal / middle / advanced / legend の分類 |
| 隠し職業     | 条件を満たすまで一覧に表示しない職業                      |

## 4. 職業階層の基本仕様

### 4.1 一般職

一般職は、基本的に初期状態から転職可能とする。

ただし、ゲーム開始直後の選択肢が多すぎると初心者が迷うため、初期選択可能職と、冒険者ギルド到達後に転職可能になる一般職を分けてもよい。

初期実装では、全一般職を最初から転職可能としてよい。

### 4.2 中級職

中級職は、指定された一般職を複数マスターすると解放される。

例：

* 魔法剣士：剣士マスター ＋ 魔法使いマスター
* 聖騎士：剣士マスター ＋ 僧侶マスター
* 忍者：盗賊マスター ＋ 格闘家マスター

### 4.3 上級職

上級職は、指定された中級職を複数マスターすると解放される。

例：

* 勇者：魔法剣士マスター ＋ 聖騎士マスター
* 剣聖：魔法剣士マスター ＋ 狂戦士マスター
* 大賢者：賢者見習いマスター ＋ 司祭マスター

### 4.4 伝説職

伝説職は、通常の職業マスター条件に加えて、称号、アイテム、イベントクリアなどの特殊条件を必要とする。

例：

* ヴァルゼリアの英雄：勇者マスター ＋ 都市防衛戦クリア ＋ 英雄の証を所持
* 竜騎士王：竜騎士マスター ＋ 古竜討伐称号
* 深淵歩き：暗黒騎士マスター ＋ 影武者マスター ＋ 深淵の紋章を所持

初期実装では、伝説職はデータ定義のみ行い、実際の解放は後続フェーズでもよい。

## 6. 職業レベル仕様

### 6.1 職業レベル

初期実装では、職業レベルは 1 から 10 までとする。

| 項目        | 値               |
| --------- | --------------- |
| 初期職業レベル   | 1               |
| 最大職業レベル   | 10              |
| マスター条件    | 職業レベル10到達       |
| 転職時の職業レベル | その職業の保存済みレベルを使用 |
| 未経験職への転職  | 職業レベル1から開始      |

職業レベルはキャラクターレベルとは別に管理する。

### 6.2 職業経験値

戦闘勝利時に、通常経験値とは別に職業経験値を付与する。

職業経験値は、現在職業にのみ加算する。

敗北時は職業経験値を付与しない。
ただし、ゲームテンポを優先する場合は、敗北時にも少量付与する仕様にしてもよい。

初期実装では、勝利時のみ付与でよい。

### 6.3 職業レベルアップ必要経験値

初期値は以下とする。

| 職業レベル  | 次レベルまでの必要職業経験値 |
| ------ | -------------: |
| 1 → 2  |            100 |
| 2 → 3  |            150 |
| 3 → 4  |            220 |
| 4 → 5  |            320 |
| 5 → 6  |            450 |
| 6 → 7  |            620 |
| 7 → 8  |            850 |
| 8 → 9  |           1150 |
| 9 → 10 |           1500 |

合計職業経験値は5360。

職業経験値の獲得量は、通常戦闘1回あたり20から50程度を想定する。
この場合、1職マスターまでおよそ120から250戦程度となる。

ゲームテンポを見て調整すること。

## 7. 職業マスター仕様

### 7.1 マスター判定

職業レベルが最大値に到達したタイミングで、その職業をマスター済みにする。

マスター時には以下を行う。

* player_jobs.is_mastered を true に更新
* player_jobs.mastered_at に現在日時を保存
* マスターボーナスを有効化
* マスター称号がある場合は付与
* 転職可能職業の再判定
* マスター通知を表示

### 7.2 マスター通知

マスター時には、以下のようなメッセージを表示する。

例：

```text
剣士をマスターしました！
攻撃力が永続的に少し上昇しました。
新たな職業に転職できる可能性があります。
```

中級職や上級職が解放された場合は、追加で表示する。

```text
新しい職業「魔法剣士」が解放されました！
```

## 8. マスターボーナス仕様

マスターボーナスは永続効果とする。

ただし、すべてのマスターボーナスを強くしすぎると、長期プレイヤーが強くなりすぎるため、初期実装では控えめにする。

推奨値は、1職マスターにつき特定能力＋1％から＋2％程度。

## 9. 職業ステータス補正

### 9.1 基本方針

現在職業によって、戦闘時のステータスに倍率補正をかける。

職業補正は、現在就いている職業のみ反映する。

マスターボーナスは現在職業に関係なく反映する。

つまり、最終ステータス計算は以下のイメージとする。

```text
最終ステータス = 素ステータス × 現在職業補正 × マスターボーナス補正 × 装備補正
```

## 10. DB設計

### 10.1 jobs テーブル

職業マスターデータを管理する。

```php
Schema::create('jobs', function (Blueprint $table) {
    $table->id();
    $table->string('key')->unique();
    $table->string('name');
    $table->enum('rank', ['normal', 'middle', 'advanced', 'legend']);
    $table->string('category')->nullable();
    $table->text('description')->nullable();

    $table->unsignedTinyInteger('max_job_level')->default(10);

    $table->unsignedSmallInteger('hp_rate')->default(100);
    $table->unsignedSmallInteger('mp_rate')->default(100);
    $table->unsignedSmallInteger('atk_rate')->default(100);
    $table->unsignedSmallInteger('def_rate')->default(100);
    $table->unsignedSmallInteger('mag_rate')->default(100);
    $table->unsignedSmallInteger('spr_rate')->default(100);
    $table->unsignedSmallInteger('spd_rate')->default(100);
    $table->unsignedSmallInteger('luck_rate')->default(100);

    $table->boolean('is_hidden')->default(false);
    $table->boolean('is_active')->default(true);
    $table->unsignedInteger('sort_order')->default(0);

    $table->timestamps();
});
```

### 10.2 job_requirements テーブル

転職条件を管理する。

1つの職業に対して、複数の条件を登録できる。

```php
Schema::create('job_requirements', function (Blueprint $table) {
    $table->id();

    $table->foreignId('job_id')->constrained('jobs')->cascadeOnDelete();

    $table->enum('requirement_type', [
        'master_job',
        'character_level',
        'title',
        'item',
        'quest',
        'event_flag'
    ]);

    $table->foreignId('required_job_id')->nullable()->constrained('jobs')->nullOnDelete();

    $table->unsignedInteger('required_value')->nullable();
    $table->string('required_key')->nullable();

    $table->timestamps();
});
```

使用例：

| job_id | requirement_type | required_job_id | required_value | required_key |
| ------ | ---------------- | --------------- | -------------- | ------------ |
| 魔法剣士   | master_job       | 剣士              | null           | null         |
| 魔法剣士   | master_job       | 魔法使い            | null           | null         |
| 勇者     | master_job       | 魔法剣士            | null           | null         |
| 勇者     | master_job       | 聖騎士             | null           | null         |
| 勇者     | character_level  | null            | 50             | null         |

### 10.3 player_jobs テーブル

プレイヤーごとの職業育成状況を管理する。

```php
Schema::create('player_jobs', function (Blueprint $table) {
    $table->id();

    $table->foreignId('player_id')->constrained('players')->cascadeOnDelete();
    $table->foreignId('job_id')->constrained('jobs')->cascadeOnDelete();

    $table->unsignedTinyInteger('job_level')->default(1);
    $table->unsignedInteger('job_exp')->default(0);
    $table->boolean('is_mastered')->default(false);
    $table->timestamp('mastered_at')->nullable();

    $table->timestamps();

    $table->unique(['player_id', 'job_id']);
});
```

### 10.4 player_current_job

players テーブルに現在職業IDを追加する。

```php
Schema::table('players', function (Blueprint $table) {
    $table->foreignId('current_job_id')->nullable()->after('id')->constrained('jobs')->nullOnDelete();
});
```

既存のplayersテーブル構成によって after の位置は調整すること。

### 10.5 job_exp_tables テーブル

職業レベルごとの必要経験値を管理する。

```php
Schema::create('job_exp_tables', function (Blueprint $table) {
    $table->id();
    $table->unsignedTinyInteger('job_level');
    $table->unsignedInteger('required_exp');
    $table->timestamps();

    $table->unique('job_level');
});
```

例：

| job_level | required_exp |
| --------: | -----------: |
|         1 |          100 |
|         2 |          150 |
|         3 |          220 |

job_level は「このレベルから次レベルへ上がるために必要な経験値」とする。

### 10.6 job_master_bonuses テーブル

マスターボーナスを管理する。

```php
Schema::create('job_master_bonuses', function (Blueprint $table) {
    $table->id();

    $table->foreignId('job_id')->constrained('jobs')->cascadeOnDelete();

    $table->enum('bonus_type', [
        'hp_rate',
        'mp_rate',
        'atk_rate',
        'def_rate',
        'mag_rate',
        'spr_rate',
        'spd_rate',
        'luck_rate',
        'gold_rate',
        'drop_rate',
        'critical_rate',
        'evasion_rate',
        'heal_rate',
        'item_effect_rate'
    ]);

    $table->integer('bonus_value')->default(0);

    $table->timestamps();
});
```

bonus_value はパーセント値で管理する。

例：

* 攻撃力＋2％ → bonus_type = atk_rate, bonus_value = 2
* 獲得ゴールド＋5％ → bonus_type = gold_rate, bonus_value = 5
* 防御力－1％ → bonus_type = def_rate, bonus_value = -1

## 11. モデル設計

### 11.1 Job モデル

関係性：

```php
class Job extends Model

    {

        return $this->hasMany(JobMasterBonus::class);
    }

    public function playerJobs()
```

### 11.2 PlayerJob モデル

```php
class PlayerJob extends Model
{
    protected $fillable = [
        'player_id',
```

## 13. 転職可能判定

### 13.1 判定方針

転職可能かどうかは job_requirements を元に判定する。

条件が1件もない職業は、基本的に転職可能とする。
ただし、rank が normal の職業のみこの扱いにする。

middle / advanced / legend で条件が未設定の場合は、データ不備として転職不可にする。

### 13.2 擬似コード

```php
public function canChangeJob(Player $player, Job $job): bool
{
    if (!$job->is_active) {
        return false;
    }

    $requirements = $job->requirements;

    if ($requirements->isEmpty()) {
        return $job->rank === 'normal';
    }

    foreach ($requirements as $requirement) {
        if (!$this->isRequirementMet($player, $requirement)) {
            return false;
        }
    }

    return true;
}
```

### 13.3 条件別判定

```php
private function isRequirementMet(Player $player, JobRequirement $requirement): bool
{
    return match ($requirement->requirement_type) {
        'master_job' => $this->hasMasteredJob($player, $requirement->required_job_id),
        'character_level' => $player->level >= $requirement->required_value,
        'title' => $this->hasTitle($player, $requirement->required_key),
        'item' => $this->hasItem($player, $requirement->required_key),
        'quest' => $this->hasCompletedQuest($player, $requirement->required_key),
        'event_flag' => $this->hasEventFlag($player, $requirement->required_key),
        default => false,
    };
}
```

### 13.4 マスター済み判定

```php
private function hasMasteredJob(Player $player, int $jobId): bool
{
    return PlayerJob::where('player_id', $player->id)
        ->where('job_id', $jobId)
        ->where('is_mastered', true)
        ->exists();
}
```

## 14. 転職処理

### 14.1 転職ルール

転職時は以下を行う。

* 指定職業が存在するか確認
* 転職可能か確認
* 現在職業と同じ場合はエラー
* player_jobs に該当職業のレコードがなければ作成
* players.current_job_id を更新
* 転職完了メッセージを返す

### 14.2 転職時にリセットしないもの

以下はリセットしない。

* キャラクターレベル
* 通常経験値
* 所持金
* 装備
* マスター済み職業
* 習得済みスキル
* 称号
* クエスト進行度

### 14.3 転職時に注意するもの

現在職業のレベル・経験値は player_jobs に保存済みなので、他職へ転職しても失われない。

一度育てた職業へ戻った場合は、以前の職業レベル・職業経験値から再開する。

### 14.4 擬似コード

```php
public function changeJob(Player $player, Job $job): void
{
    if (!$this->canChangeJob($player, $job)) {
        throw new DomainException('この職業にはまだ転職できません。');
    }

    if ((int) $player->current_job_id === (int) $job->id) {
        throw new DomainException('すでにこの職業に就いています。');
    }

    PlayerJob::firstOrCreate(
        [
            'player_id' => $player->id,
            'job_id' => $job->id,
        ],
        [
            'job_level' => 1,
            'job_exp' => 0,
            'is_mastered' => false,
        ]
    );

    $player->current_job_id = $job->id;
    $player->save();
}
```

## 15. 職業経験値付与処理

### 15.1 付与タイミング

職業経験値は戦闘勝利時に付与する。

BattleService または戦闘終了処理から JobService::addJobExp を呼び出す。

```php
$jobResult = $jobService->addJobExp($player, $jobExp);
```

### 15.2 職業経験値量

初期実装では、敵の強さに応じて職業経験値を設定する。

例：

| 敵ランク | 職業経験値 |
| ---- | ----: |
| 弱い敵  |    20 |
| 通常敵  |    30 |
| 強敵   |    50 |
| ボス   |   100 |

通常経験値とは別管理にする。

### 15.3 レベルアップ処理

職業経験値が必要値を超えた場合、職業レベルを上げる。

複数レベルアップにも対応する。

最大レベルに到達した場合は、is_mastered を true にする。

### 15.4 擬似コード

```php
public function addJobExp(Player $player, int $exp): array
{
    $currentJob = $player->currentJob;

    if (!$currentJob) {
        return [
            'gained_exp' => 0,
            'level_ups' => [],
            'mastered' => false,
            'unlocked_jobs' => [],
        ];
    }

    $playerJob = PlayerJob::firstOrCreate(
        [
            'player_id' => $player->id,
            'job_id' => $currentJob->id,
        ],
        [
            'job_level' => 1,
            'job_exp' => 0,
            'is_mastered' => false,
        ]
    );

    if ($playerJob->is_mastered) {
        return [
            'gained_exp' => 0,
            'level_ups' => [],
            'mastered' => false,
            'unlocked_jobs' => [],
        ];
    }

    $beforeAvailableJobIds = $this->getAvailableJobs($player)->pluck('id')->all();

    $playerJob->job_exp += $exp;

    $levelUps = [];

    while ($playerJob->job_level < $currentJob->max_job_level) {
        $requiredExp = $this->getRequiredJobExp($playerJob->job_level);

        if ($playerJob->job_exp < $requiredExp) {
            break;
        }

        $playerJob->job_exp -= $requiredExp;
        $playerJob->job_level++;

        $levelUps[] = $playerJob->job_level;
    }

    $mastered = false;

    if ($playerJob->job_level >= $currentJob->max_job_level) {
        $playerJob->job_level = $currentJob->max_job_level;
        $playerJob->job_exp = 0;
        $playerJob->is_mastered = true;
        $playerJob->mastered_at = now();
        $mastered = true;
    }

    $playerJob->save();

    $afterAvailableJobs = $this->getAvailableJobs($player);
    $unlockedJobs = $afterAvailableJobs
        ->whereNotIn('id', $beforeAvailableJobIds)
        ->values();

    return [
        'gained_exp' => $exp,
        'level_ups' => $levelUps,
        'mastered' => $mastered,
        'unlocked_jobs' => $unlockedJobs,
    ];
}
```

## 17. ステータス計算

### 17.1 基本方針

ステータス計算は専用サービスに寄せる。

既存の Player モデル内で直接複雑な計算をしない。

推奨構成：

* PlayerStatusService
* JobService
* EquipmentService

### 17.2 計算順

推奨する計算順は以下。

1. プレイヤーの素ステータスを取得
2. 現在職業補正を適用
3. マスターボーナスを適用
4. 装備補正を適用
5. バフ・デバフを適用
6. 最終値を整数に丸める

### 17.3 職業補正の計算例

```php
$stats['atk'] = floor($baseStats['atk'] * ($job->atk_rate / 100));
```

### 17.4 マスターボーナスの計算例

攻撃力＋2％、攻撃力＋1％、攻撃力＋4％を持つ場合、合計＋7％として扱う。

```php
$atkBonusRate = 1 + ($masterBonuses['atk_rate'] / 100);
$stats['atk'] = floor($stats['atk'] * $atkBonusRate);
```

## 23. バランス調整方針

### 23.1 上級職を万能にしすぎない

上級職は強くしてよいが、すべての上級職が同じ方向で強くなると意味がない。

役割を分けること。

例：

| 職業   | 役割       |
| ---- | -------- |
| 勇者   | 安定万能     |
| 剣聖   | 物理火力     |
| 大賢者  | 魔法・回復    |
| 暗黒騎士 | リスクあり高火力 |
| 黄金商人 | 金策・素材    |

### 23.2 目的別に最適職を変える

このシステムでは、「最強職を1つ作る」のではなく、「目的別に強い職業を作る」こと。

目的例：

* 通常周回
* ボス攻略
* 金策
* 素材集め
* レアドロップ狙い
* 高難度ダンジョン
* イベント攻略

### 23.3 永続ボーナスは控えめにする

職業マスターの積み上げは楽しいが、永続ボーナスが強すぎるとバランスが壊れる。

初期値は控えめにする。

一般職：＋1％から＋2％
中級職：＋2％から＋3％
上級職：＋3％から＋5％

## 28. 注意すべき設計ミス

### 28.1 最強職を1つ作らない

勇者を明確な最強職にすると、最終的に全員が勇者になる。

勇者は万能型にするが、火力では剣聖、魔法では大賢者、金策では黄金商人に負けるようにする。

### 28.2 永続ボーナスを強くしすぎない

マスターの積み上げは魅力だが、永続補正が強すぎると古参と新規の差が広がりすぎる。

最初は控えめにして、必要なら後から強化する。

### 28.3 条件を隠しすぎない

隠し職業以外は、条件未達成でも表示する。

プレイヤーに「次に何をすればよいか」を見せることが重要。

### 28.4 職業経験値を重くしすぎない

1職マスターが重すぎると、転職システム全体が遊ばれない。

最初はやや軽めにして、上級職・伝説職で長期目標を作る方がよい。

### 28.5 Controller にロジックを書かない

転職条件、マスター判定、経験値計算を Controller に書くと、後で破綻する。

必ず JobService に集約する。

## 29. 最終的な完成イメージ

プレイヤーは最初に一般職を選び、戦闘を重ねて職業レベルを上げる。

剣士をマスターし、魔法使いもマスターすると、魔法剣士が解放される。

さらに僧侶をマスターして聖騎士になり、魔法剣士と聖騎士をマスターすると勇者が解放される。

一方で、盗賊や商人を育てたプレイヤーは旅商人や黄金商人を目指せる。

戦闘特化だけでなく、金策、素材集め、レアドロップ、探索効率など、目的ごとに違う職業を選べる状態を目指す。

この転職システムによって、ヴァルゼリアの育成は以下の形になる。

```text
一般職を育てる
↓
職業をマスターする
↓
新しい職業が解放される
↓
目的に合わせて転職する
↓
さらに上位職を目指す
↓
伝説職や特殊職を目指す
```

このループが、FFA風ブラウザRPGとしての長期的な遊びの核になる。

# 転職システム仕様修正メモ

## 1. 修正方針

職業マスターまでの到達が早すぎると、転職システムの重みが薄くなる。

本ゲームでは、職業マスターを長期的な育成目標として扱いたいため、職業経験値の獲得量を下げ、職業マスターまでの必要戦闘回数を増やす。

一方で、職業ごとの個性は明確に出したいため、職業ステータス補正は従来案より大きくする。

ただし、すべての能力を高くするのではなく、得意能力と苦手能力をはっきり分ける。

## 2. 職業経験値の修正

### 2.1 基本方針

職業経験値の獲得量は、通常経験値よりかなり低めに設定する。

職業マスターは簡単に到達できるものではなく、同じ職業をしっかり使い込んだ結果として達成されるものにする。

初期案では1職マスターまで約120から250戦程度を想定していたが、これは軽すぎる。

修正後は、一般職でも1職マスターまで約400から700戦程度を目安にする。

中級職・上級職はさらに重くする。

## 3. 職業ランク別のマスター難易度

職業ランクごとに最大職業レベルと必要経験値倍率を分ける。

| 職業ランク | 最大職業レベル | 必要経験値倍率 | 想定            |
| ----- | ------: | ------: | ------------- |
| 一般職   |      10 |    1.0倍 | 基本職。やや重いが到達可能 |
| 中級職   |      12 |    1.5倍 | 本格育成枠         |
| 上級職   |      15 |    2.2倍 | 長期育成枠         |
| 伝説職   |      20 |    3.5倍 | やり込み枠         |

この仕様により、上位職ほどマスターの重みが増す。

## 9. job_exp_tables の修正

職業ランクごとに必要経験値を変えるため、job_exp_tables テーブルには rank を追加する。

```php
Schema::create('job_exp_tables', function (Blueprint $table) {
    $table->id();

    $table->enum('job_rank', ['normal', 'middle', 'advanced', 'legend']);
    $table->unsignedTinyInteger('job_level');
    $table->unsignedInteger('required_exp');

    $table->timestamps();

    $table->unique(['job_rank', 'job_level']);
});
```

取得時は、現在職業の rank と現在職業レベルを元に必要経験値を取得する。

```php
$requiredExp = JobExpTable::where('job_rank', $job->rank)
    ->where('job_level', $playerJob->job_level)
    ->value('required_exp');
```

## 11. 職業補正の設計ルール

### 11.1 一般職

一般職は、強い個性を持たせつつも、補正上限は基本1.50までとする。

ただし、明確な専門職であれば1.75まで許可する。

例：

* 魔法使いの魔力 1.75
* 戦士の攻撃 1.50
* 盗賊の素早さ 1.75
* 商人の運 1.50

### 11.2 中級職

中級職は、得意能力を1.75から2.00まで許可する。

ただし、苦手能力に0.75または0.50を設定する。

例：

* 狂戦士は攻撃2.00、防御0.75、精神0.50
* 忍者は素早さ2.00、防御0.75、MP0.75
* 司祭は精神2.00、攻撃0.50

### 11.3 上級職

上級職は、得意能力を2.00から2.25まで許可する。

ただし、万能型以外は明確な弱点を持たせる。

勇者のような万能型は、全体的に1.25前後でまとめ、2.00以上の突出補正は持たせない。

### 11.4 伝説職

伝説職は2.25補正を持ってよい。

ただし、転職条件・マスター難易度・運用リスクのいずれかでバランスを取る。

例：

* 終焉の魔王：魔力2.25、攻撃2.00、防御0.50、精神0.50
* 神域の守護者：HP2.25、防御2.25、素早さ0.50
* 時渡りの剣士：素早さ2.25、攻撃1.75、防御0.75

## 13. 職業補正の注意点

職業補正を強くした場合、戦闘バランスは大きく変わる。

特に以下の点に注意する。

### 13.1 攻撃2.25職の扱い

攻撃2.25の職業は、通常攻撃ダメージが大きく伸びる。

そのため、剣聖や暗黒騎士には以下のような弱点を設定する。

* MPが低い
* 防御が低い
* 精神が低い
* 回復に弱い
* 状態異常に弱い
* 長期戦に弱い

### 13.2 魔力2.25職の扱い

大賢者のような魔力2.25職は、魔法ダメージが大きく伸びる。

その代わり、以下の弱点を設定する。

* HPが低い
* 防御が低い
* 通常攻撃が弱い
* MP切れに弱い
* 物理敵に弱い

### 13.3 運2.25職の扱い

黄金商人のような運2.25職は、ドロップ率やゴールド補正に影響する場合、経済バランスに直結する。

そのため、運をそのまま獲得ゴールド倍率に直結させないこと。

運は以下のように間接的に使う。

* クリティカル補正
* レアドロップ判定
* 逃走成功率
* 一部イベント判定
* 取引イベント発生率

獲得ゴールドは、別途 master_bonus や職業固有効果で制御する。

## 15. 職業経験値と通常経験値を分ける理由

職業経験値は、キャラクターの成長速度とは別で管理する。

通常レベルは上がっているが、職業マスターはまだ遠い、という状態を作ることで、長期的な育成導線が生まれる。

また、転職条件を重くすることで、上級職に到達したときの価値が上がる。

## 16. 職業マスターまでの想定プレイ感

### 16.1 一般職

一般職は、しばらく使い込むとマスターできる。

ただし、数時間で複数職をマスターできるような軽さにはしない。

目安：

* ライトプレイヤー：数日から1週間で1職マスター
* ヘビープレイヤー：1日で1職マスターも可能
* 放置・周回勢：効率狩りで短縮可能

### 16.2 中級職

中級職は、明確に育成対象として選んで使い込む必要がある。

目安：

* ライトプレイヤー：1から2週間で1職マスター
* ヘビープレイヤー：数日で1職マスター
* イベント活用で短縮可能

### 16.3 上級職

上級職は長期目標。

マスター済み上級職を複数持っているプレイヤーは、かなりやり込んでいる状態とする。

目安：

* ライトプレイヤー：数週間から1か月以上
* ヘビープレイヤー：1から2週間
* イベントや高難度周回で短縮可能

## 17. 今回の修正で重要な結論

今回の修正により、転職システムは以下の方向になる。

```text
職業マスターは重い
転職したときの職業個性は強い
上級職は簡単には完成しない
ただし、転職直後から職業ごとの違いは体感できる
```

この方がFFA風ゲームとして長持ちする。

職業マスターが軽すぎると、プレイヤーは短期間でツリーを消化してしまう。

逆に、職業補正が弱すぎると、転職しても違いを感じにくい。

そのため、

* 職業経験値は絞る
* 必要経験値は重くする
* 職業補正は強くする
* マスターボーナスは控えめにする

この4点を基本方針とする。
