# 素材システム（ドロップ・インベントリ）実装計画

## 目的
作成された `docs/sozai.md` を元に、敵からの「素材」および「印」のドロップ機能を実装し、キャラクターがそれらを保持できるようにする。

## 提案される変更 (Proposed Changes)

### 1. データベース (Migrations & Models)
#### [NEW] `create_materials_table` & `Material` Model
*   `sozai.md` のカラム定義に従うマスターデータテーブル。
*   `id` (BigIncrements), `material_code` (MAT0001など), `name`, `category`, `rarity`, `element`, `main_use`, `npc_sale_price`, `is_tradable`, `source_enemy_id` を定義。

#### [NEW] `create_character_materials_table` & `CharacterMaterial` Model
*   キャラクターが保持する素材の中間テーブル。
*   `character_id`, `material_id`, `quantity` を定義。
*   数量が0になったレコードを削除する方針か、あるいは0で残すかを決めるが、基本は0になれば削除。

### 2. データ投入 (Seeder)
#### [NEW] `MaterialSeeder`
*   `docs/sozai.md` をTSV形式で解析し、`materials` テーブルに登録する。
*   ※ `update_enemies_data.php` などと同様に、CSV/TSVをパースするPHP処理を用意し、デプロイコマンドなどで一括更新できるようにする。

### 3. ドロップロジック (Backend)
#### [MODIFY] ドロップ判定処理の実装
*   `app/Http/Controllers/BattleController.php` (または専用のService) において、敵撃破時に素材ドロップ判定を追加。
*   **ドロップ確率（基本方針）**:
    *   雑魚の通常素材：40％
    *   雑魚の希少素材：8％
    *   やや強い敵の通常素材：45％
    *   やや強い敵の希少素材：12％
    *   レア敵素材：20％
    *   最深部候補素材：15％
    *   通常印：5％
    *   希少印：2％
    *   上位印：1％
    *   ボス鍵素材・ボス印：100％（ただし初回のみ。所持数が1以上の場合はドロップ判定しない）
*   素材や印がドロップした際、メッセージとして「スライムの粘液を落とした！」などをバトルリザルト表示に渡す。

### 4. UI追加
#### [MODIFY] ステータス・アイテム画面
*   プレイヤーが所持している素材を確認するための簡単な一覧（「素材」タブなど）を実装する。

## 懸念事項・注意点
*   ボス素材は「1回のみ」とするため、`character_materials` の該当素材の所持数をチェックし、既に持っていれば（あるいは過去に取得してフラグが立っていれば）ドロップさせないように制御する。
    ※ 武器合成などで消費して一時的に「0」になることも考慮すると、「これまでに入手したか」というフラグのほうが安全な可能性もあるが、まずは「現在持っているか、またはボスの討伐フラグが立っているか」で判定する。
*   既存の武具ドロップと併用し、独立して判定されるようにする。
