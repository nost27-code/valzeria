> **【失効ノート / 2026-07-02】このレポートの「対応案」を実行しないでください。**
> 本レポートは2026-06時点の「Gold廃止検討期」に作成されたものです。その後**Goldは再実装され、現在の正仕様**になりました（宿代、鍛冶費用、素材交換費用、ショップのGold購入、倉庫Gold拡張、市場取引などが稼働中。参照: docs/DOMAIN_RULES.md、docs/ai_development_rules.md「Gold経済のルール」）。

## 今後テスト化すべき項目

- 項目:
  - ゴールドが増えないこと
- 推奨するテスト内容:
  - 通常探索勝利、ボス勝利、チャンプ戦勝利後に `characters.money` が増えないこと。
  - `battle_logs.gold_gained` が0で保存されること。

- 項目:
  - ゴールド系UIが出ないこと
- 推奨するテスト内容:
  - 案内所、戦闘結果、ショップ、装備、管理ログに「ゴールド」「所持金」「Gold +」が表示されないことを画面テストまたは文字列テストで確認する。

- 項目:
  - 無料支給ショップの挙動
- 推奨するテスト内容:
  - Gランク武器/防具は受け取れる。
  - 消耗品は購入できず補給所案内になる。
  - 所持金不足による購入失敗分岐が存在しないこと。

- 項目:
  - 装備/素材売却停止
- 推奨するテスト内容:
  - 装備売却ルート、素材売却ルートを叩いても通貨が増えない。
  - 不要装備は分解へ誘導される。

- 項目:
  - 輝石残高整合性
- 推奨するテスト内容:
  - 有償輝石購入で `paid_kiseki` と `kiseki` が増える。
  - 無償輝石ドロップで `free_kiseki` と `kiseki` が増える。
  - 支援アイテム購入で有償/無償消費内訳と `kiseki_transactions` が一致する。

  - 輝石二重付与防止

- 項目:
  - 通常探索にボスが出ないこと
- 推奨するテスト内容:
  - 通常探索の敵抽選が `is_boss=false` のみを対象にすること。
  - ボス挑戦が `is_boss=true` のみを対象にすること。

- 項目:
  - ダンジョン/都市解放
- 推奨するテスト内容:
  - ダンジョンボス撃破で次ダンジョンが解放される。
  - 都市内通常最終ダンジョン撃破で次都市が解放される。
  - 裏ダンジョン撃破で都市解放判定が誤作動しない。

- 項目:
  - 転職条件
- 推奨するテスト内容:
  - Lv99、Lv100、現職マスター済み/未マスターの組み合わせで転職可否を確認する。
  - 正仕様が決まり次第、その条件に固定したテストへ更新する。

- 項目:
  - 転職時ステータス継承
- 推奨するテスト内容:
  - 転職後Lv1、EXP0、基礎ステータス50%継承、最低保証適用、HP/MP回復、全fractionリセットを確認する。

- 項目:
  - BP付与と割り振り
- 推奨するテスト内容:
  - レベルアップごとにBPが1増える。
  - HP/MP/ATK/DEF/MAG/SPR/SPD/LUKへの割り振りが基礎ステータスと現在HP/MPへ正しく反映される。

- 項目:
  - 職業ランクとマスター
- 推奨するテスト内容:
  - 職業経験値獲得で★1〜★10まで上がる。
  - ★10で `is_mastered` がtrueになる。
  - ランクアップ表示が「職業名のランクがNに上がった」になっている。

- 項目:
  - ドロップ整合性
- 推奨するテスト内容:
  - 敵ドロップが存在する素材/アイテムIDだけを参照している。
  - 同名敵がいる場合でも `area_id` などで正しい敵に紐付く。
  - 装備の欠片3段階が進行度に応じて落ちる。

- 項目:
  - ヴァルモン
- 推奨するテスト内容:
  - スターター選択、相棒設定、卵発見、卵孵化、敗北時卵ロスト、救助保護、餌によるLv上昇、素材発見を確認する。

- 項目:
  - チャンプ戦
- 推奨するテスト内容:
  - チャンプ本人は挑戦できない。
  - クールダウン中は再挑戦できない。
  - チャンプ撃破でチャンプ交代、履歴、報酬、ログが記録される。

- 項目:
  - 管理者機能
- 推奨するテスト内容:
  - 管理者以外が管理画面へ入れない。
  - アイテム/職業/報酬設定/トップ更新の保存が意図通り。
  - 行動ログで戦闘、輝石、課金支援、装備進化、分解、ヴァルモンが確認できる。
