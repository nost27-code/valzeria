## 1. 目的

本仕様書は、冒険都市ヴァルゼリアにおける「転職システム」と「能力引き継ぎ仕様」を定義するものである。

本作では、レベル100に到達したキャラクターのみ転職できる。

転職は、単なる職業変更ではなく、長期育成の中核となる仕組みである。

ただし、昔のFFA系ゲームで起こりがちだったステータスインフレを避けるため、能力の引き継ぎ量は慎重に調整する。

目標は以下。

```text
1周目 Lv100
↓
転職して Lv1 に戻る
↓
能力は一部だけ引き継ぐ
↓
2周目 Lv100
↓
1周目 Lv100 より 2〜3割程度強い
```

つまり、転職によって確実に強くはなるが、能力が何倍にも膨れ上がる設計にはしない。

---

## 2. 基本方針

## 2.1 転職可能条件

転職は、以下の条件をすべて満たした場合のみ可能とする。

| 条件       | 内容                |
| -------- | ----------------- |
| レベル      | Lv100以上           |
| キャラクター状態 | ログイン済み、キャラクター作成済み |
| 戦闘中でない   | 戦闘処理中ではない         |
| 転職先      | 有効な職業である          |
| 必要条件     | 転職先職業の解放条件を満たしている |

フェーズ1では、最低条件として「Lv100以上」を必須とする。

## 2.2 転職時の基本処理

転職時は以下を行う。

| 項目      | 処理                |
| ------- | ----------------- |
| レベル     | 1に戻る              |
| EXP     | 0に戻る              |
| 職業      | 選択した職業に変更         |
| 転職回数    | +1                |
| HP      | 新しい最大HPまで全回復      |
| 基礎ステータス | 引き継ぎ計算後の値に再設定     |
| 装備      | 維持                |
| 所持GOLD  | 維持                |
| 所持装備    | 維持                |
| エリア解放状況 | 維持                |
| ボス撃破状況  | 維持                |
| PvPランク  | 維持、または将来シーズン制で別管理 |
| 公開ログ    | 転職ログを流す           |

## 2.3 転職の位置づけ

転職は「強くてニューゲーム」ではない。

転職直後はLv1に戻るため、一時的に弱くなる。
ただし、1周目のLv1よりは強く、再育成することで前回のLv100より少し強くなる。

この「少し強くなる」を繰り返すことが長期育成の軸になる。

## 4. 能力引き継ぎの考え方

## 4.1 絶対に避けたいこと

以下は避ける。

```text
転職するたびに能力が倍々で増える
古参プレイヤーに新規が絶対追いつけない
PvPが転職回数だけで決まる
ボスや通常敵のバランスが崩壊する
```

そのため、転職時に現在能力をそのまま大きく引き継いではいけない。

## 4.2 引き継ぎ設計の目標

転職後、再びLv100まで育成したとき、前回Lv100時点より2〜3割程度強くなることを目標にする。

例。

```text
1周目 Lv100時点のSTR：250
↓
転職
↓
2周目 Lv100時点のSTR：300〜325程度
```

この程度なら、成長実感はあるが、インフレは抑えられる。

## 4.3 引き継ぎ対象

転職時に引き継ぐのは、装備込みステータスではなく、基礎ステータスのみとする。

理由は、装備込みで計算すると、強い装備をつけた状態で転職することで能力引き継ぎが不自然に増えてしまうため。

```text
引き継ぎ対象：基礎ステータス
引き継ぎ対象外：装備補正込みステータス
```

## 5. 推奨する能力引き継ぎ計算式

## 5.1 基本式

転職後のLv1基礎ステータスは、以下で計算する。

```text
転職後基礎能力 =
初期基礎能力
+ floor((転職前基礎能力 - 初期基礎能力) × 引き継ぎ率)
```

## 5.2 引き継ぎ率

フェーズ1では、引き継ぎ率は 15% を推奨する。

```text
引き継ぎ率：15%
```

理由は、Lv100までの成長分の15%をLv1時点に戻すことで、2周目Lv100時点の能力が1周目より2〜3割程度高くなりやすいため。

## 5.3 なぜ4分の1ではなく15%か

以前の案では「能力は4分の1程度まで低下」としていた。

ただし、単純に現在能力の25%を残すと、初期値が低い能力では思ったより大きく残り、転職を繰り返すほど伸びが速くなる可能性がある。

そのため、以下のようにする方が安全。

```text
現在能力そのものの25%を残す
```

ではなく、

```text
初期値から伸びた分の15%だけを残す
```

この方がインフレを抑えやすい。

## 5.4 計算例

初期STRが9、Lv100時点の基礎STRが250の場合。

```text
転職後STR =
9 + floor((250 - 9) × 0.15)

= 9 + floor(241 × 0.15)
= 9 + 36
= 45
```

転職後はLv1でSTR45になる。

その後、再びLv100まで育成すると、1周目より強くなるが、極端なインフレにはなりにくい。

## 5.5 HPの引き継ぎ率

HPは他能力より数値が大きくなりやすい。

そのため、HPだけ引き継ぎ率を低くする。

推奨。

```text
HP引き継ぎ率：10%
その他能力引き継ぎ率：15%
```

例。

初期HPが105、Lv100時点の基礎HPが450の場合。

```text
転職後HP =
105 + floor((450 - 105) × 0.10)

= 105 + floor(345 × 0.10)
= 139
```

転職後Lv1のHPは139になる。

## 5.6 転職回数による補正

フェーズ1では、転職回数による追加倍率は入れない。

理由は、引き継ぎ率に加えて転職回数補正まで入れると、インフレしやすくなるため。

将来的に入れる場合でも、上限を設ける。

例。

```text
転職回数補正は最大 +10% まで
```

ただし、フェーズ1では不要。

## 6. 転職時の具体処理

## 6.1 転職前チェック

転職ボタン押下時、以下をチェックする。

* ログイン済み
* キャラクター存在
* キャラクターLv100以上
* 転職先職業が存在する
* 転職先職業が有効
* 転職先の解放条件を満たしている
* 戦闘中ではない

## 6.2 転職処理フロー

```text
転職ボタン押下
↓
条件チェック
↓
転職前の基礎能力を取得
↓
初期基礎能力を取得
↓
引き継ぎ計算
↓
職業を変更
↓
レベルを1に戻す
↓
EXPを0に戻す
↓
転職回数を+1
↓
基礎能力を引き継ぎ後の値に更新
↓
現在HPを新しい最大HPにする
↓
転職履歴を保存
↓
公開ログを作成
↓
転職完了画面を表示
```

## 6.3 トランザクション

転職処理は必ずDBトランザクションで行う。

理由は、以下が途中で失敗するとデータ不整合が起きるため。

* レベルリセット
* 職業変更
* 能力更新
* 転職回数更新
* 転職履歴保存

Lv100以上の場合、転職可能な職業にボタンを表示する。

```text

[戦士に転職する]

[魔法使いに転職する]
```

## 9.4 現在職業への転職

同じ職業への転職は許可する。

理由は、同じ職業を繰り返して特化型育成したいプレイヤーがいるため。

例。

```text
戦士 → 戦士
```

これにより、STR特化やDEF特化が作りやすくなる。

ただし、将来的に職業履歴による上位職解放を入れる場合も、同職転職履歴は保存する。

## 13. レベルアップ処理との連携

## 13.1 職業ごとの成長値を使う

今後のレベルアップでは、現在職業の jobs テーブルに設定された成長値を使う。

現在の固定成長から、職業別成長へ切り替える。

処理例。

```text
現在職業を取得
↓
jobs の growth_min / growth_max を取得
↓
各能力をランダム成長
↓
キャラクター基礎ステータスに加算
```

## 13.2 Lv100上限

フェーズ1では、Lv100を上限とする。

Lv100到達後は、EXPを増やさないか、増やしてもレベルアップしない。

推奨は以下。

```text
Lv100到達後もEXPは増えない
```

理由は、転職を促すため。

## 13.3 Lv100到達ログ

Lv100到達時、公開ログに流す。

```text
【成長】テストプレイさんがLv100に到達しました！
```

また、左カラムの次の目標を以下にする。

```text
Lv100に到達しました。転職所で新たな職業を選びましょう
```

## 15. 引き継ぎ計算詳細

## 15.1 初期基礎能力

初期基礎能力は、職業ではなくキャラクターの基本初期値を基準にする。

例。

```text
HP 105
STR 9
DEF 9
AGI 8
MAG 12
LUK 6
```

この値は設定値として管理する。

## 15.2 計算式

HP。

```text
after_max_hp =
initial_hp + floor((before_max_hp - initial_hp) × 0.10)
```

その他能力。

```text
after_stat =
initial_stat + floor((before_stat - initial_stat) × 0.15)
```

## 15.3 最低保証

計算結果が初期値を下回る場合は、初期値に丸める。

```text
after_stat = max(initial_stat, calculated_stat)
```

## 15.4 小数点処理

小数点以下は floor で切り捨てる。

## 15.5 装備補正は除外

before_stat は基礎能力を使う。

装備込みステータスは絶対に使わない。

## 22. バランス調整方針

## 22.1 引き継ぎ率はまず低めにする

最初から高く設定しない。

推奨初期値。

```text
HP：10%
その他能力：15%
```

実際のテストで、2周目Lv100時点が1周目より弱すぎる場合は、以下の範囲で調整する。

```text
HP：10〜12%
その他能力：15〜18%
```

20%以上は慎重に扱う。

## 22.2 目標値

2周目Lv100時点で、1周目Lv100より2〜3割強い程度を目標にする。

## 22.3 PvP実装前に必ず確認

PvP実装前に、以下を確認する。

* 転職1回と0回の能力差
* 転職2回以上の伸び方
* 装備込みで差が開きすぎていないか
* ボス難易度が崩壊していないか

## 24. 結論

転職は、冒険都市ヴァルゼリアの長期育成を支える重要機能である。

ただし、能力引き継ぎを強くしすぎると、ステータスインフレが起き、PvPや敵バランスが壊れる。

そのため、フェーズ1では以下の控えめな引き継ぎ率を採用する。

```text
HP：成長分の10%
その他能力：成長分の15%
```

転職後はLv1に戻るが、成長した能力の一部を引き継ぐ。

この仕様により、プレイヤーは「もう一度育てる意味」を持ちながら、極端なインフレを避けられる。

まずは初期4職、Lv100転職、控えめな能力引き継ぎから実装する。
