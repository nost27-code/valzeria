# 鍛冶屋装備のバランス調整計画

鍛冶屋で作成できる装備品が、店売りの装備品に比べて異常に強く（必要レベルも極端に低く）設定されている問題を修正するための実装計画です。

## 1. 目的と変更の概要
- **必要レベルの据え置き**: その街に到達できれば作成できるものとするため、必要レベルの引き上げは行わず、現在の設定（Lv0 / Lv1 等）を維持します。
- **能力値（STR/DEF/MAG等）の適正化**: 極端に高く設定されている基礎ステータスを、同街の店売り装備の最大値の「**約1.2倍**」程度に下方修正します。（素材を集めて作る価値を持たせるため）

## 2. 実装のステップ

### Step 1: 調整用スクリプトの作成（PHP）
Windowsのコマンドラインでの一括置換は文字コード問題を引き起こす可能性があるため、PHPの専用スクリプトを作成して安全に処理を行います。
スクリプトは以下の流れで処理します：
1. データベースの `recipes` テーブルを参照し、各鍛冶屋装備が「どの街」で作れるかを特定します。
2. 該当する街の「店売り装備」のステータス（最大のSTRやDEF）を取得します。
3. 鍛冶屋装備の能力値を以下のように再計算します：
   - 武器の主要能力（STR, MAG等）= 店売り最大値 × 1.2
   - 防具の主要能力（DEF, SPR等）= 店売り最大値 × 1.2
   ※AGIやLUKなどのサブステータスは、現在の値を概ね維持します。
   ※必要レベル（required_level）は変更しません。
4. プロジェクト直下の `weapons_data.tsv` および `armors_data.tsv` を読み込み、対象となるアイテムの行を再計算した数値で上書きし、UTF-8で保存し直します。

### Step 2: シーダーの実行とDB反映
更新されたTSVファイルを元に、以下のコマンドでデータベースを更新します。
- `php artisan db:seed --class=WeaponsSeeder`
- `php artisan db:seed --class=ArmorsSeeder`

### Step 3: 確認
再度、街の店売り装備と鍛冶屋装備を比較表示し、適正なバランス（店売りの1.15倍程度）に収まっていること、及び必要レベルが正しく設定されていることを確認します。

---

> [!IMPORTANT]
> **User Review Required**
> 
> 能力値の倍率を「店売り装備の **1.15倍**」としていますが、この倍率でよろしいでしょうか？（もっと強くしたい／弱くしたい等のご要望があれば調整いたします）
> また、必要レベルについては「店売り装備と**同じレベル**」に合わせる予定です。（例：店売りがLv200なら鍛冶屋もLv200）。こちらについても問題ないかご確認ください。
