# 冒険都市ヴァルゼリア フェーズ1 武具屋・装備 実装仕様書

## ファイル名

valzeria_phase1_shop_equipment_spec.md

---

## 1. 目的

本仕様書は、冒険都市ヴァルゼリアのフェーズ1実装において、武具屋、装備購入、所持装備、装備変更、装備補正込みステータス、戦闘への装備反映を実装するための仕様を定義する。

現在、迷宮探索と戦闘処理は実装済みであり、戦闘に勝利するとEXPとGOLDを獲得できる状態である。

しかし、現状ではGOLDの使い道がないため、戦闘報酬としてGOLDを得る意味が弱い。

本仕様では、GOLDを使って武器、防具、装飾品を購入し、装備によってステータスを強化し、より強い敵やボスに勝てるようにする。

これにより、以下の育成ループを成立させる。

```text
迷宮へ行く
↓
敵を倒す
↓
EXPとGOLDを得る
↓
武具屋で装備を買う
↓
装備を変更する
↓
ステータスが上がる
↓
強い敵に勝てる
↓
ボスを倒せる
↓
次のエリアへ進む
```

フェーズ1では、複雑な装備強化や売却、倉庫、鑑定は実装せず、まずは「買う」「所持する」「装備する」「戦闘に反映する」までを完成させる。

## 3. 現在の課題

## 3.1 GOLDの使い道がない

現状、戦闘でGOLDを獲得できるが、消費先がない。

そのため、戦闘報酬としてのGOLDの価値が弱い。

武具屋を実装することで、GOLDに以下の意味を持たせる。

* 武器を買う
* 防具を買う
* 装飾品を買う
* 強くなる
* 次のエリアやボスに挑む準備をする

## 3.2 装備による成長実感がない

レベルアップだけだと、成長がやや単調になる。

装備を導入することで、プレイヤーは「次に買いたい装備」を目標にできる。

例。

```text
鉄の剣を買うためにGOLDを貯める
↓
鉄の剣を装備する
↓
STRが上がる
↓
敵へのダメージが増える
↓
小鬼の森で勝ちやすくなる
```

この流れが、FFAらしい育成の中毒性につながる。

## 5. 装備カテゴリ仕様

## 5.1 装備カテゴリ

装備カテゴリは以下の3つとする。

| カテゴリ | type      | 役割                 |
| ---- | --------- | ------------------ |
| 武器   | weapon    | 主にSTR、MAGを上げる      |
| 防具   | armor     | 主にDEF、最大HPを上げる     |
| 装飾品  | accessory | 主にAGI、LUK、補助能力を上げる |

## 5.2 装備スロット

キャラクターは各カテゴリにつき1つずつ装備できる。

| スロット      | 装備可能カテゴリ  |
| --------- | --------- |
| weapon    | weapon    |
| armor     | armor     |
| accessory | accessory |

## 5.3 左カラム表示

現在の装備欄には以下を表示する。

```text
現在の装備

武　木の剣
鎧　皮の鎧+1
飾　魔除けの護符
```

フェーズ1では、強化値 `+1` はまだ実装しない。
既存表示に `皮の鎧+1` がある場合は、仮表示として扱うか、初期データから一旦外してよい。

フェーズ1の正式表示例。

```text
現在の装備

武　木の剣
鎧　布の服
飾　魔除けの護符
```

## 6. 装備マスター仕様

## 6.1 items テーブル

装備データは `items` テーブルで管理する。

テーブル名。

```text
items
```

カラム案。

| カラム            | 型         | 内容                      |
| -------------- | --------- | ----------------------- |
| id             | bigint    | 主キー                     |
| name           | string    | 装備名                     |
| type           | string    | weapon、armor、accessory  |
| description    | text      | 説明文                     |
| rarity         | string    | normal、rare、epic、legend |
| price          | int       | 購入価格                    |
| hp_bonus       | int       | 最大HP補正                  |
| str_bonus      | int       | STR補正                   |
| def_bonus      | int       | DEF補正                   |
| agi_bonus      | int       | AGI補正                   |
| mag_bonus      | int       | MAG補正                   |
| luk_bonus      | int       | LUK補正                   |
| required_level | int       | 必要レベル                   |
| is_shop_item   | boolean   | 店売り対象か                  |
| is_active      | boolean   | 有効か                     |
| sort_order     | int       | 表示順                     |
| created_at     | timestamp | 作成日時                    |
| updated_at     | timestamp | 更新日時                    |

## 6.2 rarity

レアリティは以下とする。

| rarity | 表示名  | 用途              |
| ------ | ---- | --------------- |
| normal | 通常   | 店売り基本装備         |
| rare   | レア   | 少し強い装備、将来のドロップ用 |
| epic   | エピック | 高性能装備、将来用       |
| legend | 伝説   | 最上位装備、将来用       |

フェーズ1では、基本的に `normal` の店売り装備のみでよい。

## 6.3 is_shop_item

`is_shop_item = true` の装備のみ、ショップに表示する。

将来、ドロップ限定装備やイベント装備を実装する場合は `is_shop_item = false` にする。

## 6.4 is_active

`is_active = false` の装備はショップ表示や装備候補から除外する。

## 7. 所持装備仕様

## 7.1 character_items テーブル

キャラクターが所持している装備は `character_items` で管理する。

テーブル名。

```text
character_items
```

カラム案。

| カラム           | 型         | 内容                          |
| ------------- | --------- | --------------------------- |
| id            | bigint    | 主キー                         |
| character_id  | bigint    | キャラクターID                    |
| item_id       | bigint    | 装備ID                        |
| is_equipped   | boolean   | 装備中か                        |
| equipped_slot | string    | weapon、armor、accessory、null |
| acquired_from | string    | initial、shop、drop、admin     |
| created_at    | timestamp | 入手日時                        |
| updated_at    | timestamp | 更新日時                        |

## 7.2 同一装備の複数所持

フェーズ1では、同じ装備を複数購入できる仕様でよい。

理由は、実装がシンプルであり、将来の売却や倉庫にもつなげやすいため。

ただし、同じ装備を何個も持つ意味はまだ薄いため、UI上では以下のどちらかを選ぶ。

案1。

```text
同じ装備でも購入可能
```

案2。

```text
すでに所持している装備は購入不可
```

推奨は案2。
フェーズ1では売却がないため、同じ装備を誤購入すると不満につながる可能性がある。

仕様としては以下を推奨する。

```text
同じ装備は1キャラクターにつき1つまで購入可能
```

## 7.3 初期装備

| weapon    | 木の剣    |
| armor     | 布の服    |
| accessory | 魔除けの護符 |

## 8.4 数値方針

フェーズ1では、装備補正を強くしすぎない。

理由は、ステータスインフレを抑えるためである。

装備によって確かに強くなるが、装備だけでボスを簡単に倒せるほどにはしない。

基本方針。

```text
レベルアップによる成長 + 装備補正 + エリア相性
```

この3つの合算で勝敗が決まるようにする。

## 12. ステータス計算仕様

## 12.1 基本方針

キャラクターのステータスは、基礎ステータスと装備補正を分けて管理する。

DBの characters テーブルには基礎ステータスを保存する。

表示や戦闘では、装備補正込みの最終ステータスを計算して使用する。

```text
最終ステータス = 基礎ステータス + 装備補正
```

## 12.2 装備補正対象

装備で補正される能力。

* 最大HP
* STR
* DEF
* AGI
* MAG
* LUK

## 12.3 現在HPの扱い

最大HPが装備によって増減する場合、現在HPの扱いに注意する。

フェーズ1では、以下の仕様を採用する。

### 最大HPが増える装備を装備した場合

現在HPは増えない。

例。

```text
装備前：現在HP 90 / 最大HP 108
装備後：現在HP 90 / 最大HP 118
```

理由は、装備変更だけで回復できてしまうのを防ぐため。

### 最大HPが下がる装備に変更した場合

現在HPが新しい最大HPを超えている場合は、新しい最大HPに丸める。

例。

```text
装備前：現在HP 115 / 最大HP 118
装備後：最大HP 108
現在HPは108に調整
```

## 12.4 ステータス表示

左カラムでは、装備補正込みの最終ステータスを表示する。

表示例。

```text
STR 17
DEF 11
AGI 10
MAG 16
LUK 7
```

## 12.5 基礎値と補正値の表示

フェーズ1では、左カラムには最終値のみ表示でよい。

将来的には、詳細画面で以下のように表示してもよい。

```text
STR 17
基礎値 10 + 装備 7
```

## 13. 戦闘への装備反映

## 13.1 基本方針

戦闘時は、必ず装備補正込みのステータスを使用する。

BattleService内で characters テーブルの基礎値を直接使わず、ステータス計算サービスを通す。

## 13.2 CharacterStatusService

装備補正込みステータスを返すサービスを用意する。

サービス名案。

```text
CharacterStatusService
```

責務。

* 基礎ステータス取得

## 18.3 購入処理のトランザクション

購入処理はDBトランザクション内で行う。

理由は、GOLD減算と所持装備作成を一体で扱うため。

処理イメージ。

```text

  character.gold -= item.price
  character_items create
commit
```

## 19.3 装備処理のトランザクション

装備変更はDBトランザクション内で行う。

処理イメージ。

```text

  同じ equipped_slot の装備を is_equipped = false にする

```

| --------- | ------------- |

| accessory | accessory     |

## 20. CharacterStatusService仕様

## 20.1 責務

CharacterStatusServiceは、基礎ステータスと装備補正を合算して最終ステータスを返す。

## 20.2 返却データ例

```text

```

## 20.3 補正計算

```text
最終最大HP = character.max_hp + 装備 hp_bonus 合計
最終STR = character.str + 装備 str_bonus 合計
最終DEF = character.def + 装備 def_bonus 合計
最終AGI = character.agi + 装備 agi_bonus 合計
最終MAG = character.mag + 装備 mag_bonus 合計
最終LUK = character.luk + 装備 luk_bonus 合計
```

## 20.4 最低値

## 23. 戦闘結果への影響確認

## 23.1 装備前後の変化

装備変更後、戦闘で以下の変化が起きること。

* STRが上がると与ダメージが増える
* DEFが上がると被ダメージが減る
* AGIが上がると先攻や回避に影響する
* LUKが上がると会心率に影響する
* 最大HPが上がると耐久力が上がる

## 23.2 BattleService側の修正

BattleServiceでキャラクター基礎値を直接参照している場合、装備補正込みの値に差し替える。

修正前。

```text
character.str
character.def
character.agi
```

修正後。

```text
battleStats.str
battleStats.def
battleStats.agi
```

## 27. バランス調整方針

## 27.1 序盤価格

序盤装備は、数回戦闘すれば買える価格にする。

目安。

| 装備段階  |      価格感 |
| ----- | -------: |
| 初期装備  |       0G |
| Lv2装備 |  80〜200G |
| Lv5装備 | 300〜500G |

## 27.2 GOLD報酬との関係

現在の敵から得られるGOLDが以下程度の場合。

```text
スライム：6G
迷いウサギ：8G
草原コウモリ：10G
小鬼：14G
```

鉄の剣120Gは、序盤で10〜20戦程度で買える。

このくらいなら、目標として成立する。

## 27.3 装備補正の強さ

装備補正は強すぎないようにする。

特に、鋼の剣や鉄の鎧を装備した時点で小鬼の森が簡単になりすぎる場合は、価格または補正値を調整する。

## 29. 結論

今回の実装では、GOLDの使い道を作り、装備によってキャラクターを強化できるようにする。

これにより、冒険都市ヴァルゼリアの基本ループは以下の形になる。

```text
戦う
↓
GOLDを得る
↓
装備を買う
↓
強くなる
↓
強敵に勝てる
```

フェーズ1の時点では、売却、強化、倉庫、鑑定などは不要である。

まずは、武器屋、防具屋、装飾屋、購入、装備変更、戦闘反映までを確実に通す。

この実装が完了すると、ヴァルゼリアは「戦闘して経験値が増えるゲーム」から、「GOLDを稼いで装備を整え、強敵に挑むFFA風RPG」へ進化する。
