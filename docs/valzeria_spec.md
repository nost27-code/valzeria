# ヴァルゼリアの冒険者 仕様書

この文書は、今後の実装判断で優先して参照する正仕様です。既存コード、古いdocs、Seeder、設定ファイルに異なる記述が残っている場合でも、この文書とユーザー指定の最新前提を優先します。実装との差分は「要確認事項」に記録し、勝手に旧仕様へ戻さないでください。

## ゲーム概要

- ゲーム名: ヴァルゼリアの冒険者
- ドメイン: valzeria.com
- ジャンル: FFA風スマホ向けブラウザRPG
- 目指す体験: 街を拠点に、ダンジョン探索、戦闘、回復、装備更新、職業育成、ドロップ収集、ボス攻略を短いテンポで繰り返す。
- 基本ループ: 街 -> ダンジョン選択 -> 通常探索またはボス挑戦 -> 戦闘 -> 経験値、職業経験値、ドロップ、まれなGold、輝石などの報酬 -> レベルアップ、BP付与、職業ランク成長 -> 回復、装備、育成 -> ボス撃破で次ダンジョンまたは次都市を解放。
- UI方針: スマホ優先。FFA/CGIゲームらしいテキスト中心の懐かしさを残しつつ、カード型、リスト型で見やすくする。過剰演出や重いSPA化は避ける。

## 現在の確定仕様

- Goldは通常ゲーム内通貨。主な入手は素材・装備・換金品の売却で、敵からの直接入手は低確率かつ少額にする。
- 課金/支援通貨は「輝石」。有償輝石、無償輝石、合計輝石を扱う。Goldと輝石は役割を分け、Goldを課金必須通貨にしない。
- Lv上限は255。
- レベルアップ時にBPを1付与し、プレイヤーが任意ステータスへ割り振る。
- ステータスは HP / SP / ATK / DEF / MAG / SPR / SPD / LUK（2026-07-02裁定）。MPは旧表記または内部カラム名（`mp_base` 等）としてのみ残し、プレイヤー向け表示・新規ドキュメントでは使わない。DBカラムのリネームは行わない。
- 職業はプレイヤーレベルとは別に職業経験値と職業ランクを持つ。
- 職業ランクは★1から★10。★10でマスター。
- 転職は「Lv30以上 + 要求されている職のマスター」で可能（2026-07-02裁定、現実装どおり）。転職時はLv1に戻り、能力値を一定割合で引き継ぐ。Lv100転職案は旧設計・未採用。
- 通常探索とボス挑戦は分ける。ボスは通常探索には出現させず、ボス挑戦のみで出現する。
- 各都市には複数ダンジョンがあり、都市内の最終ダンジョン撃破で次都市が解放される。
- チャンプ戦はFFA風のPvP/疑似PvP要素として導入予定または実装中の要素。
- ヴァルモン牧場、ヴァルモン育成要素は拡張予定または実装中の要素。
- 管理者機能では、マスタデータ管理、不正チェック、複数アカウント対策などを扱う。

## 非採用・廃止済み仕様

以下は今後の正仕様として採用しません。

- Goldから輝石への変換、Goldを前提にした課金誘導、敵が毎回Goldを落とす設計。
- 銀行預金まで対象にする救助費請求、宿屋料金、ショップ購入費、ステータス振り直し費用など、過剰な所持金消費を前提にした設計。
- 装備更新を「購入費を貯めて買う」ことだけに寄せる設計。
- ボスを通常探索のランダム敵として出現させる設計。
- 古いdocsや設定ファイルに残る高額Gold前提の価格設計。
- 金策専用職や過剰なGold増加スキル。gold_bonus 系は低確率Goldやドロップ補助と整合する範囲でのみ扱う。

## プレイヤー成長仕様

- プレイヤーLvは1から255。
- Lv255到達後は追加経験値を増やさない。実装上はLv255到達時に超過経験値を0に戻す挙動がある。
- 必要経験値は現実装では `LevelService::getRequiredExp()` が `Lv * Lv * 20 + Lv * 30` で算出している。
- レベルアップ時は職業倍率、転職回数補正を加味して基礎ステータスが上昇する。
- レベルアップ時にHP/SPは全回復しない。最大値を超えないよう現在値をクランプする。
- レベルアップごとに `bonus_points` を1付与する。
- BPは `BonusPointService` で任意ステータスに割り振る。
- BP割り振りの現実装:
  - HP: +10
  - SP: +5（内部カラムはmp系）
  - ATK/DEF/MAG/SPR/SPD/LUK: +1

## ステータス仕様

正仕様上の表記は以下に統一します。

| 表示名 | 役割 | 実装上の主な対応カラム |
| --- | --- | --- |
| HP | 生命力 | `hp_base`, `current_hp` |
| SP | 技や魔法のリソース（旧表記: MP） | `mp_base`, `current_mp` |
| ATK | 物理攻撃力 | `attack_base`, 装備の `str_bonus` |
| DEF | 物理防御力 | `defense_base`, 装備の `def_bonus` |
| MAG | 魔法攻撃力 | `magic_base`, 装備の `mag_bonus` |
| SPR | 精神力、魔法防御、回復力 | `spirit_base`, 装備の `spr_bonus` |
| SPD | 素早さ | `speed_base`, 装備の `agi_bonus` |
| LUK | 運 | `luck_base`, 装備の `luk_bonus` |

実装には `str`、`agi`、`luck`、`mp_base` など過去名や内部名が残る場合があります。ユーザー向け表示、仕様書、今後の説明では SP / ATK / SPD / LUK に寄せます（MP / STR / AGI は旧表記）。DBカラムのリネームは行いません。

## 職業・転職・職業ランク仕様

- 職業階層は一般職、中級職、上級職、伝説職。
- 複数職業のマスターによって上位職を解放する設計。
- 職業は `job_classes`、職業履歴は `character_jobs`、職業経験値表は `job_exp_tables`、解放条件は `job_requirements` を主に使う。
- 職業ランクは★1から★10。★10でマスター。
- 実装上は `job_level` が職業ランクを表す。ユーザー向けには「職業レベル」ではなく「職業ランク」と呼ぶ。
- 戦闘勝利時に職業経験値を得る。ランクアップ時は「〇〇のランクがNに上がった」の形で表示する。
- 転職条件の正仕様は「Lv30以上 + 要求されている職のマスター」（2026-07-02裁定、現実装どおり）。Lv100到達案は旧設計・未採用であり、コードへ反映しない。Lv100へ変更する場合は別途「転職条件再設計タスク」として扱う。
- 転職時はLv1、EXP0に戻り、基礎能力を一定割合で引き継ぐ。
- 現実装では `CharacterJobChangeService::calculateInheritedStats()` が基礎ステータスの50%を引き継ぎ、最低保証値を適用している。

## 都市・ダンジョン進行仕様

基本都市は以下の10都市です。

1. 王都アークレア
2. 港町マリネス
3. 精霊都市エルフィア
4. 鉄鋼都市グランベルグ
5. 雪都フロストリア
6. 砂都サンドラ
7. 魔導都市ルミナス
8. 魔界都市ネクロム
9. 天空都市セレスティア
10. 魔王城ヴァルゼリア

- 都市マスタは `CitySeeder`、ダンジョンマスタは `AllDungeonsSeeder` が主な実装根拠。
- 王都アークレアが初期都市。
- 各都市の通常ダンジョンはボス撃破で順番に解放する。
- 都市内の通常最終ダンジョンを撃破すると、次都市が解放される。
- 現実装では通常ダンジョンID 1から70を都市進行の本線とし、ID 71以降を裏ダンジョン扱いで都市最終判定から除外している。
- 裏ダンジョン、伝説職解放用ダンジョン、秘境などは基本ループを壊さない範囲の拡張要素として扱う。

## 戦闘仕様

- 戦闘は1対1を基本とする。
- 通常探索とボス挑戦は別ルート、別ボタンで扱う。
- 通常探索では `Enemy::where(is_boss=false)` を抽選する。
- ボス挑戦では `Enemy::where(is_boss=true)` を対象にする。
- 戦闘はターン制の自動戦闘。
- SPD比較と乱数で先攻後攻を決める。
- 物理攻撃、魔法攻撃、命中、回避、会心、職業スキルを扱う。
- 敵タイプごとに行動パターンを変える。例: 魔法型、耐久型、高速型、アンデッド型、竜型、標準型。
- 敗北時はHPを一定割合で残して街または探索終了へ戻す。現実装では最大HPの30%、SPは10%相当になる箇所がある。
- 戦闘結果では、勝敗、与ダメージ、被ダメージ、EXP、職業EXP、ドロップ、レベルアップ、職業ランクアップ、称号、ボス撃破結果を分かるようにする。
- 戦闘報酬のGoldは低確率にする。主な金策はドロップ品・素材・装備・換金品の売却。

## 装備・アイテム・ドロップ仕様

- 装備は武器、防具、装飾品を基本とする。
- 装備変更でステータスへ反映する。
- Goldでの市場購入や合成費用は扱うが、通常成長やドロップ収集を壊さない範囲に抑える。
- ショップ（装備屋）では武器・防具をGoldで購入できる（現実装: `ShopService` が `is_shop_item` 対象の価格判定と所持金チェックを行う）。回復アイテムはショップ購入ではなく補給所で毎日の無料枠として受け取る。
- 売却は正仕様として使う。売却価格が0または未設定の素材・装備は売却不可。不要装備は「売却」または「分解」を選べる。
- ドロップは素材、装備、印、輝石、ヴァルモン卵などを扱う。
- 装備の欠片系は、基本的に「装備の欠片」「上質な装備の欠片」「強装備の欠片」の3段階を正とする。
- 素材交換所は合成とは別施設として扱う。
- 敵、素材、装備、ドロップのID参照整合性を重視する。存在しないIDや名前だけに依存した紐付けを避ける。

## ヴァルモン仕様

- ヴァルモンは相棒、育成、牧場、探索支援の拡張要素。
- 現実装ではスターター選択、相棒設定、卵発見、卵孵化、素材発見、素材/装備を餌にした育成がある。
- ヴァルモンの現実装Lv上限は100。
- 探索中に卵を発見する場合がある。敗北時に卵を失う可能性があるが、救助系支援で保護される場合がある。
- 相棒ヴァルモンは探索中に素材を見つける場合がある。
- 今後はヴァルモン牧場、育成、見た目、役割、収集の楽しさを拡張する予定。

## チャンプ戦仕様

- チャンプ戦はFFA風のPvP/疑似PvP要素。
- 現実装では `ChampBattleService`、`ChampState`、`ChampBattleLog`、`ChampHistory` を中心に構成されている。
- 現在のチャンプに挑戦し、一定ターン内にダメージを与える。チャンプHPを削り切るとチャンプ交代。
- 挑戦にはクールダウンがある。現実装は10分。
- チャンプ戦でもEXP、職業EXP、素材報酬を得る。
- チャンプ自身は自分に挑戦できない。
- ランキング、履歴、公開ログ、トップ画面表示と連動する。

## 課金・輝石仕様

- 課金通貨は輝石。
- 有償輝石、無償輝石、合計輝石を区別する。
- Stripe決済で輝石パックを購入する。
- `config/kiseki.php` が輝石パックの主な設定。
- 戦闘勝利時に低確率で無償輝石がドロップする実装がある。
- 輝石は支援アイテムなどに使う。ゲーム進行の必須通貨にしすぎない。
- 課金・輝石まわりでは二重決済、二重付与、有償/無償の混同、ログ欠落を避ける。
- 輝石をGoldの代替として汎用消費通貨化しない。便利支援、保険、時短、拡張枠などに限定する。

## 管理者機能仕様

- 管理者機能では、マスタデータ管理、プレイヤー状態確認、行動ログ確認、輝石購入管理、不正チェック、複数アカウント対策を扱う。
- 現実装の管理者ルートには、アイテム、職業、プレイヤー、行動ログ、報酬設定、輝石購入、トップ更新の管理がある。
- 管理者操作は本番データに直接影響するため、削除、上書き、Seeder再実行、マスタ大量変更は事前確認を必須とする。
- 不正対策では、短時間大量戦闘、異常な輝石増減、素材/装備増殖、複数アカウント疑い、決済未完了の輝石付与を確認対象にする。

- 伝説職、裏ダンジョン、秘境、極印などの長期目標拡張。
- ヴァルモン牧場、ヴァルモン育成、探索支援、収集要素の拡張。
- チャンプ戦の報酬、ランキング、履歴表示の強化。

## 要確認事項

- 転職条件: 2026-07-02裁定済み。「Lv30以上 + 要求職のマスター」が正仕様（現実装どおり）。要確認事項から除外。
- 転職時の継承率: 現実装は基礎ステータス50% + 最低保証。正仕様として50%で確定するか要確認。
- 転職時の `spirit_fraction` リセット: 現実装では他のfractionはリセットしているが `spirit_fraction` の明示リセットが見当たらないため要確認。
- Gold正仕様: `characters.money`、`enemies.gold_reward`、`battle_logs.gold_gained`、`items.sell_price`、`materials.npc_sale_price`、合成費用を使う。価格はインフレしないよう控えめに設定する。
- `config/valzeria_guide.php` のGold案内は、売却中心の新仕様と合うよう必要に応じて調整する。
- 管理者のアイテム管理に `price`、`sell_price` 入力が残る。正仕様に合わせて非表示または互換値扱いにするか要確認。
- 管理者の行動ログに `Gold +...` 表示が残る。0表示でも旧仕様を想起させるため要確認。
- ショップ画面のソートに price 系が残る。無料支給/ドロップ中心の仕様に合わせるか要確認。
- 職業スキルに `gold` 系 damage_type や gold bonus 名が残る。素材/ドロップ補正に読み替えるか要確認。
- チャンプ戦は実装済み要素があるが、正式な報酬バランス、PvPとしての扱い、ランキング仕様は要確認。
- ヴァルモンは実装済み要素があるが、牧場の正式仕様、育成上限、進化、探索支援の長期設計は要確認。
- 輝石ドロップ率、1日上限、課金支援アイテムの価格と効果は運営バランスとして要確認。
