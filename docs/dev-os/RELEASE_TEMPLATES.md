# 更新履歴・告知テンプレート

3種類の文面を1回の実装ごとに作る。順番は 管理者サマリ → ユーザー告知 → SNS/note（必要時のみ）。
共通原則:
- **不具合修正と機能追加を混ぜない**（見出しを分ける）
- 弱体化・調整は隠さないが、内部理由（不正対策・収益等）は書かない
- ドロップ率・内部確率・計算式の具体値はユーザー告知には書かない
- β版であること・今後調整がありうることを、調整系告知には必ず添える
- 過剰な期待を煽る表現（「超強力」「激レア」等）を使わない

---

## 1. 管理者向け更新サマリ（config/admin_update_summaries.php）

既存ルール（AGENTS.md）に従う。形式:

```php
[
    'id' => 'YYYY-MM-DD-short-slug',
    'date' => 'YYYY-MM-DD',
    'category' => 'added|changed|fixed|balance|internal',
    'title' => '15〜35字の日本語',
    'detail' => 'プレイヤー向けにそのまま転記できる1〜2文。',
],
```

サンプル:
```php
[
    'id' => '2026-07-05-valmon-ranch-backgrounds',
    'date' => '2026-07-05',
    'category' => 'added',
    'title' => '牧場の背景を追加',
    'detail' => 'ヴァルモン牧場の背景を輝石ショップから購入して切り替えられるようにしました。',
],
```

## 2. ユーザー向けお知らせ（ゲーム内/公式）

テンプレート:

```
【アップデート】YYYY/MM/DD

▼ 追加
・<新機能を1行。入手方法・場所は書くが、確率は書かない>

▼ 変更・調整
・<変更内容と、プレイヤーへの影響を正直に。弱体化は「〜を調整しました」でなく「〜を引き下げました」と書く>

▼ 不具合修正
・<「〜される問題を修正しました」の形。原因の内部詳細は書かない>

※ヴァルゼリアの冒険者はβ版として運営中です。今後もバランス調整（能力や獲得量の引き下げを含む）を
　行う場合があります。ご了承ください。
```

サンプル:
```
【アップデート】2026/07/05

▼ 追加
・ヴァルモン牧場に背景の着せ替え機能を追加しました。輝石ショップから購入できます。

▼ 変更・調整
・秘境主の強さを引き上げ、序盤エリアでの秘境晶片の獲得量を控えめにしました。

▼ 不具合修正
・10回探索中に深度入口へ到達した際、進行ボタンが表示されない問題を修正しました。

※β版のため、今後もバランス調整を行う場合があります。
```

## 3. SNS / note向け告知

方針: 新規獲得向け。ゲームの雰囲気>網羅性。修正の羅列はnoteへ、SNSは目玉1つに絞る。

SNS（X等・140字目安）テンプレート:
```
【ヴァルゼリアの冒険者 更新】
<目玉機能を1つ、体験ベースで1〜2文>
<プレイ導線: ブラウザで無料 / URL>
#ブラウザゲーム #ヴァルゼリアの冒険者
```

サンプル:
```
【ヴァルゼリアの冒険者 更新】
相棒ヴァルモンの牧場に背景着せ替えが登場。お気に入りの風景でヴァルモンを眺められるようになりました🏞️
ブラウザで無料プレイ → valzeria.com
#ブラウザゲーム #ヴァルゼリアの冒険者
```

note記事テンプレート:
```
# 【更新情報】<目玉のタイトル>（YYYY/MM/DD）

いつも遊んでいただきありがとうございます。ヴァルゼリアの冒険者、本日の更新内容です。

## 今回の目玉: <機能名>
<スクショ1枚 + 何ができるか、どう楽しいかを2〜4文。開発の意図を1文添えると読み物になる>

## そのほかの変更
<ユーザー向けお知らせの「変更・調整」「不具合修正」を転記>

## 今後について
<次に予定しているものを、確約しない書き方で1〜2行。「準備中です」「検討しています」>

ヴァルゼリアの冒険者はβ版として運営しています。バランス調整や仕様変更を行うことがありますが、
より長く楽しめるゲームにするための調整ですので、ご理解いただけると嬉しいです。
```

---

## 運用手順

1. 実装完了時: Codexの報告から admin_update_summaries.php 追記（Codexにやらせる）
2. デプロイ前: UPDATE_LOG.md の Unreleased に転記
3. デプロイ後: ユーザー向けお知らせを作成・掲示
4. 目玉がある更新のみ: SNS/note文面を作成（note RSS通知が全キャラの通知ベルに流れる点に留意。小さな更新でnoteを書くと通知が薄まる）
