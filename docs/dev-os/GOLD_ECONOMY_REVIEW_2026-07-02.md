## 1. Gold経済サマリー

構造は健全。Goldの発行（ミント）経路は「素材/装備/換金品の売却」「NPC調達納品報酬」「低確率の戦闘Gold」に絞られ、消滅（バーン）経路は「宿代・鍛冶・進化・素材交換・倉庫拡張・市場手数料3%・NPC出品購入・敗北ロスト10%」と多層。プレイヤー間市場は手数料3%を除きGold移転であり発行ではない。**最大の不確定要素はNPC調達納品の報酬単価**（新規発行でありながらDB設定値をこのレビューで未照会）と、**終盤の売却収入の実分布**。全Gold増減が `gold_transactions.type` 付きで記録済みのため、推測ではなく実データで検証できる下地が既にある——これが本レビューの最重要事実。

## 4. コード・Seeder・docs間の不整合

1. **【要確認・重要】装備売値の正本が二重**: コードは `items.sell_price`（DBカラム）を参照するが、config/gold.php にもランク別 `equipment_sell_prices` 表が存在する。configが「Seederの初期値供給用」なのか「死に設定」なのか未確認。両者が乖離するとバランス調整時に片方だけ直す事故になる。
2. **DOMAIN_RULES.md Economy節の先頭3行がプレースホルダのまま**（`Gold: <short>` / `輝石: <short>` / `Paid/free: <short>`）。経済の正典ファイルの冒頭が空欄なのは象徴的な欠落。
3. **武器価格docsの二重系統**: buki.md（例: 潮騎士の剣 25,000G）と buki_normalized.tsv（同 61,490G）で同一装備の価格が2.5倍違う。DB実値がどちらかは未照会（**仮置き**）。マスタdocsを価格の根拠に使ってはいけない実例。
4. 協会ランク別軽減表（コード内RANKS・実装指示書）と実効の一律10%の乖離 — 裁定済み（停止中と記録済み）だが、コード内定数は将来の再開時に要再検証。
5. 敵Goldドロップ率: config既定5%/12%に対し、docsにはこの率の明記がない（ai_dev_rulesは「低確率」とのみ）。DOMAIN_RULESへの数値記載を推奨。

## 5. 序盤/中盤/終盤のGold体感予測（仮置き前提の定性評価）

- **序盤（アークレア〜マリネス）**: 収入は素材売却頼み。ショップ武器がマリネス帯で2.5万G（buki.md基準・仮置き）だとすると、敵Gold期待値数G/戦では購入は遠く、**装備更新はドロップ・支給が主導**という設計意図に合致。宿代10G固定と救済宿泊で詰み防止済み。**リスク: 探索力500消化分の素材売却収入が装備1個分に届かないと「買える楽しみ」が長期間発生しない**（実単価の照会が必要）。
- **中盤（グランベルグ〜ルミナス）**: 鍛冶+3〜+4（B〜A倍率で数千〜4万G）、倉庫拡張、素材交換のGold費が立ち上がる。調達納品報酬が乗る時期でもあり、**発行と消滅が最も拮抗する帯**。ここでの gold_transactions 実測が最優先。
- **終盤（ネクロム〜魔王城以降）**: 供給側はSS以上装備売却0Gと敵EXP抑制（サンドラ以降）に呼応して伸びが鈍る一方、消滅側はSS/SSS/EPIC鍛冶（1回9万〜180万G）が支配的。**構造上はデフレ気味＝健全**。ただし換金品・調達単価が終盤帯で高額設定ならここが崩れる（仮置き）。

## 6. インフレリスク（順位付き）

1. **NPC調達納品報酬**: 唯一の「労働型ミント」。reward_gold_per_unit×納品量に日次上限が見当たらない（未確認）。素材はドロップで無限供給されるため、単価が高いと「探索→納品」が無限錬金になる。
2. **換金品の売値×ドロップ率の積**: 高額換金品が高頻度で落ちる設定になっていないか（仮置き）。
3. **黄金ゴブリン**: 平均Gold×倍率。10回探索で遭遇頻度が上がっており、倍率次第でスパイクが常態化する。
4. 敵Gold直接ドロップ: 5%抽選＋少額基準で構造的に低リスク。
5. 銀行利息・ログインボーナス等の無条件ミント: **存在しない（確認済み）**。健全。

## 7. 課金・輝石との競合リスク

- **倉庫拡張が唯一の直接競合**: 輝石50で+500 vs Gold100万で+500（10万×10回）。「即時=輝石、コツコツ=Gold」の役割分担は成立しており、暗黙の相場「輝石1≒2万G」を形成している。**今後Gold建て商品を追加する際はこの相場感と矛盾しないこと**（例: 輝石5の小瓶に相当する効果をGold10万未満で売らない）。
- 探索力回復（小瓶/薬）は輝石専売でGold代替なし — 競合なし、課金の主力として保護されている。妥当。
- 救助保険証・緊急救助要請の入手経路が輝石建てかGold建てか未確認（**仮置き**）。輝石建てなら「敗北時の保険」という課金価値が敗北Goldロストの銀行回避（後述）と整合するか要確認。
- **課金圧の見え方**: Gold詰まり（装備買えない・強化できない）が長引くと補給商会への誘導が「圧」に見える。現状は探索力回復のみの販売なので直接の圧は低いが、序盤の売却収入が細すぎる場合は間接圧になる。

## 10. 追加すると効果が大きいGoldシンク案（提案のみ）

1. **街の復興（既定路線・最有力）**: 復興予定素材9種＋Gold拠出の組み合わせ。都市ごとの復興段階×累進Gold要求は、終盤の余剰Goldと余剰素材を同時に吸える。番付（復興貢献）とも接続可能。
2. **協会寄付の再開**: 実装済み・閾値1億Gまでの累進設計が既にある。救助費軽減は弱い恩恵なので、称号・冒険者カード装飾・番付露出を主報酬にすれば経済歪みなく大量吸収できる。
3. **牧場のGold建て装飾**: 背景21種は輝石販売予定のため**競合回避が必須**。輝石=背景（見た目の主役）、Gold=柵・小物・餌皿など補助装飾、と階層を分ければ両立する。
4. **鍛冶の「やり直し」課金（Gold建て銘リロール等）は非推奨**: 射幸性が強く、β段階では事故リスク（誤消費・弱体化告知の負担）が大きい。
5. 市場の出品枠拡張（Gold建て・回数制）: listing_fee=0の現状を維持したまま、ヘビー出品者からだけ回収できる。
