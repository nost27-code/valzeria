# 画像生成依頼テンプレート

用途別6種。生成後は必ず「出力後の確認観点」を通してから public/images/ へ配置する。
共通ルール:
- ファイル形式は webp に変換して配置（既存慣習: icon_XXX.webp、npc_XXX.webp 等の連番）
- 文字（日本語/英語）を画像に入れない（UIテキストはBlade側で重ねる）
- 実在IP・有名ゲームのキャラ・ロゴに似せない
- 参考画像を渡す場合は「構図参照」「画風参照」「色参照」のどれかを明記し、模写はさせない

---

## 1. 背景画像用（都市・ダンジョン・施設ヘッダー・牧場背景）

```
目的: <例: ヴァルモン牧場の購入背景（輝石ショップ販売分）>
使用場所: <例: 牧場画面の背景。上に暗いオーバーレイ(bg-slate-900/60)+白テキストが載る>
画風: ファンタジーRPGの背景イラスト。手描き風、落ち着いた彩度。写実的すぎない
構図: <例: 広めの空間、中央下1/3にキャラが立てる余白。強い被写体を中央に置かない>
サイズ・比率: 16:9（1600x900目安）。スマホでは中央クロップされる前提で左右端に重要要素を置かない
背景有無: 背景そのもの
透過: 不要
絶対に守る条件:
- 人物・モンスターを描き込まない（キャラは別レイヤーで載る）
- 文字・ロゴ・UIを入れない
- オーバーレイ後も視認できる明度差（暗すぎる夜景はNG、白テキストが沈む白飛びもNG）
避ける表現: 現代物（電線・車）、過度なホラー、宗教シンボル
参考画像: <既存の同系統背景を1枚添付し「この画風・彩度に合わせる」と指定>
出力後の確認: 16:9か / スマホ幅中央クロップで成立するか / オーバーレイ+白文字を重ねて視認できるか / 既存背景と並べて彩度が浮かないか
```

## 2. ドット絵キャラ用（敵・NPC・ヴァルモン）

```
目的: <例: 新ダンジョンの通常敵「◯◯」>
使用場所: <例: 戦闘画面の敵表示>
画風: 16bit風ドット絵（SFC後期RPG調）。輪郭は暗色1px、アンチエイリアス最小限
構図: 正面〜やや斜め向き、全身、足元まで。キャンバス中央、上下左右に4〜8pxの余白
サイズ・比率: <例: 96x96 または 128x128>。拡大表示前提なのでドットが潰れない整数倍で
背景有無: なし
透過: 必須（透過PNG→webp変換）
絶対に守る条件:
- 既存の敵/NPCドット絵と等身・ドット密度を揃える（参考画像必須）
- 影は足元に楕円1つまで
避ける表現: グラデーション多用、3Dレンダ風、リアル等身
参考画像: <既存スプライト2〜3体を「等身・密度参照」として添付>
出力後の確認: 透過が抜けているか（白フチ残りがないか）/ 既存敵と並べて等身が揃うか / 縮小表示（スマホ）で判別できるか
```

## 3. スプライトシート用

```
目的: <例: ◯◯の歩行/待機アニメーション>
使用場所: <例: 探索画面の相棒表示。public/tools/sprite-splitter.html で分割する>
画風: テンプレート2に準拠（同一キャラの連続フレーム）
構図: 横N列×縦M行のグリッド。各セル同サイズ、キャラの足元Y座標を全フレームで固定
サイズ・比率: <例: 1セル96x96、4列x1行=384x96>
背景有無: なし
透過: 必須
絶対に守る条件:
- セル間でキャラの位置ズレなし（アニメ時にガタつく）
- グリッド線・番号を描き込まない
- フレーム間で色数・光源を変えない
避ける表現: フレームごとの画風ブレ、モーションブラー
参考画像: <既存シート1枚を「セルサイズ・アニメパターン参照」で添付>
出力後の確認: sprite-splitter.htmlで分割して等間隔か / 再生してガタつかないか / 全フレーム透過か
```

## 4. 素材アイコン用（素材・アイテム・換金品）

```
目的: <例: 新素材「◯◯の結晶」のアイコン>
使用場所: 倉庫・図鑑・交換所の一覧（icon_XXX.webp 連番に追加）
画風: 既存アイコンに合わせたゲームアイコン調。単一オブジェクト、軽い縁取り、シンプルな陰影
構図: 対象物1つを中央に大きく。斜め俯瞰か正面。装飾背景なし
サイズ・比率: 正方形（256x256で生成→表示は縮小）
背景有無: なし
透過: 必須
絶対に守る条件:
- 1アイコン1オブジェクト（複数個を散らさない）
- 同カテゴリの既存アイコン（例: 他の結晶系）と色相で区別できること ← 同名・類似素材の混同事故が過去にあるため最重要
- 32px相当に縮小しても判別できるシルエット
避ける表現: 細かすぎるディテール、文字、キラキラエフェクト過多
参考画像: <同カテゴリ既存アイコン3〜4個を添付し「この列に並ぶ」と指定>
出力後の確認: 32px縮小で判別可か / 既存の類似素材アイコンと取り違えないか / 透過か / 連番・DB上のアイコン割当が正しいか
```

## 5. スマホUI画面イメージ用（実装前のモック）

```
目的: <例: 牧場背景ショップ画面のレイアウト検討>
使用場所: 実装しない。Codexへの指示書に添付するイメージ共有用
画風: フラットなモバイルゲームUI。Tailwind調（角丸カード、slate系ダーク基調）
構図: スマホ縦画面（375x812相当）。上部ヘッダー＋コンテンツ＋下部の構成
サイズ・比率: 9:19.5 縦長
背景有無: あり（画面全体）
透過: 不要
絶対に守る条件:
- ダミーテキストは日本語（実際の文言に近いもの）
- 既存画面のスクショを添付し、ヘッダー・配色・カード様式を合わせる
避ける表現: iOS/Android標準部品の模倣、実在ゲームUIの模倣
出力後の確認: これはモックでありデザイン正本ではないことを指示書に明記したか
```

## 6. 既存テイスト踏襲用（追加系すべての共通ラッパー）

既存シリーズ（背景・アイコン・NPC等）に「1枚追加」するときは、上のテンプレートに以下を追記する:

```
テイスト踏襲:
- 参考画像: 同シリーズの既存画像を3枚以上添付（多いほどブレない）
- 揃える要素: 線の太さ / 彩度 / 光源方向 / 等身 / 余白量
- 明示する差分: 「◯◯だけを変える（例: 色違い、季節違い）。他は参考と同一に保つ」
- 並べ確認: 生成物を既存3枚と横に並べたスクショで最終判断する（単体で見て判断しない）
```

---

## 運用メモ

- 生成→確認→webp変換→連番配置→DB/マスタのアイコン割当→図鑑/一覧で表示確認、までを1タスクにする
- アイコン割当ミス（icon番号ズレ）は図鑑で気づきにくい。割当変更時は代表画面のスクショ確認を必須にする
