# 実装指示書: 奥義マスタの効果値・説明文の全面整合（TASK_E後に実施）

> 前提: TASK_E（構造化フィールド＋エンジン対応）が完了していること。

## 目的
`database/data/job_arts.json` の memo（プレイヤー向け説明。奥義一覧画面にそのまま表示される）と実効果を全282件で一致させる。方針は「①構造化フィールドで実装できる効果は実装」「②未実装システム（ターン制・DoT・反撃・解除・命中/回避/会心操作・条件付き効果）への言及はmemoから削除し実効果に書き直す」。

## 共通ルール（全件に適用）
1. **ターン表記の全廃**: 「（3ターン）」「2ターン」等は全て削除し、継続効果は「（戦闘中）」表記に統一。`duration_turns` はjsonに残してよいが表示・実装では使わない
2. **デバフ量の標準値**（ボスは自動で半減。TASK_E実装）:
   - ATK/MAG低下: ★1=6% / ★5=10% / ★9=14%（複数ステ同時の★9は各10%）
   - DEF低下: ★1=10% / ★5=15% / ★9=20%（SPR低下はDEFの半分・切り捨て）
   - SPD低下: ★1=10% / ★5=15% / ★9=20%
3. **バフのmemo表記**: 実装は「物理職=ATK/DEF上昇、魔法職=MAG/SPR上昇（量はpower連動10/15/20%・上限1.5倍）」。memoは「自身の戦闘力を小/中/大上昇（戦闘中）」に統一（個別ステ名は書かない）
4. **多段攻撃**: 総威力は据え置き（1Hit威力=power/Hit数）。「n〜m回」表記は固定回数に変更

## 既存仕様への影響・バランス根拠
- ATK低下をDEF低下より低い%にする理由: ダメージ式が減算式（`ATK − DEF/2`）のため同%でもATK低下の実効が大きい。同格戦（敵STR100・自DEF100）でATK−6%は被ダメ約12%減となり、戦士★1「挑発撃」（自DEF+10%＝被ダメ約10%減）と釣り合う
- 多段化(hit_count)は総威力据え置きのため期待DPS不変（Hitごとに命中判定が入るぶん僅かに期待値減）
- 攻撃+回復系（B表）は「発動しなかった攻撃が発動するようになる」ため対象職の強化。回復量はheal_percentで明示し、必殺技の同系統（聖騎士6%・司祭8%等）と同水準に設定済み
