# 仕様変更時の影響範囲チェック

使い方: 仕様変更を決める**前**に、該当する変更対象の行を確認し、影響先を指示書の「既存仕様への影響」に転記する。
複数行に該当する変更（例: 探索力×課金アイテム）は両方の行を確認する。

## 影響マトリクス

### 探索（間隔・探索力・深度・危険度）
- 影響しやすい機能: スタミナ回復・小瓶/薬（課金）、素材/Gold獲得ペース、10回探索、秘境出現率、宿屋利用頻度、モンスター印収集速度、ランキング（勝利数系番付）
- 確認DB: characters（探索力カラム）、character_consumable_items、battle_logs
- 確認画面: 戦闘結果、メイン画面探索タブ、上部ステータスバー、補給商会
- 確認バランス: 1時間あたり獲得EXP/Gold/素材が変更前後でどう変わるか。課金回復アイテムの価値が上下しすぎないか
- 確認ログ: battle_logs、公開ログ（すれ違い・逸品ドロップ頻度も変わる）
- 確認告知: 回復ペース変更は必ずユーザー告知（実質弱体化になりうる）
- 将来拡張: 探索力上限拡張アイテム・イベント倍率を塞がない

### ヴァルモン（追加・効果変更）
- 影響: 卵ドロップ、図鑑、牧場（背景ショップ予定）、探索後素材発見、追撃・応急回復、称号、冒険者カード（ヴァルモンケース）、所持ヴァルモン数番付
- 確認DB: valmons系、character所持、素材発見対象マスタ
- 確認画面: 牧場、探索結果、図鑑、冒険者カード、番付掲示板
- 確認バランス: 素材発見率が素材経済（交換所・市場）を歪めないか
- 確認ログ: 探索結果ログ、公開ログ（レア卵入手を流すか）
- 確認告知: 新ヴァルモンは告知向きの目玉。ドロップ率は書かない
- 将来拡張: 背景ショップ（輝石課金・21種予定）と牧場UIの整合

### Gold（消費先追加・価格変更）
- 影響: 市場、素材交換所（G費用）、鍛冶費用、宿代、銀行、倉庫Gold拡張、総資産番付、換金品売値、救済宿泊の発生率
- 確認DB: characters.money、gold_transactions（type別）、items.sell_price、materials.npc_sale_price
- 確認画面: 銀行、市場、鍛冶屋、番付掲示板（総資産）、管理画面 inn-analytics
- 確認バランス: 都市の推奨帯での獲得Goldと照合。合成専用アイテムの再計算を忘れない（CLAUDE.md明記の見落としポイント）
- 確認ログ: gold_transactions に type とメタデータが残るか
- 確認告知: 値上げ系は「調整」として理由つきで告知
- 将来拡張: Gold→輝石変換は導入しない（恒久ルール）

### 輝石・課金（商品追加・価格変更）
- 影響: Stripe商品/価格ID、有償無償区別、補給商会、倉庫拡張、日次購入上限、売上メール、operator-analytics
- 確認DB: kiseki_transactions、stripe_orders、character_consumable_items
- 確認画面: 輝石ショップ、補給商会、管理画面売上系
- 確認バランス: 無課金プレイで詰まない導線が残るか。既購入者への補填要否（値下げ時）
- 確認ログ: 二重処理防止、購入ログ、消費内訳
- 確認告知: 価格変更は事前告知が原則。値下げは既購入者対応を先に決める
- 将来拡張: 商品IDのハードコードを増やさない（config/kiseki.php集約）

### 素材・ドロップ（追加・率変更・廃止）
- 影響: 敵ドロップテーブル、素材交換所レシピ、合成/進化素材、アイテム図鑑、ヴァルモン発見候補、市場流通、NPC調達、素材倉庫上限圧迫、素材総所持数番付
- 確認DB: enemy_drops、materials、交換レシピ、npc_material_stocks
- 確認画面: アイテム図鑑（廃止素材は非表示ルール）、素材交換所、市場、倉庫
- 確認バランス: ドロップ率目安（normal 5-10% / rare 1-3% / epic 0.3-1% / legend 0.05-0.2%）。禁止ドロップ（石・高純度石・精錬核系は敵ドロップ禁止）
- 確認ログ: ドロップ公開ログ条件（SSS/EPIC/逸品のみ）
- 確認告知: 廃止素材は所持プレイヤーへの扱い（削除/補填）を明記
- 将来拡張: 復興予定素材（WEV0023等9種）は削除しない

### 装備（追加・強化・進化・銘）
- 影響: 鍛冶費用/素材、進化レシピ、銘抽選（affix_enabled）、ステータス計算、逸品番付、装備倉庫上限、公開ログ、ドロップ率、職業装備制限
- 確認DB: items、character_items（個体カラム）、進化素材マスタ
- 確認画面: 装備画面、鍛冶屋、進化合成、図鑑
- 確認バランス: 価格帯から強さを逆算。攻撃上限を上げるなら敵HPもスケール。銘引き継ぎルール（進化時: 銘は引き継ぐ、+強化はリセット）
- 確認ログ: 逸品/SSS/EPICの公開ログ
- 確認告知: 新装備は入手方法をぼかして告知
- 将来拡張: SS→SSS等の上位進化枠を塞がない

### NPC・ランク戦（NPC追加・順位仕様変更）
- 影響: arena_npc_rankings、順位シフト、TOP10プレイヤー保護、NPC自動戦闘バッチ、公開ログ（番付上昇/TOP10入り）、通知、酒場NPC名簿、NPC市場出品
- 確認DB: arena_npc_rankings、npc_material_stocks、market_listings（seller_type=npc）
- 確認画面: 闘技場、ランキング、NPC詳細モーダル、管理画面 npc-market-analytics
- 確認バランス: NPC表示Lv上限50、legend/非戦闘NPC不出場、NPC勝利でプレイヤー売上を発生させない
- 確認ログ: 順位低下通知、公開ログの流量
- 確認告知: NPC強さの内部数値は公開しない
- 将来拡張: NPC画像は npc_id と public/images/npc/npc_XXX.webp の対応を維持

### 職業・転職（職追加・ランク仕様変更）
- 影響: 職業EXP倍率（基本1/中級2/上級5/伝説10）、Job EXP上限3/回、奥義セット（通常3+ボス3、Cost5）、転職条件、BP割振り必須ルール、ステータス計算、次やること誘導
- 確認DB: job_classes、skills（skill_type=job_art）、キャラ職業履歴
- 確認画面: 神殿、奥義画面、職業詳細モーダル
- 確認バランス: 転職後Lv1+基礎半減ルールとの整合
- 確認告知: 新職は解放条件を告知に含めるか判断
- 将来拡張: ランク10マスター上限は全職共通で維持

## 使い方の例

「探索間隔を10秒→8秒にする」なら:
1. 「探索」の行を読む → 時間あたりGold/EXP/素材が+25%になる → Gold行・素材行も確認
2. 課金回復アイテムの相対価値が下がる → 輝石行も確認
3. 指示書の「既存仕様への影響」に転記し、バランス確認欄に「1時間あたり獲得量の変化」を書く
