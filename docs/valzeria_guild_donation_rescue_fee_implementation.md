# 冒険者協会寄付・救助費軽減システム 実装指示書

## 1. 目的

冒険者協会にGoldを寄付できる仕組みを追加する。

累計寄付額に応じて「協会ランク」が上がり、戦闘敗北時の救助費が軽減される。

本システムの目的は以下。

* 余りがちなGoldの長期的な使い道を作る
* 全滅時の救助費に対する納得感を高める
* 冒険者協会という世界観を強化する
* 強さを直接買わせず、利便性・安心感としてGoldを消費させる

今回は、寄付による恩恵は **救助費軽減のみ** とする。

称号、NPC出現条件、倉庫割引、鍛冶割引などは今回の実装範囲外。

---

## 2. 基本仕様

プレイヤーは冒険者協会に任意のGoldを寄付できる。

寄付したGoldは累計寄付額として記録する。

累計寄付額に応じて協会ランクが決まる。

協会ランクが上がるほど、全滅時の救助費率と救助費上限が軽減される。

| ------ | -----------: | -----: | ---------: | -------: | -------: |
| 一般冒険者  |           0G |    10% |        15% |      20% | 200,000G |
| 銅級冒険者  |     100,000G |     9% |        14% |      19% | 180,000G |

| 白金冒険者  |  10,000,000G |     6% |        11% |      16% | 120,000G |
| 黒金冒険者  |  30,000,000G |   5.5% |        10% |      15% | 100,000G |

| 伝説級後援者 | 100,000,000G |     5% |         8% |      12% |  50,000G |

救助費率は最低5%までとする。

救助費0%にはしない。

---

## 4. 救助費計算仕様

### 4.1 基本計算

戦闘敗北時、手持ちGoldから救助費を差し引く。

銀行預金がある場合、銀行預金は救助費の対象外とする。

```text
救助費 = floor(手持ちGold × 救助費率)
救助費 = max(100, 救助費)
救助費 = min(救助費上限, 救助費)
救助費 = min(手持ちGold, 救助費)
```

所持Goldが100G未満の場合は、所持Gold全額までしか徴収しない。

---

### 4.2 危険度による救助費率

危険度によって適用する救助費率を変える。

|    危険度 | 適用救助費率          |
| -----: | --------------- |
|  0〜74% | 通常救助費率          |
| 75〜99% | 危険度75〜99%時の救助費率 |
|   100% | 危険度100%時の救助費率   |

例：

プレイヤーの協会ランクが金級冒険者の場合。

|    危険度 | 救助費率 |
| -----: | ---: |
|  0〜74% |   7% |
| 75〜99% |  12% |
|   100% |  17% |

## 5. 敗北時処理フロー

```text
1. 戦闘でプレイヤーが敗北する
2. 現在の協会ランクを取得する
3. 現在の危険度を取得する
4. 危険度に応じた救助費率を決定する
5. 救助費を計算する
6. プレイヤーの手持ちGoldから救助費を差し引く
7. 探索度・連戦数・危険度をリセットする
8. プレイヤーを街へ戻す
9. 敗北メッセージを表示する
```

敗北時は、既存仕様どおり探索度・連戦数・危険度を必ずリセットする。


## 11. DB追加案

### 11.1 players テーブルに追加する場合

既存のプレイヤーテーブルに寄付額を持たせる場合。

```sql
ALTER TABLE players
ADD COLUMN guild_donation_total BIGINT NOT NULL DEFAULT 0 AFTER gold;
```

協会ランクは累計寄付額から毎回計算するため、DBに保存しなくてもよい。

ただし、表示や集計を簡単にしたい場合は `guild_rank` を保存してもよい。

```sql
ALTER TABLE players
ADD COLUMN guild_rank VARCHAR(30) NOT NULL DEFAULT '一般冒険者' AFTER guild_donation_total;
```

推奨は以下。

```text
guild_donation_total は保存する。
guild_rank は保存せず、表示時・救助費計算時に累計寄付額から判定する。
```

理由は、ランク条件を後から変更したときに整合性を保ちやすいため。

---

### 11.2 donation_logs テーブル

寄付履歴を残す場合。

```sql
CREATE TABLE guild_donation_logs (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  player_id BIGINT NOT NULL,

  donation_total_after BIGINT NOT NULL,

  INDEX idx_player_id (player_id),
  INDEX idx_created_at (created_at)

```

Phase 1では必須ではないが、不具合調査や不正対策のため、可能なら作成する。

## 13. 救助費計算関数

擬似コード：

```php
function calculateRescueFee(
    int $gold,
    int $dangerRate,
    int $guildDonationTotal
): array {
    $guildRank = getGuildRankByDonation($guildDonationTotal);

    if ($dangerRate >= 100) {
        $rate = $guildRank['danger_rate_100'];
    } elseif ($dangerRate >= 75) {
        $rate = $guildRank['danger_rate_75'];
    } else {
        $rate = $guildRank['normal_rate'];
    }

    $fee = (int) floor($gold * $rate);
    $fee = max(100, $fee);
    $fee = min($guildRank['fee_cap'], $fee);
    $fee = min($gold, $fee);

    return [
        'fee' => $fee,
        'rate' => $rate,
        'rank_name' => $guildRank['rank_name'],
        'fee_cap' => $guildRank['fee_cap'],
    ];
}
```

## 14. 注意点

* 救助費は手持ちGoldからのみ徴収する。
* 銀行預金がある場合、銀行預金は救助費対象外にする。
* 救助費率は最低5%までにする。
* 救助費0%にはしない。
* 寄付でATK、DEF、MAGなどの戦闘ステータスは上げない。
* 寄付で秘境素材、地図片、レア装備は配布しない。
* 寄付額は累計で管理する。
* 寄付の取り消し・返金は不可。
* 救助費計算前に協会ランクを必ず取得する。
* ランク条件は定数またはマスタ化しておくと、後から調整しやすい。
