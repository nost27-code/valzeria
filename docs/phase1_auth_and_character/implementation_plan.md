# 実装計画: フェーズ1 認証・キャラクター作成

## 目標
`docs/base.md` の「22.1 フェーズ1：認証・キャラ作成」に基づき、初期のログイン機能とキャラクター作成機能、およびホーム画面の基盤を実装します。

## ⚠️ User Review Required (要確認事項)
段階的実装のルールに基づき、まずはこのフェーズ1に絞って作業を進めます。以下の「Open Questions」についてご回答をお願いいたします。

## ❓ Open Questions

2. **ホーム画面のUIデザイン（ガチャガチャ感）について**
   先ほどご相談いただいた「装備枠」「他プレイヤー一覧」「チャットエリア」などを詰め込んだ「FFAらしい情報量の多いUI（ガチャガチャ感）」は、今回のフェーズ1におけるホーム画面（`/home`）作成時に取り入れてデザインしてよろしいでしょうか？

## 提案される変更

### 1. DBマイグレーションとSeeder
- **`users` テーブル**: Google認証情報 (`google_id`, `avatar_url`, `role` 等) を追加。
- **`characters` テーブル**: 仕様書に基づき、`gender`, `icon_path`, 基礎ステータス (`hp_base`, `attack_base` 等)、`wins`, `total_score` などのカラムを追加（先ほど作成したマイグレーションを改修）。
- **`jobs` テーブル**: 職業マスターを作成。
- **Seeder**: 初期職業（戦士、魔法使い、僧侶、盗賊）の基礎データを投入する `JobSeeder` を作成。

### 2. ルーティングとコントローラー
- `routes/web.php` を更新。
- トップ画面（`/`）: ログイン案内画面
- 認証ルート（`/auth/google`, `/auth/google/callback`）: Socialiteを用いたログイン処理、またはモックログイン処理
- キャラ作成（`/character/create`）: Livewireコンポーネント
- ホーム画面（`/home`）: Livewireコンポーネント

### 3. サービス層 (Service Layer)
仕様書「17.1 主要サービス」に従い、以下のサービスクラスを作成。
- `AuthService`: ログイン後のユーザー作成・取得ロジック
- `CharacterService`: キャラクター初期作成、基礎ステータス割り当てロジック

### 4. フロントエンド (Livewire / Tailwind)
- 先ほど作成した `MainScreen` コンポーネントを `Home` コンポーネントとしてリネーム・改修し、指定されたガチャガチャ感のあるUIへ再設計します。
- キャラクター作成画面（`CharacterCreate`）の実装（職業選択、名前入力など）。

## 検証プラン
### 手動検証
- ローカルサーバー（`php artisan serve`）を起動。
- トップ画面からログインが成功することを確認。
- キャラクター未作成の場合、作成画面へ遷移し、正しくデータが保存されることを確認。
- 作成完了後、ホーム画面に遷移し、自身のキャラクター情報が正しく表示されることを確認。
