# 街機能 実装指示書に基づく実装計画

## 1. 概要
ご提示いただいた指示書に従い、プレイヤーの拠点となる「街（City）」システムを実装します。街ごとに探索可能なダンジョン（エリア）を切り替え、進行状況に応じて新たな街や職業を解放していく仕組みを構築します。

## 2. データベース設計の対応
現在のプロジェクトは、ダンジョンを `areas` テーブル、プレイヤー情報を `characters` テーブルで管理しています。この構成に合わせて指示書の設計を適応させます。

### 2.1 新規テーブル: `cities`
| カラム | 型 | 内容 |
|---|---|---|
| id | bigInt | 街ID |
| name | string | 街名（王都アークレア 等） |
| description | text | 街説明 |
| recommended_level_min | integer | 推奨Lv下限 |
| recommended_level_max | integer | 推奨Lv上限 |
| sort_order | integer | 表示順 |
| unlock_condition_type | string | 解放条件種別（例: `area_cleared`） |
| unlock_condition_value | string | 解放条件値（例: クリア必須のエリアID） |
| is_initial | boolean | 初期街フラグ |

### 2.2 既存テーブルの改修
**`characters` テーブル**
* `current_city_id` (unsignedBigInteger): 現在地の街ID
* `highest_city_id` (unsignedBigInteger): 到達済みの最大街ID

**`areas` テーブル**（※指示書の `dungeons` に該当）
* `city_id` (unsignedBigInteger): 所属する街ID

## 3. 各フェーズの実装内容

### Phase 1: 基礎システムの構築
1. **マイグレーションの作成**: `cities` の作成、`characters` と `areas` のカラム追加。
2. **モデルとリレーション定義**: `City`, `Character`, `Area` 間のリレーションを設定。
3. **Seederの作成**: 初期街「王都アークレア」や「港町マリネス」などのデータを投入。既存のエリア（見習い訓練場など）を王都アークレアに紐付け。
4. **探索画面 (Explore) の改修**:
   * 現在の街（`character->current_city_id`）に紐付くエリアのみを一覧表示するようクエリを変更。
5. **街移動画面 (City Travel) の作成**:
   * 現在解放済みの街（`highest_city_id` 以下の街）を一覧表示。
   * 「移動する」ボタンで `current_city_id` を更新（スタミナ消費・費用ゼロ）。

### Phase 2: 進行と解放の連動
1. **10連戦の最深部ボス撃破によるクリア判定**:
   * 現在の戦闘ロジック（`BattleService` または `ExploreController`）において、エリアの最終戦にボスを出現させ、勝利時にエリアクリアフラグ（`character_area_progresses` など）を立てる。
2. **次街の解放ロジック**:
   * 戦闘勝利時の処理で、クリアしたエリアが「現在の街の最終エリア」であり、かつ「次の街の解放条件」を満たした場合、`highest_city_id` を更新して次の街を解放する。
3. **職業解放との連携**:
   * 現在地の街ID（または `highest_city_id`）を条件として、転職所で表示・選択可能な職業をフィルタリングするロジックを追加。

### Phase 3: UIと演出の強化
1. **ナビゲーションとヘッダーの改修**:
   * 常に現在地の街名を表示。
   * 街の施設メニュー（武器屋、防具屋、宿屋など）へのリンクは全街で共通とする。
2. **世界地図の表示**:
   * 探索画面等に世界地図の画像を表示し、進行状況に応じた雰囲気演出を追加（未解放街のグレーアウト等はCSSで制御）。

## 4. ユーザーへの確認事項（オープンクエスチョン）

> 2. 初期実装の Seeder データとして、「王都アークレア」「港町マリネス」「精霊都市エルフィア」「鉄鋼都市グランベルグ」の4つを用意し、既存のエリアを王都アークレアに割り当てる形で進めてよろしいでしょうか？

承認いただき次第、Phase 1 のマイグレーション・モデル・Seeder作成および探索画面のUI切り替え実装に着手します。
