## トラブルと解決策の記録（教訓）

### 1. 発生したバグの事象と原因
以前のエージェントが職業データのTSVインポートを行う際、TSVのパース時に列の読み込みインデックスが1つズレてしまいました。その結果、本来の職業体系が崩れてしまっただけでなく、基本職の `hp_rate` カラムがすべて `0` になり、キャラクターがレベルアップしてもHPが一切成長しないという致命的なバグが埋め込まれていました。

### 2. 解決策
- オリジナルの Seeder 定義と、退避されていたボーナス定義データをマージする安全な PHP スクリプトを作成・実行し、正しい TSV ファイルを復元しました。
- シーダー側の処理を、固定インデックスではなくヘッダー対応の `array_combine` を使った動的マッピング方式に変更し、パース時の列ズレが起きないようにしました。

### 3. 今後の教訓・注意点
- **TSV/CSV取り込み時は動的マッピングを徹底**: 列番号を固定値（`$row[5]` など）で指定すると、フォーマットの微細な変化でデータがズレてバグを引き起こすため、必ずヘッダーを読み取ってカラム名でマッピングすること。
- **インポート後は必ず確認クエリを実行**: シード処理やインポート処理を行った後は、必ず Tinker や SQL で代表的なレコードを抽出し、特定のステータス（成長倍率など）が意図通りに登録されているか（`0` や `NULL` になっていないか）の確認テストを徹底すること。

### 4. [2026-06-10] マスタ最終更新時のデグレ対策
- **事象**: `JobSystemSeeder` 自体は正常に修正されていたものの、過去のバグ補正用スクリプト `fix_basic_jobs.php` が残っており、本番アップデートAPIの実行時にこの古いTSV仮データが自動実行されて基本職データが上書きされてしまう危険性がありました。
- **解決策**: `fix_basic_jobs.php` の中身を無効化し、空のメッセージ出力のみに書き換えることで、安全に最新マスタデータが維持されるように修正しました。
