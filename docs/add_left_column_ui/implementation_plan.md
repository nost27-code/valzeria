# 左カラム（プレイヤーステータス・メニュー）追加とレイアウト改修

ユーザーから要望された「左側にカラムを追加し、プレイヤーの基礎情報やステータス、装備が記載された枠を追加する」対応、および提示されたデザイン画像に基づき、実装計画を提案します。

## 背景と目的
現在の中央表示のレイアウトから、仕様書（`valzeria_ui_lively_ffa_additional_spec.md`）にある「3カラム構成」へと移行し、左側に常に自分のステータスが見えるFFA（CGIゲーム）らしいUIを構築します。

## 提案する変更内容

### 1. 共通レイアウト（`app.blade.php`）の改修
- 全体を囲むコンテナを導入し、PC画面では「左カラム（ステータス・メニュー）」と「右カラム（メインコンテンツ＋他プレイヤー等）」が横並びになるレイアウトへ変更します。
- スマホ（モバイル）表示時は、レスポンシブ対応を組み込みます。

### 2. 左カラム用コンポーネントの新規作成
`resources/views/livewire/left-sidebar.blade.php` などのコンポーネントを新設し、以下の要素を配置します。

#### ステータスパネル（添付画像の再現）
- **装飾**: 四隅にゴールドのフレーム（L字）を配置し、カード全体に金色の枠線を設けます。
- **基本情報**: アバター画像（円形）、キャラクター名、Lv、職業、称号バッジ。
- **詳細情報**: PvPランク、転生回数、所属ギルド。
- **ゲージ**: HPバー（赤）、EXPバー（青）。
- **ステータス**: STR, DEF, AGI, MAG, LUK、GOLD。小さなアイコンを添えて表示します。
- **現在の装備**: 武器（武）、防具・盾（盾）、装飾品（飾）の一覧。未装備の場合は「なし」と表示します。

#### ナビゲーションメニュー
ステータスパネルの下部に、以下のメニューリンクを追加します。
- 宿場町（ホーム）
- 迷宮へ向かう（現在実装中の探索）
- 闘技場（準備中）
- 武具屋・防具屋・装飾屋へのリンク
- 転職所
- その他

### 3. 表示データの取得ロジックの追加
- Livewireコンポーネントを使用し、認証中ユーザーのキャラクター情報（HP、EXP、装備状況など）を自動で取得・表示するようにします。

## ユーザーへの確認事項（Open Questions）

> [!IMPORTANT]
> 1. **スマホでの表示順について**
> スマホの縦長画面で見た場合、「ステータスパネル」と「メイン画面（ショップや戦闘画面）」のどちらを**上**に表示すべきでしょうか？
> FFA風ゲームだとステータスが一番上にあることが多いですが、毎回スクロールが必要になるため、「ステータスは一番下にする」あるいは「ボタンで開閉できるサイドバーにする」などのアプローチもあります。
>
> 2. **アバター画像について**
> 現在システム内にキャラクターのアバター画像を用意していませんが、添付いただいた画像のようなデフォルトのキャラクター画像（ダミー画像）をひとまず当てはめておいてよろしいでしょうか？

## 検証計画
- PC画面で左にステータス、右にショップなどの画面が崩れず表示されることを確認する。
- 添付画像のデザインテイスト（ゴールドの枠、HPバー等）が再現されているか確認する。
- スマホ画面に切り替えてもレイアウトが破綻しないことを確認する。
