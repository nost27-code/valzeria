# 確定成長方式（小数点累積）の実装完了報告

計画通り、レベルアップ時のステータス成長ロジックを「乱数によるブレがある方式」から「端数をストックする完全な確定成長方式」へと改修いたしました！

## 変更内容のサマリー

1. **データベースのカラム追加**
   - キャラクターのテーブル（`characters`）に、各ステータスの小数点以下の端数を保存する `_fraction` カラム（`hp_fraction`, `attack_fraction` 等の7つ）を追加しました。
   - `Character.php` モデルにて、これらのカラムを正しく `float` 型として扱うよう設定しました。

2. **レベルアップ処理の改修 (`LevelService.php`)**
   - 以前の `rand()` を用いた計算を廃止しました。
   - レベルアップごとに「期待値（固定値） × 職業倍率 × 転生ボーナス」の確定値を算出し、端数（小数点以下）をストックするようにしました。
   - ストックされた端数が `1.0` を超えると、自動的にステータスが `+1` される仕組みになっています。
   - （例：毎回4.7ずつ成長する職業の場合、「4」ずつ基礎ステータスが上がりつつ、内部で端数が「0.7 → 1.4 → 2.1...」と貯まり、2回目のレベルアップ時には+5上がるような美しい成長曲線を描きます）

3. **転職時のリセット処理 (`CharacterJobChangeService.php`)**
   - 転職を行った際は、新しく生まれ変わる（基礎値が50%になる）という仕様に基づき、前職で貯まっていた端数ストックは全て `0.0` にリセットされるようにしました。

## 期待される効果
この改修により、「職業の倍率が高ければ高いほど、一切のブレなく確実に強くなる」という実力重視のシステムが完成しました。エクセル等でシミュレーションした結果が、ゲーム内でも誤差なく完全に一致するようになります！
