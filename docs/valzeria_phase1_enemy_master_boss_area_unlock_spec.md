# 冒険都市ヴァルゼリア フェーズ1 敵マスター・ボス戦・エリア解放 実装仕様書

## ファイル名

valzeria_phase1_enemy_master_boss_area_unlock_spec.md

---

## 1. 目的

本仕様書は、冒険都市ヴァルゼリアのフェーズ1実装において、現在スライム固定になっている戦闘処理を拡張し、エリアごとに複数の敵が出現する仕組み、ボス戦、ボス撃破によるエリア解放、次の目標更新、公開ログ反映を実装するための仕様を定義する。

現在の状態では、迷宮探索と戦闘処理は実装済みである。
ただし、敵がスライム固定であるため、冒険している感覚やエリアごとの差が弱い。

本仕様では、以下のゲームループを成立させる。

```text
エリアを選ぶ
↓
そのエリアに応じた敵が出現する
↓
通常戦闘でレベルを上げる
↓
エリアボスに挑戦する
↓
ボスを倒す
↓
次のエリアが解放される
↓
左カラムの次の目標が更新される
↓
街ログに撃破・到達ログが流れる
```

この仕様により、単なる戦闘機能から、エリアを進めていく冒険RPGとしての体験に進化させる。

## 3. 現在の課題

## 3.1 敵が固定で単調

現状はスライムのみが出現しているため、探索するエリアを変えても体験差が出にくい。

必要な改善は以下。

* エリアごとに敵を分ける
* 敵ごとに能力値を変える
* 敵ごとにEXPとGOLDを変える
* 出現しやすい敵、出現しにくい敵を設定する
* ボス敵を通常敵と分ける

## 3.2 エリアを進める理由が弱い

ボス戦やエリア解放がないと、プレイヤーは「なぜ次のエリアを目指すのか」が分かりにくい。

必要な改善は以下。

* 各エリアにボスを設定する
* ボス撃破で次エリアを解放する
* 未解放エリアに解放条件を表示する
* 左カラムの次の目標に、今やるべきことを表示する

## 5. 敵マスター仕様

## 5.1 敵マスターの役割

敵マスターは、通常探索とボス戦で使用する敵データを管理する。

敵はエリアごとに紐づける。
同じ敵名を別エリアに出す場合でも、エリア別に別レコードとして登録してよい。

## 5.2 enemies テーブル

テーブル名。

```text
enemies
```

カラム案。

| カラム               | 型         | 内容         |
| ----------------- | --------- | ---------- |
| id                | bigint    | 主キー        |
| area_id           | bigint    | 出現エリアID    |
| name              | string    | 敵名         |
| description       | text      | 敵説明        |
| level             | int       | 敵レベル       |
| max_hp            | int       | 最大HP       |
| str               | int       | 攻撃         |
| def               | int       | 防御         |
| agi               | int       | 素早さ        |
| mag               | int       | 魔力         |
| luk               | int       | 運          |
| exp_reward        | int       | 勝利時EXP     |
| gold_reward       | int       | 勝利時GOLD    |
| appearance_weight | int       | 通常探索時の出現重み |
| is_boss           | boolean   | ボスか        |
| is_active         | boolean   | 出現対象か      |
| sort_order        | int       | 表示順        |
| created_at        | timestamp | 作成日時       |
| updated_at        | timestamp | 更新日時       |

## 5.3 appearance_weight の扱い

通常敵は `appearance_weight` によって抽選する。

例。

| 敵名     | 出現重み |
| ------ | ---: |
| スライム   |   50 |
| 迷いウサギ  |   35 |
| 草原コウモリ |   15 |

この場合、合計100のうち、スライムが最も出やすい。

ボス敵は通常探索では抽選しない。
そのため、`is_boss = true` の敵は通常探索の敵抽選から除外する。

## 5.4 is_active の扱い

`is_active = false` の敵は出現しない。

管理画面やSeederで敵を登録しておき、後から有効化できるようにするためのカラムである。

## 7. キャラクター別エリア進行管理

## 7.1 進行管理の目的

エリアの解放状態やボス撃破状態は、全体共通ではなくキャラクターごとに管理する。

理由は、プレイヤーごとに進行状況が異なるためである。

## 7.2 character_area_progress テーブル

テーブル名。

```text
character_area_progress
```

カラム案。

| カラム              | 型         | 内容       |
| ---------------- | --------- | -------- |
| id               | bigint    | 主キー      |
| character_id     | bigint    | キャラクターID |
| area_id          | bigint    | エリアID    |
| is_unlocked      | boolean   | 解放済みか    |
| boss_defeated    | boolean   | ボス撃破済みか  |
| unlocked_at      | timestamp | 解放日時     |
| boss_defeated_at | timestamp | ボス撃破日時   |
| created_at       | timestamp | 作成日時     |
| updated_at       | timestamp | 更新日時     |

## 7.3 初期作成時の進行データ

キャラクター作成時に、全エリア分の進行データを作成する。

初期状態は以下。

| エリア     | is_unlocked | boss_defeated |
| ------- | ----------- | ------------- |
| はじまりの草原 | true        | false         |
| 小鬼の森    | false       | false         |
| 古びた洞窟   | false       | false         |

開発環境で小鬼の森を最初から触らせる場合は、Seederまたは環境設定で小鬼の森のみ `is_unlocked = true` にしてよい。

## 7.4 進行データが存在しない場合

何らかの理由で `character_area_progress` が存在しない場合は、AreaService側で補完作成する。

これにより、既存キャラクターにも後からエリア進行を追加できる。

## 8. 通常探索の敵抽選仕様

## 8.1 通常探索の対象敵

通常探索では、以下の条件を満たす敵を抽選対象とする。

```text
area_id = 選択エリアID
is_boss = false
is_active = true
```

## 8.2 抽選方式

`appearance_weight` を使って重み付き抽選を行う。

処理イメージ。

```text
1. 対象エリアの通常敵を取得
2. appearance_weight の合計値を計算
3. 1〜合計値の乱数を生成
4. 累積重みで該当敵を決定
```

## 8.3 抽選例

はじまりの草原。

| 敵名     | 重み | 出現率目安 |
| ------ | -: | ----: |
| スライム   | 50 |   50% |
| 迷いウサギ  | 35 |   35% |
| 草原コウモリ | 15 |   15% |

## 8.4 敵データがない場合

対象エリアに出現可能な敵が1体も存在しない場合は、探索処理を中断し、以下のエラーを表示する。

```text
このエリアにはまだ敵が設定されていません。
```

## 9. ボス戦仕様

## 9.1 ボス敵の取得条件

ボス戦では、以下の条件を満たす敵を取得する。

```text
area_id = 選択エリアID
is_boss = true
is_active = true
```

ボスは原則として、1エリアにつき1体とする。

## 9.2 ボス挑戦条件

ボスに挑戦できる条件は以下。

* ログイン済みである
* キャラクター作成済みである
* 対象エリアが解放済みである
* 対象エリアのボスが未撃破である
* ボス敵データが存在する
* HPが1以上である
* 戦闘クールタイム中ではない

## 9.3 ボス撃破済みの場合

すでにボス撃破済みの場合、ボスに再挑戦できない仕様とする。

表示例。

```text
このエリアのボスはすでに撃破済みです。
```

将来的に再戦を許可する場合は、報酬なしまたは報酬減少とする。
フェーズ1では再戦不可でよい。

## 9.4 ボス戦のクールタイム

ボス戦後は、通常探索より長いクールタイムを設定する。

| 戦闘種別 | クールタイム |
| ---- | -----: |
| 通常探索 |     5秒 |
| ボス戦  |    15秒 |

## 9.5 ボス戦結果表示

ボス戦後は、通常戦闘結果に加えて以下を表示する。

* ボス撃破有無
* 次エリア解放有無
* 解放されたエリア名
* 公開ログ反映有無

表示例。

```text
ボス撃破！

小鬼の森の親分を倒しました。
古びた洞窟が解放されました！

EXP +120
GOLD +80

[古びた洞窟へ向かう]
[迷宮へ戻る]
[街へ戻る]
```

## 10. エリア解放仕様

## 10.1 基本方針

エリアは、前エリアのボス撃破によって順番に解放される。

```text
はじまりの草原
↓
草原の大スライム撃破
↓
小鬼の森 解放
↓
小鬼の森の親分撃破
↓
古びた洞窟 解放
```

## 10.2 解放対象の判定

`areas.unlock_required_area_id` を参照する。

例。

| エリア     | unlock_required_area_id |
| ------- | ----------------------: |
| はじまりの草原 |                    null |
| 小鬼の森    |              はじまりの草原のID |
| 古びた洞窟   |                 小鬼の森のID |

ボス撃破時、以下の条件に合うエリアを検索する。

```text
unlock_required_area_id = 撃破したエリアID
```

該当するエリアを、キャラクターの進行データ上で解放する。

## 10.3 解放処理

ボス撃破後の処理。

```text
1. 現在エリアの boss_defeated を true にする
2. boss_defeated_at に現在時刻を保存
3. 次エリアを検索
4. 次エリアの is_unlocked を true にする
5. unlocked_at に現在時刻を保存
6. 公開ログを作成
7. 戦闘結果画面に解放情報を表示
```

## 10.4 すでに解放済みの場合

次エリアがすでに解放済みの場合、重複処理しない。

公開ログも重複して流さない。

## 10.5 最終エリアの場合

次エリアが存在しない場合は、エリア解放処理は行わない。

表示例。

```text
古びた洞窟のボスを撃破しました！
現在実装されているエリアはここまでです。
```

## 11. 迷宮カードのUI更新仕様

## 11.1 解放済みカード

解放済みエリアは通常表示する。

表示内容。

* エリア名
* 説明
* 推奨Lv
* ボス状態
* 探索するボタン
* ボスに挑むボタン

## 11.2 ボス撃破済みカード

ボス撃破済みの場合は、以下のように表示する。

```text
ボス：撃破済み
```

ボスに挑むボタンは非表示、または disabled とする。

推奨は非表示。

理由は、プレイヤーに再挑戦できると誤解させないため。

## 11.3 未解放カード

未解放エリアはカード自体を表示する。
ただし、背景画像や文字色を薄くし、ボタンは未解放表示にする。

表示例。

```text
古びた洞窟
奥深くへと続く危険な洞窟
推奨Lv：10〜
解放条件：小鬼の森クリア

[未解放]
```

## 11.4 未解放カードを表示する理由

未解放エリアを隠さず表示する理由は以下。

* 次の目標が分かる
* 冒険が続く感じが出る
* プレイヤーのモチベーションになる
* FFAらしい「まだ行けない場所がある」感が出る

## 12. 次の目標更新仕様

## 12.1 左カラムの次の目標

左カラムには、プレイヤーの進行状況に応じた「次の目標」を表示する。

現在の例。

```text
エリアのボスを倒して、次の迷宮を解放しよう
```

この文言を進行状態に応じて更新する。

## 12.2 目標判定ロジック

優先順位は以下。

1. 未撃破ボスがいる解放済みエリアがある
2. 次の未解放エリアがある
3. 現在実装済みエリアをすべてクリア済み
4. HPが少ない
5. レベルが低い

## 12.3 目標文言例

### はじまりの草原ボス未撃破


## 15. サービス設計

## 15.1 作成・拡張するサービス

今回の仕様では以下のサービスを作成または拡張する。

| サービス                 | 役割            |
| -------------------- | ------------- |
| EnemyService         | 敵取得、敵抽選       |
| AreaService          | エリア解放、進行状態取得  |
| BossService          | ボス挑戦条件、ボス撃破処理 |
| ExplorationService   | 通常探索処理        |
| BattleService        | 自動戦闘処理        |
| PublicLogService     | 公開ログ作成        |
| CharacterGoalService | 次の目標文言生成      |

## 15.2 EnemyService

責務。

* エリアの通常敵一覧を取得
* 出現重みに基づいて敵を抽選
* エリアのボス敵を取得
* 敵データ未設定時の例外処理

メソッド例。

```text
getNormalEnemiesByArea(areaId)
pickRandomEnemy(areaId)
getBossByArea(areaId)
```

## 15.3 AreaService

責務。

* キャラクターのエリア進行状態取得
* エリアが解放済みか判定
* ボス撃破状態判定
* 次エリア解放
* 進行データ補完

メソッド例。

```text
getAreaProgress(character)
isAreaUnlocked(character, area)
isBossDefeated(character, area)
unlockNextArea(character, clearedArea)
initializeProgressIfMissing(character)
```

## 15.4 BossService

責務。

* ボス挑戦条件チェック
* ボス戦実行前のバリデーション
* ボス撃破時の進行更新
* 次エリア解放
* 公開ログ作成

メソッド例。

```text
canChallengeBoss(character, area)
handleBossVictory(character, area, enemy)
```

## 15.5 CharacterGoalService

責務。

* 左カラムの次の目標文言を返す

メソッド例。

```text
getNextGoal(character)
```

## 22. バランス調整方針

## 22.1 序盤の難易度

はじまりの草原は、Lv1でも勝てる敵を中心にする。

ただし、HPが減ったまま連戦すると負ける可能性は残す。
これにより、宿屋の意味が生まれる。

## 22.2 小鬼の森の役割

小鬼の森は、Lv1では厳しく、Lv4〜5程度で安定する程度にする。

ここで、レベル上げとエリア選択の意味を持たせる。

## 22.3 古びた洞窟の役割

古びた洞窟は、フェーズ1時点での最初の難所とする。

Lv10未満では厳しく、ある程度育成してから挑む想定。

## 22.4 ボス難易度

ボスは、推奨Lvの上限付近で勝てる程度にする。

例。

| ボス       |  推奨勝利Lv |
| -------- | ------: |
| 草原の大スライム |   Lv4〜5 |
| 小鬼の森の親分  |  Lv8〜10 |
| 洞窟の番人    | Lv13〜15 |

## 24. 結論

今回の実装では、スライム固定の戦闘から脱却し、エリアごとに異なる敵が出現する仕組みを作る。

さらに、各エリアにボスを設定し、ボス撃破によって次のエリアが解放されるようにする。

これにより、冒険都市ヴァルゼリアの基本体験は以下に変わる。

```text
ただ戦闘するゲーム
↓
エリアを進めていく冒険RPG
```

フェーズ1において最も重要なのは、プレイヤーに「次はあそこを突破したい」と思わせることである。

そのために、敵マスター、ボス戦、エリア解放、次の目標、公開ログをセットで実装する。
