# 酒場NPC出現・冒険者名簿システム 実装指示書

## 1. 目的

酒場にランダムでNPC冒険者が出現する仕組みを追加する。

NPCは単なる会話相手ではなく、以下の役割を持たせる。

* 世界観の補強
* 冒険者が他にも存在する感覚の演出
* 探索、職業、秘境、称号、装備、ボス攻略のヒント導線
* NPC図鑑による収集要素
* 将来的な職業解放、秘境解放、称号条件への接続

本システムにより、酒場を「雰囲気施設」ではなく、冒険情報とNPC収集の中心施設として機能させる。

本作は、戦闘・装備・職業育成・ボス攻略・称号収集が中核のFFA風ブラウザRPGであるため、酒場NPCもそれらの長期導線に接続する。

---

## 2. 基本仕様

酒場を開いたとき、条件を満たしたNPCの中からランダムで最大3人を表示する。

表示例：

```text
酒場には、今日も冒険者たちが集まっている。

今日は 3人 の冒険者がいるようだ。

・疾風のコジロウ
・見習い僧侶マノン
・片目のレン
```

各NPCを選択すると、会話テキストを表示する。

初めて会話したNPCは「冒険者名簿」に登録される。

* NPCイベント連鎖

* NPC共闘バフ

## 4. NPC区分

NPCは以下の4区分で管理する。

| 区分 | 説明                | 想定人数 | 出現しやすさ |
| -- | ----------------- | ---: | -----: |
| 一般 | 序盤から出る冒険者         |  20人 |     高い |
| 熟練 | 中盤以降に出る冒険者        |  20人 |     普通 |
| 英雄 | 職業・都市制覇に絡む有名冒険者   |  20人 |     低い |
| 伝説 | エンドコンテンツ・世界観核心NPC |  10人 |     極低 |

初期実装では、70人全員をマスタに登録してよい。

ただし、序盤から全員が出現しないように、出現条件で制御する。

---

## 5. 出現条件

NPCごとに出現条件を持たせる。

条件例：

* 探索回数
* 酒場訪問回数
* プレイヤーレベル
* 到達都市
* 特定ダンジョンクリア
* 特定都市制覇
* 職業ランク
* 職業マスター
* 称号獲得数
* 秘境発見数
* レアドロップ獲得数
* 敗北回数
* 闘技場勝利数
* 魔王撃破済み

Phase 1では、複雑なAND条件を無理に実装せず、条件タイプ1つ＋条件値1つでよい。

例：

```text
appear_condition_type = total_exploration_count
appear_condition_value = 100
```

## 6. 酒場での抽選仕様

### 6.1 抽選タイミング

酒場画面を開いたときにNPCを抽選する。

ただし、画面更新のたびにNPCが変わると不自然なので、1日単位で固定する。

推奨仕様：

```text
プレイヤーごとに、日付単位で酒場NPCを固定する。
```

例：

```text
player_id + yyyy-mm-dd
```

をキーにして、その日の出現NPCを保存する。

---

### 6.2 表示人数

酒場に表示するNPCは最大3人。

候補NPCが3人未満の場合は、条件を満たす人数だけ表示する。

例：

```text
0人：今日は知った顔はいないようだ。
1人：今日は 1人 の冒険者がいるようだ。
2人：今日は 2人 の冒険者がいるようだ。
3人：今日は 3人 の冒険者がいるようだ。
```

---

### 6.3 抽選対象

以下を満たすNPCを抽選候補にする。

```text
npc.is_active = 1
かつ
プレイヤーが出現条件を満たしている
```

遭遇済みNPCも再出現してよい。

ただし、未遭遇NPCを少し優先すると図鑑が埋まりやすくなる。

---

### 6.4 抽選重み

NPC区分ごとに基本出現重みを持たせる。

| 区分 | 基本重み |
| -- | ---: |
| 一般 |  100 |
| 熟練 |   60 |
| 英雄 |   25 |
| 伝説 |    8 |

さらに未遭遇NPCは重みを1.5倍にする。

```text
遭遇済み：weight × 1.0
未遭遇：weight × 1.5
```

これにより、図鑑収集が進みやすくなる。

---

### 6.5 同一日の再抽選

同じ日に酒場を開いた場合、同じNPCを表示する。

再抽選は翌日以降。

ただし、将来的に「酒場チケット」などで再抽選可能にする余地は残す。

## 7. 会話仕様

NPCをタップすると会話画面を表示する。

表示例：

```text
疾風のコジロウ

「おっ、あんたも冒険者か？
小鬼の森では、奥へ進みすぎると妙な気配が濃くなるらしいぜ。
引き際も冒険者の腕ってやつだな。」

【冒険者名簿に登録されました】
```

2回目以降は登録メッセージを表示しない。

## 8. 冒険者名簿

酒場またはメニューから「冒険者名簿」を開けるようにする。

表示例：

```text
冒険者名簿

登録数：12 / 70

001 疾風のコジロウ
002 豪腕のムサシ
003 放浪のヤマト
004 ？？？
005 草原帰りのニコ
```

未遭遇NPCは「？？？」表示。

遭遇済みNPCをタップすると、詳細を表示する。

## 9. NPC詳細表示

遭遇済みNPCの詳細では、以下を表示する。

* NPC名
* 区分
* 簡単な説明
* 初遭遇日
* 関係性
* 会話文
* ヒント内容

表示例：

```text
疾風のコジロウ

区分：一般冒険者

説明：
身軽さを売りにする若い冒険者。
まだ実力は発展途上だが、探索の勘は鋭い。

関係性：
豪腕のムサシを一方的にライバル視している。

ヒント：
連戦を続けるほど、珍しいものを見つけやすくなる。
ただし危険も増していく。
```

未遭遇NPCの詳細は表示しない。

## 10. DB設計

### 10.1 npc_master

NPCの基本情報を管理する。

```sql
CREATE TABLE npc_master (
  npc_id INT PRIMARY KEY,
  npc_name VARCHAR(100) NOT NULL,
  npc_rank VARCHAR(20) NOT NULL,
  npc_title VARCHAR(100) NULL,
  description TEXT NULL,
  appear_condition_type VARCHAR(50) NOT NULL,
  appear_condition_value VARCHAR(100) NOT NULL,
  base_weight INT NOT NULL DEFAULT 100,
  talk_text TEXT NOT NULL,
  hint_text TEXT NULL,
  relation_text TEXT NULL,
  related_npc_id INT NULL,
  reward_type VARCHAR(50) NULL,
  reward_value VARCHAR(100) NULL,
  is_active TINYINT NOT NULL DEFAULT 1,
  sort_order INT NOT NULL DEFAULT 0,
  created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP
);
```

---

### 10.2 player_npc_encounters

プレイヤーごとのNPC遭遇状況を管理する。

```sql
CREATE TABLE player_npc_encounters (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  player_id BIGINT NOT NULL,
  npc_id INT NOT NULL,
  first_encountered_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  encounter_count INT NOT NULL DEFAULT 1,
  last_encountered_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  UNIQUE KEY uq_player_npc (player_id, npc_id),
  INDEX idx_player_id (player_id),
  INDEX idx_npc_id (npc_id)
);
```

---

### 10.3 player_tavern_daily_npcs

プレイヤーごとの当日酒場NPCを保存する。

```sql
CREATE TABLE player_tavern_daily_npcs (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  player_id BIGINT NOT NULL,
  tavern_date DATE NOT NULL,
  npc_id INT NOT NULL,
  slot_no INT NOT NULL,
  created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  UNIQUE KEY uq_player_date_slot (player_id, tavern_date, slot_no),
  INDEX idx_player_date (player_id, tavern_date)
);
```

## 11. 出現条件タイプ

Phase 1では以下を実装する。

| condition_type          | 意味        | value例      |
| ----------------------- | --------- | ----------- |
| always                  | 常時出現可能    | 0           |
| total_exploration_count | 累計探索回数    | 100         |
| tavern_visit_count      | 酒場訪問回数    | 10          |
| player_level            | プレイヤーレベル  | 50          |
| reached_city_id         | 指定都市到達    | 3           |
| cleared_city_id         | 指定都市制覇    | 3           |
| mastered_job_id         | 指定職業マスター  | 1           |
| job_rank                | 指定職業ランク以上 | job_id:rank |
| title_count             | 称号獲得数     | 50          |
| hidden_area_count       | 秘境発見数     | 1           |
| rare_drop_count         | レアドロップ獲得数 | 10          |
| defeat_count            | 敗北回数      | 1           |
| arena_win_count         | 闘技場勝利数    | 10          |
| defeated_final_boss     | 魔王撃破済み    | 1           |

`job_rank` の value は以下のように文字列で保持する。

```text
1:5
```

意味：

```text
job_id = 1
rank >= 5
```

## 12. 酒場画面の処理フロー

```text
1. 酒場画面を開く
2. 当日の player_tavern_daily_npcs を確認
3. 既に保存済みなら、そのNPCを表示
4. 未保存なら、条件を満たすNPC候補を取得
5. 抽選重みに基づいて最大3人を抽選
6. player_tavern_daily_npcs に保存
7. 酒場画面に表示
```

---

## 13. NPC会話時の処理フロー

```text
1. NPCを選択
2. npc_master から会話文を取得
3. player_npc_encounters を確認
4. 未遭遇なら新規登録
5. 遭遇済みなら encounter_count +1
6. last_encountered_at を更新
7. 会話文を表示
8. 初遭遇の場合は「冒険者名簿に登録されました」を表示
```

---

## 14. 冒険者名簿の処理フロー

```text
1. npc_master を sort_order 順に取得
2. player_npc_encounters と LEFT JOIN
3. 遭遇済みならNPC名を表示
4. 未遭遇なら「？？？」を表示
5. 登録数 / 全NPC数 を表示
```

## 17. 報酬仕様

Phase 1では報酬なしでよい。

Phase 2以降で以下を追加する。

| reward_type | 内容        |
| ----------- | --------- |
| gold        | Gold付与    |
| item        | アイテム付与    |
| material    | 素材付与      |
| title       | 称号付与      |
| map_piece   | 秘境地図の欠片付与 |
| job_hint    | 職業解放ヒント表示 |
| unlock_flag | 特定フラグ解放   |

報酬は初回遭遇時のみ付与を基本とする。

再遭遇時に毎回報酬を付ける場合は、バランス崩壊を避けるため低価値報酬に限定する。

## 19. バランス方針

NPC出現は、プレイヤーに毎日酒場を見る理由を作るための要素である。

ただし、必須要素にしすぎるとストレスになる。

そのため、Phase 1では攻略必須にしない。

基本は以下の位置づけにする。

```text
知らなくても進める
知っていると少し得をする
集めると世界観が深まる
```

伝説職や秘境解放に使う場合も、いきなり必須条件にせず、まずはヒント提供に留める。

---

## 20. 注意点

* 酒場を開くたびにNPCが変わらないようにする
* 同じ日に同一NPCが複数枠に出ないようにする
* 未遭遇NPCを優先しすぎてレア感が消えないようにする
* 報酬を強くしすぎない
* NPC会話は長文にしすぎない
* スマホ画面では1NPCをカード表示にする
* 未遭遇NPCの名前は「？？？」にする
* 伝説NPCは序盤で出現しないようにする
* NPC図鑑は称号システムと接続しやすい設計にする

## 24. NPCデータ登録時の補足

上記の `出現条件` は、実装時には `appear_condition_type` と `appear_condition_value` に分解して登録する。

例：

| NPC名       | 出現条件         | appear_condition_type   | appear_condition_value                 |
| ---------- | ------------ | ----------------------- | -------------------------------------- |
| 疾風のコジロウ    | 探索10回以上      | total_exploration_count | 10                                     |
| 豪腕のムサシ     | 探索20回以上      | total_exploration_count | 20                                     |
| 放浪のヤマト     | 探索50回以上      | total_exploration_count | 50                                     |
| 酒場娘リリィ     | 酒場訪問10回以上    | tavern_visit_count      | 10                                     |
| 剣聖ハヤト      | 剣士マスター       | mastered_job_id         | 剣士のjob_id                              |
| 魔王軍離反者ヴェルン | 魔王城到達＋ネクロム制覇 | custom_flag             | reached_maou_castle_and_cleared_necrom |

Phase 1では複雑な複合条件を厳密に実装しなくてよい。

複合条件が必要なNPCは、以下のどちらかで対応する。

* `custom_flag` として個別判定を実装する
* Phase 2以降まで非表示にする

## 25. 初期登録時の注意

初期NPC70人は、すべて `is_active = 1` で登録してよい。

ただし、伝説NPCは序盤で出現しないように、必ず重い出現条件を設定する。

特に以下のNPCは、条件が満たされるまで絶対に抽選対象に含めない。

## 27. 再遭遇セリフの扱い

Phase 1では、再遭遇時も同じ `talk_text` を表示してよい。

Phase 2以降で、以下のカラムを追加する。

```sql
ALTER TABLE npc_master
ADD COLUMN repeat_talk_text TEXT NULL AFTER talk_text;
```

再遭遇時は `repeat_talk_text` が設定されていればそれを表示する。
未設定なら `talk_text` を表示する。

## 28. セリフ設計ルール

NPCセリフは以下の方針で統一する。

* 1人あたり1〜3文まで
* 80文字〜140文字程度を目安にする
* NPCの性格が分かる言い回しにする
* 直接的な攻略説明にしすぎない
* ただし、何のヒントかはプレイヤーが分かるようにする
* 伝説NPCほど抽象度を上げる
* 一般NPCほど実用的なヒントにする

区分ごとのセリフ傾向は以下。

| 区分 | セリフ傾向               |
| -- | ------------------- |
| 一般 | 実用的、親しみやすい、序盤ヒント    |
| 熟練 | 判断・育成・職業ヒント         |
| 英雄 | 職業解放、都市制覇、強敵攻略の示唆   |
| 伝説 | 伝説職、裏ダンジョン、世界観核心の示唆 |
