# ドロップ・合成防具のバランス調整実装計画

本番環境の合成・ドロップ防具について、店売り最高峰防具に比べて防御力やHPが高くなりすぎている問題を修正し、適正なゲームバランスに調整（デフレ化）します。また、要求レベルを撤廃し、価格を店売りの1.2倍に設定します。

## ユーザーレビューが必要な箇所

> [!NOTE]
> すでに該当のドロップ・合成防具を所持しているプレイヤーキャラクターがいる場合、この調整を実行すると防御力やHPが大幅に減少します。これは想定通りのデフレ（バランス正常化）調整です。
> また、要求レベルは一律で `1` に設定（撤廃）されるため、レベルに関わらず誰でも装備できる状態を維持します。

## 提案される変更

### 1. バランス調整スクリプトの作成・実行

#### [NEW] [scratch_rebalance_drop_armors.php](file:///c:/Users/yuta/tool/tool/ffa/scratch/rebalance_drop_armors.php)
* データベースの `areas` テーブルからダンジョン名と `city_id` の対応を読み込みます。
* `docs/armor_master.tsv` のうち、ドロップ品・合成品（`is_shop_item == 0`）について以下の調整を行います：
  * **要求レベルの撤廃**: `required_level` をすべて `1` に設定します。
  * **価格の設定**: 該当エリアの都市の店売り基準価格の **1.2倍** に再設定します（100円未満は四捨五入して丸めます）。
  * **売却価格の設定**: 新しい価格の **25% (4分の1)** に設定します。
  * **ステータスの再計算**: 都市の店売り基準価格から適正なベースパワー `basePower` を算出し、サブタイプ（重装、軽装、法衣、外套、旅装、革鎧）ごとのステータス倍率を適用して再計算します。
    * ベースパワー式: `basePower = (10 + (基準価格 * 0.0016)) * 1.2` (ドロップ品としての1.2倍の強さボーナス)
* 再計算したデータを `docs/armor_master.tsv` および `armors_data.tsv` に上書き保存します。

### 2. デプロイとデータベースの再構築
* `local_deploy.php` を実行し、本番サーバーへファイルを転送します。
* `https://valzeria.com/dev/execute-update` を呼び出して `ArmorsSeeder` を実行し、本番データベースのデータを更新します。

## 検証計画

### 手動検証
1. **マスタデータの目視確認**:
   * 生成された `docs/armor_master.tsv` を確認し、例えば `終焉の聖鎧` (EXランク, 元DEF 895) の防御力が **約240**、HPが **約400** 前後、価格が店売り最強の1.2倍（約 90,100 * 1.2 = 108,100 ➔ **108,100 G**）程度になっていることを確認。
   * 要求レベルが `1` になっていることを確認。
2. **本番環境へのデプロイ後の検証**:
   * 一時的な確認ルート等を通じて、本番データベース内の合成防具のパラメータが正しく調整されていることを確認。
