## Recent implementation state

Keep this section short. Current state only, not changelog.

- 補給所は薬草・回復薬・魔力水を各10個/日の無料補給枠で扱い、所持上限で受け取れない分や当日未受取分を補給所に保管中として表示し、持ち越し分/本日ストック/今日の未受取の内訳も表示する。
- 補給商会では探索力回復商品を販売する。探索力の小瓶は輝石5（旧10）で+50、1日3個まで。探索力の薬は輝石12（旧25）で+150、1日2個まで。購入時は `character_consumable_items` の所持数に入り、倉庫の所持品欄から使用した時に探索力を回復する。使用による回復分は探索力上限を超えて蓄積できる。自然回復は上限未満の時だけ進む。素材倉庫Gold拡張は10万Goldで素材倉庫を+50し、最大100回まで購入できる。
- 敵ドロップ武器・防具のうち `items.affix_enabled=true` のものは、ドロップ時に能力銘/種族銘/良品/逸品を確率抽選する。銘補正は `character_items` の個体カラムに保存し、装備中のみステータスへ加算する。武器の種族銘はPvEの敵への与ダメージ増加、防具の防護銘はPvEの敵からの被ダメージ軽減として適用し、PvP/チャンプ戦には適用しない。
- 装備進化合成で消費する進化元に銘付き装備が含まれる場合、接頭辞/接尾辞、能力補正、種族特効/耐性、良品/逸品の品質は進化後の装備個体へ引き継ぐ。+強化値は従来どおり進化後にリセットする。
- 通常探索の汎用装備ドロップ率は標準で武器1.5%、防具1%、装飾0.5%を目安に抑制している。敵固有装備ドロップもマスタ/既存DBともに初期値の約半分に調整している。
- 下部チャットの個別チャット/手紙ログは本人に関係するものだけ取得し、全体タブには表示せず個人(手紙)タブだけに表示する。ホーム画面だけでなく、戦闘/チャンプ戦/ランク戦/決闘系の `facility` レイアウト画面下部にも同じ `ChatLog` を表示する。戦闘結果テンプレートはタイトルが「入口発見」「探索を継続」に変わっても戦闘結果レイアウトとして扱い、チャット欄と「戦利品を持って帰る」帰還ボタンを維持する。
- 管理画面の「管理人チャット」から `type=admin` の公開ログを投稿でき、プレイヤー側の全体チャットには管理人メッセージとしてヴァルゼリアブルーで表示する。同画面には直近活動中の冒険者名も表示する。
- 管理画面の `/admin/public-logs` では、下部チャットに表示される `public_logs` を種別・日付・本文/名前で検索し、単体または複数選択で削除できる。管理人メッセージ（`type=admin`）は通常の選択削除から保護し、種別を「管理人」に絞って保護解除した場合だけ削除できる。
- 管理画面の `/admin/operator-analytics` では、新規登録、既存ログから推定した活動者、通常戦闘/チャンプ戦/ランク戦、チャット投稿、fulfilled売上の日別推移、7/14/30日伸び率、CSV出力を確認できる。厳密な日別ログイン履歴は未作成のため、活動者は既存ログと `characters.last_seen_at` に基づく推定値として表示する。
- 管理画面の `/admin/inn-analytics` では、`gold_transactions.type=inn` の支払い履歴から宿屋の期間売上、宿泊回数、利用者数、平均宿代、累計宿屋売上、最近の宿泊履歴を確認できる。宿泊時のGold取引メタデータには今後分の `fee`、`paid`、`rescued`、`level` を保存し、救済宿泊の内訳はメタデータ記録後の宿泊分から反映する。
- 新規キャラクター作成時は、全体チャットに `info` ログとして「新しい冒険者『名前』がヴァルゼリアの地に降り立ちました。」を1回流す。公開するのはキャラクター名のみで、ユーザー名やメールアドレスは含めない。
- 2026/06/30以降に登録したユーザーのキャラクターには、7月登録キャンペーンとして探索力の小瓶10個と「7月登録キャンペーンを受け取りました」通知を自動付与する。管理画面の `/admin/player-controls` では対象/配布済/未配布人数を表示し、画面表示時に未配布者も同期する。同じ通知タイプ `newcomer_stamina_bottle_gift` があるキャラクターは配布済みとしてスキップする。
- Stripe決済完了時は購入者に購入完了メール、運営に売上通知メールを送る。運営通知の宛先は `ADMIN_EMAIL` を優先し、未設定時は `CONTACT_MAIL_ADDRESS` に送る。実送信には本番環境の `MAIL_MAILER=smtp` 等のメール設定が必要。
- 管理ダッシュボードは `config/admin_update_summaries.php` の最新50件を「最近の更新情報」として表示し、今後の意味ある実装タスクは必要に応じて同ファイルへ運営向けサマリを追記する。
- 装備ドロップの公開ログはSSS/EPIC、または銘抽選で「逸品」になった装備を対象にする。通常のAランク装備獲得だけでは下部の全体チャットへ流さない。
- プレイヤーLv到達の成長公開ログはLv100以上の節目だけを下部の全体チャットへ流し、成長ログの表示色は通常ログと同じ控えめな色にする。
- 管理画面の `/admin/npc-market-analytics` では、NPCごとの調達納品量、現在在庫、出品中数量、販売済み数量、販売額、素材別明細、直近販売履歴を確認できる。
- ヘルプページと街の案内所は `HelpContentService` 経由で `config/help_content.php` の初期文言を表示し、管理画面の「ヘルプ文言管理」から `game_texts` の上書き文言を編集できる。
- 素材倉庫の標準上限は500個。素材倉庫+500拡張は50輝石、装備倉庫+100拡張は50輝石で、追加分は標準上限に加算される。素材倉庫Gold拡張は10万Goldで素材倉庫を+50し、最大100回まで購入できる。過去に倉庫+50拡張を購入済みの冒険者には、素材倉庫は購入1回あたり差分+450、装備倉庫は購入1回あたり差分+50を補填する。
- アイテム図鑑 `/item-book` では、装備品合成・鍛冶/素材交換・敵ドロップなど現在のプレイで必要な素材だけを所持/未所持問わず確認できる。廃止済み素材は、交換レシピが残っていても表示しない。入手方法、素材交換所の作り方、用途、主なドロップ元を表示し、素材整理の都合を説明する文言はプレイヤー画面に出さない。素材交換所の全レシピ参照は `MaterialExchangeService::catalogRecipes()` を使い、交換実行ロジックとは分離する。素材カードはアコーディオン表示で、閉じた状態ではアイコン、素材名、カテゴリ、所持状況、作成可否、ドロップ有無のバッジを表示する。用途欄に図鑑内の具体素材名が出る場合は、その素材カードへ同ページ内で移動できる。薬草などの回復アイテムを素材交換所で作る回復調合は、探索継続力への影響が大きいため一時停止している。
- 鍛冶屋では武器・防具・装飾品を最大+5まで強化できる。武器・防具は従来どおり基礎性能+3%目安か最低+強化値分で伸び、装飾品は+1ごとに装飾品全体で合計+2の追加値を元の非ゼロ能力値比率で配分する。+1/+2は欠片中心、+3は石+欠片、+4は高純度石・石・共通素材・粗精錬核、+5は高純度石・石・共通素材・精錬核を要求する。鍛冶Goldは +1〜+5 の基本費用100G/500G/1,500G/5,000G/15,000Gに装備ランク倍率（G1.0、F1.2、E1.5、D2.0、C3.0、B5.0、A8.0、S15.0、SS30.0、SSS60.0、EPIC120.0）を掛ける。粗精錬核は素材交換所で織成核殻、晶糸核芯、魔物の欠片20個、5,000Gから作り、織成核殻は王都の織布、潮風の布片、精霊樹の繊維、黒鉄の装甲片を各10個、晶糸核芯は氷晶の織糸、砂金繊維、魔導繊維を各10個で作る。粗精錬核系の3レシピは素材不足でも素材交換所に表示する。素材交換所では敵固有素材を獣牙・小鬼の牙・獣の毛皮・魔鉱片などの共通素材へ圧縮でき、牙/爪/小鬼/ゴブリン系は汎用牙素材として獣牙へ寄せる。共通素材+100Gから強化石の欠片/守護石の欠片/調律石の欠片を合成できる。敵固有素材の共通化では、刻印、王印、神印、進化証、英雄の証、討伐証、導石、古代片、秘境晶、極印、ボス特異素材、進行キーを含む素材名は対象外にし、変換元素材名と交換先素材名が同じ自己変換レシピは作らない。黒結晶/呪い/闇/瘴気/腐、聖片/神殿/天使/司祭/聖、魔核/魔導炉/炉心/核は、鉱片/魔鉱/水晶/結晶/雷石/火種/鉱石より先に分類する。欠片20個+500Gを対応する石1個へ精製でき、高純度石は石5個+魔鉱片10個+2,000Gで精製する。精錬核は、粗精錬核1個、魔物の魔核20個、覇王黒晶、蒼炉魔晶、星樹氷晶を各1個、10,000Gで素材交換所から作る。覇王黒晶は王紋鋼・砂王金晶・ヴァルゼリア黒核を各5個、蒼炉魔晶は海鳴りの蒼鉱・炉心鋼・ルミナス魔晶・深魔骨核を各5個、星樹氷晶は精霊樹の琥珀・氷帝晶・セレスティア星晶を各5個で作る。素材交換所の受け取り素材名は詳細ページへ遷移せず、用途・必要素材・所持数をモーダルで表示する。敵ドロップの強化系補助枠では強化石の欠片・守護石の欠片・調律石の欠片だけを、各通常ダンジョンの約半数の通常敵から1%で許可し、石・高純度石・精錬核系素材は敵ドロップさせない。
- 装備の分岐進化S→SSは、各系統の古代片 x5 と素材交換所で作る複合素材 x3 を要求する。複合素材は聖天織晶、冥黒織晶、翠嵐織晶、鋼氷護晶、星導魔晶、風精織晶、砂海旅晶の7種で、都市素材・高位都市素材・地域素材を組み合わせて錬成する。聖天織晶は王都の守護布、冥黒織晶は瘴気の革片、翠嵐織晶は天空竜の織布も要求する。武器・防具のSS→SSS分岐進化は、対応する秘境晶 x20 を要求する。装飾品のSS→SSS進化は、聖剣・魔剣・迅刃・重装・魔装・軽装・旅装の秘境晶を各2個ずつ使って素材交換所で作る原初装飾晶 x1 を要求し、裏ダンジョンボス撃破条件は要求しない。
- プレイヤー向けの探索目安は推奨Lvではなく `CharacterPowerService` の目安戦力で表示する。上部ステータスバー、左サイドバー、チャンプカードには現在戦力を表示し、通常ダンジョンの目安は実際に出る通常敵（雑魚/やや強い）の戦力範囲から算出する。内部の推奨Lvは敵生成・深度補正・通常敵データがない表示のフォールバック用に残す。戦力計算ではSP/MPを補助評価として控えめに扱い、AGIは最大35%まで緩やかに火力補正する。
- 酒場NPCは `npc_id` に対応する `public/images/npc/npc_001.webp`〜`npc_070.webp` をキャラ画像として表示する。未遭遇NPCは名簿で画像を伏せる。
- NPC調達依頼は酒場NPC本人の `npc_id` に紐づく。納品された素材は `npc_material_stocks` にNPC在庫として保存され、`market:generate-npc-listings` が在庫からNPC市場出品を作る。NPC出品は買う一覧の在庫・最安値に通常出品として混ざり、内部では `market_listings.seller_type=npc` / `seller_npc_id` でプレイヤー出品と区別する。市場出品NPCは `npc_rank=hero/legend` を除外し、購入時はプレイヤー売上通知や売上Goldを発生させない。
- 神殿の転職ページでは、職業カードの詳細モーダルから特徴、職業管理の成長倍率に基づく伸びやすい能力、覚える奥義、マスター恩恵、必要条件を確認できる。未使用BPが残っている場合は転職不可で、転職前にすべて能力へ割り振る必要がある。転職可能な職業は詳細モーダルから既存の転職確認へ進める。奥義は `skills.skill_type=job_art`、職業固有の従来必殺技は `job_classes.skill_id` 経由の別系統として戦闘中に発動し、消費SPは基本消費の半分になる。奥義セットは通常戦用3枠とボス戦用3枠を別々に保存し、通常探索では通常戦セット、ボス挑戦ではボス戦セットを使う。発動方針はセット全体ではなく各スロットに `積極` / `通常` / `温存` を保存する。
- 職業ランクは全職共通で10がマスター上限。必要職業EXPは基本職1倍、中級職2倍、上級職5倍、伝説職10倍で補正する。1回の報酬処理で付与される職業EXPは、探索深度・亜域・チャンプ戦などの補正後も最大3に抑える。
- 戦闘結果では獲得EXP/Job EXPに加え、次のプレイヤーLvまでの残りEXPと、現在職の次ランクまでの残りJob EXPを表示する。職業はプレイヤー向けにはレベルではなくランクとして扱う。
- 通常探索の連戦待機は表層10秒、深部15秒、深層/最深層/異界層20秒。深部/深層/最深層/異界層では、階層入場時に危険度を0%へ戻し、次階層へ進むために危険度を再蓄積する。敵ステータスは深度帯の目標Lvに対する種族・派生・役割ベースの敵ステータス生成式を下限として補正する。
- 通常探索勝利時は、低確率で直近30分以内に活動している同じ街の別冒険者とすれ違う「出会い」ログを探索結果に表示し、薬草を1個付与する。出会いは結果画面に相手のキャラ画像、Lv、職業、受け取った薬草も表示する。受け取った薬草は倉庫の所持品欄に表示し、探索中の持ち込み回復アイテムが初期化済みなら現在探索の残数にも反映する。戦闘、進行、チャット通知には影響しない。候補がいない場合は発生しない。
