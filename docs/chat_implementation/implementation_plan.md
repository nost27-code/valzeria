# チャット機能実装計画

## 概要
現在UIのみが存在しているチャットエリア（`Livewire\ChatLog`）に対し、実際にプレイヤーが発言し、それがログに反映される機能を実装します。

## 実装内容
1. **Livewireコンポーネントの改修** (`app/Livewire/ChatLog.php`)
   - 入力用プロパティ (`$message`) の追加
   - 送信メソッド (`sendMessage`) の実装
   - `PublicLogService` を使用して、入力されたテキストを `type: 'chat'` としてデータベースに保存。
   - 保存形式は「`キャラクター名：メッセージ`」のように発言者がわかるようにします。
2. **UIビューの改修** (`resources/views/livewire/chat-log.blade.php`)
   - 既存の input タグに `wire:model="message"` を紐付け
   - 既存の発言ボタンに `wire:click="sendMessage"` または form の submit イベントを紐付け
   - Enterキーでの送信対応（`wire:keydown.enter="sendMessage"`）
   - 最新のチャットが自動で表示されるように、Livewireのポーリング（`wire:poll` またはそれに準ずる定期更新）を追加。

## オープンクエスチョン（ユーザー確認事項）
> [!IMPORTANT]
> 1. 現在UI上に「全体」「ギルド」「個人」の宛先セレクトボックスがありますが、今回の実装ではまず**「全体チャット」のみ**を実装し、ギルドや個人チャットは今後の機能追加時に対応する形でよろしいでしょうか？
> 2. スパム投稿（連投）を防ぐための簡易的なクールタイム（例：1回の発言につき5秒間は発言できない等）は導入しますか？（ひとまず無しで、自由に発言できる形でも大丈夫です）
